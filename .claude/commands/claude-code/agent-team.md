# Agent Team Orchestrator

引数で指定されたタスクを、複数のエージェントをチームとして協調させて完遂します。

## タスク

$ARGUMENTS

## コンテキスト

- 現在のブランチ: !`git branch --show-current`
- リポジトリの状態: !`git status --short`

## 実行フロー

以下の4フェーズで段階的に実行してください。

---

### ⚠️ Phase 0: 必須前提（スキップ禁止）

**必ず最初に TeamCreate でチームを作成すること。**
TeamCreate なしでタスクを直接実行することは禁止です。

```
TeamCreate:
  team_name: "agent-team-{タスク名の短縮}"
  description: "タスクの説明"
```

- タスク完了後は必ず **TeamDelete** でチームをクリーンアップすること
- サブエージェントへの指示・進捗確認は **SendMessage** で行うこと（テキスト出力だけでは伝わらない）

---

### Phase 1: タスク分析・計画（Plan エージェント）

Task tool で `Plan` サブエージェントを起動し、以下を実行：

1. タスクの全体像を把握する（コードベースの探索、関連ファイルの特定）
2. タスクを**独立して実行可能なサブタスク**に分解する
3. 各サブタスクについて以下を決定：
   - サブタスクの内容と目標
   - 使用すべきサブエージェントタイプまたはスキル名
   - 依存関係（他のサブタスクの完了を待つ必要があるか）
   - 推定される作業範囲（ファイル数、変更内容）
4. 並列実行可能なグループと、順次実行が必要なグループを明確に区分する
5. **利用可能なスキル・サブエージェントを探索し、各サブタスクにマッチングする**
   - `Glob: .claude/skills/**/SKILL.md`（スキル一覧）
   - `Glob: .claude/agents/**/*.md`（サブエージェント一覧）
   - マッチング優先度：直接マッチ > 工程別 > キーワード > general-purpose

Plan エージェントへのプロンプトには、タスクの説明（$ARGUMENTS）とコンテキスト情報を含めること。

---

### Phase 2: 並列実行（チームベース）

Plan エージェントの計画結果に基づき、以下の手順で実行：

1. **TeamCreate** でチームを作成（Phase 0 で未作成の場合はここで作成）
2. **TaskCreate** で全サブタスクをTODOリストに登録（依存関係も設定）
3. **並列実行可能なサブタスク群**を特定し、Task tool で**同時に複数のサブエージェントを起動**
   - Task 呼び出し時に `team_name` と `name` パラメータを指定してチームに参加させる
   - 独立したサブタスクは必ず**1つのメッセージ内で複数のTask呼び出し**を行い並列化
   - 依存関係があるサブタスクは前のグループ完了後に順次起動
4. 各サブエージェントへの指示には以下を明記：
   - 担当するサブタスクの内容
   - 期待される成果物（コード変更、調査結果、テスト結果など）
   - 参照すべきファイルや前のフェーズの結果
   - **割り当てられたスキル名**（マッチングがあれば `Skill` ツールで使用するよう指示）
5. サブエージェント完了後、**TaskUpdate** でTODOを更新

#### サブエージェントタイプ・スキルの使い分け

| 優先度 | 判定方法 | 説明 |
|--------|---------|------|
| 1 | 直接マッチ | TODOキーワードがスキルdescriptionに含まれる → そのスキルを使用 |
| 2 | 工程別 | テスト実行 → `api-test-fixer`、品質チェック → `sail-check-fixer` |
| 3 | キーワード | `api-*` / `admin-*` / `*fixer*` 等のパターンでマッチ |
| 4 | general-purpose | 上記にマッチしない場合のみ（全ツール使用可能） |

**補足: サブエージェントタイプの特性**

| タイプ | 用途 | 特徴 |
|--------|------|------|
| `Explore` | コード探索・調査・分析 | 高速、読み取り専用、Haiku使用 |
| `general-purpose` | コード修正・実装・複雑な推論 | 全ツール使用可能 |
| `Bash` | コマンド実行・ビルド・テスト | ターミナル操作専門 |

---

### Phase 3: 結果統合・完了確認

全サブエージェントの完了後、メインセッションで：

1. 各サブエージェントの成果を確認・統合
2. サブタスク間の整合性を検証（コンフリクトがないか、インターフェースが一致しているか）
3. 必要に応じて追加の修正を実施
4. **TaskList** で全TODOが完了していることを確認
5. **SendMessage(type: shutdown_request)** で各メンバーを終了
6. **TeamDelete** でチームをクリーンアップ
7. ユーザーに最終結果のサマリーを報告

---

## 制約事項

- 各サブエージェントが同じファイルを同時に編集しないよう、ファイルの担当を明確に分ける
- サブエージェントの数は3〜5に制限（コスト効率のため）
- 調査のみのタスクには `Explore`（Haiku）を使いコストを抑える
- 破壊的な操作（ファイル削除、force push等）はメインセッションで確認してから実行
- **TeamCreate は必ず実行すること**（「小さなタスクだから不要」という判断は禁止）
