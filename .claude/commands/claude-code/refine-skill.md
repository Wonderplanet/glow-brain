# refine-skill: 使用スキルをフィードバック反映で改善

本セッションで使用したスキルに対してユーザーが行った指摘・修正要求を分析し、
スキルファイルを修正して今後の精度・効率を向上させます。

## ステップ

### Step 0: Plan Modeへ移行

最初に `EnterPlanMode` ツールを呼び出してください。

### Step 1: セッション振り返りと分析

本会話の履歴を振り返り、以下を整理してください：

1. **使用されたスキル一覧**
   - 本セッションで Skill ツールを使って起動されたスキル名を列挙する
   - スキル名は `.claude/skills/{skill-name}/` ディレクトリに対応

2. **指摘・修正要求の特定**
   - ユーザーが「〜が違う」「〜してほしかった」「〜がうまくいかない」などと述べたやり取りを特定
   - エラーや手戻りが発生した箇所も含める

3. **指摘の本質を整理**
   - 何が問題だったか（誤った前提・抜け漏れ・手順の不備・説明不足 など）
   - 修正対象スキルはどれか（複数可）

指摘が見当たらない場合は「本セッションではスキルへの指摘が確認できませんでした」とユーザーに報告し、終了してください。

### Step 2: 対象スキルファイルの調査

特定したスキルごとに以下を調査してください：

1. **SKILL.md の読み込み**
   - パス: `.claude/skills/{skill-name}/SKILL.md`
   - 問題に関連するセクションを特定する

2. **references/ の調査（存在する場合）**
   - パス: `.claude/skills/{skill-name}/references/`
   - 指摘内容と関連するドキュメントを特定する

3. **問題箇所の特定**
   - どのセクション・記述が誤解や抜け漏れを引き起こしたか
   - 追加・修正・削除すべき内容を明確にする

### Step 3: 修正計画の立案と承認

調査結果をもとに修正方針を決定し、`ExitPlanMode` で計画を提示してください。

計画には以下を含めること：

- **対象ファイル**: 修正するファイルのパス一覧
- **修正内容**: セクションごとの変更概要（何を追加・変更・削除するか）
- **修正の根拠**: ユーザーの指摘のどの部分に対応するか

ユーザーの承認を得てから次のステップに進んでください。

### Step 4: スキルファイルの修正

承認後、Edit または Write ツールで対象ファイルを修正してください。

修正完了後、ユーザーに以下を報告します：

- 修正したファイルの一覧
- 各ファイルの変更概要
- 次回スキル使用時に期待される改善点

## 注意事項

- **修正対象はスキルファイルのみ**: `.claude/skills/` 以下のファイルのみ変更する
- **最小限の変更**: 指摘に直結しない箇所は変更しない（過剰な改変を避ける）
- **既存構造を尊重**: SKILL.md のフォーマット・セクション構成を維持する
