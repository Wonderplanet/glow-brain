# Claude Code ガイド相談

Claude Code の公式ドキュメントに基づいた、正確で説得力のある情報を提供する専門ガイドエージェントを起動します。

## 引数

- `$ALL`: （オプション）最初に相談したい質問内容。省略した場合は対話的に相談を開始します。

## タスク

このコマンドは、claude-code-guide subagent を呼び出して、以下のステップで進めます：

### 1. **ガイドエージェントの起動**

Task ツールを使って `claude-code-guide` subagent を起動してください：

**相談内容がある場合**（$ALL が指定されている場合）:
```
Claude Codeに関する以下の質問について、公式ドキュメントから正確な情報を抽出して回答してください：

$ALL

回答の際は：
- 公式ドキュメントから必ず引用して根拠を明確にする
- 具体的なコード例や設定例を含める
- ベストプラクティスと注意点を明記する
- 関連する機能やドキュメントへの参照も提供する
```

**対話的に開始する場合**（$ALL が空の場合）:
```
Claude Codeについて対話的に相談します。以下の流れで進めてください：

1. 相談できるトピック（hooks, skills, MCP servers, settings, API usage など）を簡潔に紹介
2. ユーザーの質問に対して、公式ドキュメントから正確な情報を抽出して回答
3. 各回答では：
   - 公式ドキュメントから必ず引用して根拠を明確にする
   - 具体的なコード例や設定例を含める
   - ベストプラクティスと注意点を明記する
   - 関連する機能やドキュメントへの参照も提供する

相談は自然な会話形式で進め、ユーザーが理解できるまで丁寧に説明してください。
```

### 2. **相談セッションの進行**

エージェントとの相談を進めてください。以下のポイントに注意：

- **正確性**: 公式ドキュメントからの引用を必ず含める
- **具体性**: 実用的なコード例や設定例を求める
- **網羅性**: 関連する機能やベストプラクティスも確認する
- **理解度**: 不明点があれば追加質問をする

### 3. **相談結果のレポート作成**

相談がある程度まとまったら、以下の形式で `.claude/docs/guide-chat-reports/` ディレクトリにレポートを自動保存してください：

**ファイル名**: `yyyymmdd_[日本語トピック名].md`

例: `20251227_hooksの設定方法.md`, `20251228_MCPサーバー作成.md`

**レポート形式**:

```markdown
# Claude Code ガイド相談レポート

**日付**: YYYY年MM月DD日
**トピック**: [相談の主なトピック]

## 相談内容

[相談した質問や議論のテーマを箇条書きで]

## 重要な結論・学び

### [トピック1]

[公式ドキュメントからの引用と説明]

**コード例**:
```[言語]
[具体的なコード例]
```

**ベストプラクティス**:
- [ポイント1]
- [ポイント2]

**注意点**:
- [注意すべき点]

### [トピック2]

[同様の形式で記載]

## 参照ドキュメント

- [公式ドキュメントへのリンクや参照パス]

## 次のアクション

[今後試してみたいこと、さらに調べるべきことなど]

---

*このレポートは `/guide-chat` コマンドにより自動生成されました*
```

### 4. **レポート保存の自動化**

エージェントとの相談が完了したら、以下の手順でレポートを保存：

1. `.claude/docs/guide-chat-reports/` ディレクトリを作成（存在しない場合）
2. 相談内容を上記の形式でまとめる
3. `yyyymmdd_日本語トピック名.md` 形式でファイル名を生成して保存
4. ユーザーに保存場所を報告

## 出力形式

### 相談開始時

```
🔍 Claude Code ガイドエージェントを起動しています...

[エージェントとの対話が始まります]
```

### 相談終了時

```
✅ 相談が完了しました

📝 レポートを保存しました: .claude/docs/guide-chat-reports/yyyymmdd_[トピック].md

今回の相談で学んだ主なポイント:
- [ポイント1]
- [ポイント2]
- [ポイント3]

このレポートは今後の開発作業で参照できます。
```

## 注意事項

- **公式情報優先**: 必ず公式ドキュメントから情報を抽出し、推測は避ける
- **引用の明示**: 回答には必ず出典を含める
- **実践的な内容**: 理論だけでなく、実用的なコード例を含める
- **レポート品質**: 後で読み返したときに有用な情報となるよう、詳細に記録する
- **継続的な学習**: 保存されたレポートは `.claude/docs/guide-chat-reports/` に蓄積され、ナレッジベースとなる

## 使用例

### 基本的な使用（対話的）

```
/guide-chat
```

対話形式でトピックを選択し、相談を進めます。

### 特定の質問から開始

```
/guide-chat hooksの設定方法と使い分けについて教えてください
```

指定した質問から相談を開始します。

```
/guide-chat MCPサーバーの作成方法とベストプラクティス
```

MCP サーバーに関する詳細な相談を開始します。

## レポートの活用

保存されたレポートは以下のように活用できます：

1. **過去の相談内容の参照**: `.claude/docs/guide-chat-reports/` を検索
2. **ナレッジベースとして**: 似たような疑問が生じた際に参照
3. **チーム共有**: レポートをチームメンバーと共有して知識を共有
4. **プロジェクトドキュメント**: プロジェクト固有の Claude Code 活用方法をドキュメント化
