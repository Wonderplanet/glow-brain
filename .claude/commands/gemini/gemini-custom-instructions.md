# Geminiカスタム指示ジェネレーター

Google Geminiのカスタムgem用の高品質なカスタム指示を生成します。公式のベストプラクティスに準拠し、効果的で明確な指示文を作成します。

**注意**: このコマンドでは、Claude Codeの機能(skills, subagents, commands等)は使用できません。純粋にGemini用のテキストベースの指示を生成します。

## ユーザーの要件

```
$@
```

## 公式リファレンス

このコマンドは以下のGoogle公式ガイドラインに準拠しています:

- **Tips for creating custom Gems**: https://support.google.com/gemini/answer/15235603?hl=en
- **How to use Gems**: https://support.google.com/gemini/answer/15236405?hl=en
- **Get started with Gems in Gemini Apps**: https://support.google.com/gemini/answer/15236321?hl=en

## タスク

以下のステップで、Geminiカスタム指示を生成してください:

### 1. 要件の分析と確認

まず、ユーザーから提供された要件(`$@`)を分析し、以下を明確にします:

- **目的**: このGemは何を達成するためのものか
- **対象ユーザー**: 誰が使用するか（一般ユーザー、専門家、学生など）
- **主要なタスク**: どのような作業を自動化・支援するか
- **出力形式**: どのような形で結果を返すべきか
- **トーン**: フォーマル、カジュアル、教育的など

必要に応じて、AskUserQuestionツールを使用して追加の詳細を確認してください。

### 2. 4つの要素フレームワークに基づく指示文の構成

Google公式が推奨する**4つの要素フレームワーク**に従って指示文を構成します:

#### A. ペルソナ (Persona)
- **役割と専門性を定義**: 「あなたは〇〇の専門家です」
- **トーンとスタイルを設定**: フレンドリー、プロフェッショナル、教育的など
- **制約や特性を明示**: 「簡潔に答える」「初心者向けに説明する」など

#### B. タスク (Task)
- **具体的な行動を明記**: 「〇〇を分析する」「△△を生成する」
- **主要な責任を列挙**: このGemが実行すべき具体的なタスク
- **範囲を明確化**: 何をするか、何をしないか

#### C. コンテキスト (Context)
- **背景情報を提供**: なぜこのGemが必要か
- **使用シーン**: どのような状況で使われるか
- **前提知識**: ユーザーがどの程度の知識を持っているか
- **参照情報**: 関連する概念、用語、ドメイン知識

#### D. フォーマット (Format)
- **出力構造を定義**: 箇条書き、段落、表、コードブロックなど
- **セクション構成**: どのような順序で情報を提示するか
- **長さの指定**: 簡潔に/詳細に
- **必須要素**: 常に含めるべき情報

### 3. ベストプラクティスの適用

以下の公式ベストプラクティスを適用します:

#### 明確さと詳細性
- **具体的に書く**: 曖昧な表現を避け、具体的な指示を与える
- **例を含める**: 期待される出力の例を示す
- **制約を明示**: 「〜しない」「〜を避ける」も明記

#### 構造化と可読性
- **セクション分け**: 見出しや箇条書きを使用
- **優先順位**: 重要な指示を先に配置
- **段階的な指示**: ステップバイステップで説明

#### テスト可能性
- **検証可能な指示**: 「良い結果」ではなく「3つの選択肢を提示」
- **一貫性**: 同じ入力に対して似た出力が得られるように

#### Claude Code機能の除外
- **スキル、サブエージェント、カスタムコマンドへの言及を避ける**
- **Geminiのネイティブ機能のみを使用**: テキスト生成、分析、要約など
- **ファイルアップロード機能**: Gemini Gemsは参照ファイルをサポート

### 4. カスタム指示文の生成

上記の要素を統合し、以下の構造でカスタム指示文を生成してください:

```
# [Gem名]

## ペルソナ
[役割、専門性、トーン、スタイルを定義]

## あなたのタスク
[具体的に実行すべきことを列挙]

1. **[主要タスク1]**
   - [詳細な説明]
   - [具体的な手順]

2. **[主要タスク2]**
   - [詳細な説明]
   - [具体的な手順]

[必要に応じて追加]

## コンテキストと前提
[背景情報、使用シーン、前提知識を説明]

## 出力形式

[期待される出力の構造を明記]

### 例
[具体的な出力例を1-2個提示]

## 重要な注意事項

[制約、避けるべきこと、品質基準を明記]

- **[注意点1]**: [説明]
- **[注意点2]**: [説明]

## 参考リソース（オプション）

[必要に応じて、関連する概念や用語の説明を追加]
```

### 5. 品質チェック

生成した指示文が以下の基準を満たしているか確認してください:

- [ ] 4つの要素（ペルソナ、タスク、コンテキスト、フォーマット）がすべて含まれている
- [ ] 具体的で明確な指示になっている
- [ ] 曖昧な表現が排除されている
- [ ] 出力形式が明確に定義されている
- [ ] 例が含まれている（可能な場合）
- [ ] 制約や注意事項が明記されている
- [ ] Claude Code固有の機能（skills, subagents, commands）への言及がない
- [ ] Geminiのネイティブ機能のみを使用している
- [ ] 読みやすく、構造化されている

### 6. 使用方法ガイドの提示

生成したカスタム指示文をユーザーに提示し、以下の使用方法を説明してください:

#### Gemini Gemsでの使用手順

1. **Gemini Appsにアクセス**: https://gemini.google.com/
2. **Gems作成画面を開く**: 左側のメニューから「Gems」を選択
3. **新しいGemを作成**: 「Create new」または「+ New Gem」をクリック
4. **指示文を貼り付け**: 生成した指示文を「Instructions」フィールドに貼り付け
5. **名前を設定**: Gemに分かりやすい名前を付ける
6. **プレビューでテスト**: 右側のプレビューウィンドウでテスト
7. **保存**: 問題なければ「Save」をクリック

#### 改善のヒント

- **テストと反復**: 実際に使ってみて、必要に応じて指示文を調整
- **例を追加**: より多くの具体例を追加すると精度が向上
- **参照ファイルの活用**: 関連ドキュメントをGemにアップロードして精度向上
- **Geminiの支援機能**: 魔法の杖アイコンを使ってGemini自身に指示文の改善を依頼

## 注意事項

### Claude CodeとGemini Gemsの違い

- **ツールアクセス**: Gemini Gemsはファイルシステムやbashコマンドに直接アクセスできません
- **カスタム機能**: Claude Codeのskills、subagents、commandsのような拡張機能は使用できません
- **対話形式**: テキストベースの対話に限定されます
- **参照ファイル**: Gemini Gemsは参照ファイルのアップロードをサポートしています

### 公式ガイドラインの遵守

Google公式のベストプラクティスに準拠することで、以下のメリットがあります:

- **一貫性のある出力**: 同じ入力に対して安定した結果
- **保守性**: 後から指示文を理解しやすい
- **拡張性**: 新しい要件を追加しやすい
- **効率性**: 繰り返しタスクの自動化に最適
