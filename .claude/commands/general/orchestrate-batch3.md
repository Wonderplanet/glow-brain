# 3並列バッチ オーケストレーター

## 引数

$ARGUMENTS

---

## あなたの役割

あなたはこのセッションにおける**オーケストレーター専任**です。

- 実際の作業はすべてサブエージェント（Task tool）に委譲する
- メインセッションはコーディネーション・進捗管理・品質確認のみ行う
- 各バッチのサブエージェント全員の完了を待ってから次バッチへ進む

---

## Step 0: 現状スキャンと実行計画の立案

### 0-1. タスク分析

`$ARGUMENTS` を読み込み、以下を明確化する:

| 項目 | 内容 |
|------|------|
| **フェーズ構成** | このタスクは何フェーズに分かれるか |
| **対象リスト** | 各フェーズで処理すべき項目の全リスト |
| **完了条件** | 各項目・各フェーズの完了をどのファイル/状態で判定するか |
| **作業ディレクトリ** | サブエージェントに渡すベースパス |

### 0-2. 現状スキャン

Glob / Grep / Read ツールで関連ディレクトリをスキャンし:

- **完了済み項目** → スキップリストに追加
- **未完了項目** → 処理対象リストに追加

### 0-3. 実行計画の出力

以下の形式で計画をユーザーに提示してから実行に入る:

```
【実行計画】
総対象: XX項目
完了済みスキップ: XX項目
実行対象: XX項目

Phase 1: [フェーズ名] — XX項目 → XXラウンド（3並列）
Phase 2: [フェーズ名] — XX項目 → XXラウンド（3並列）
...
最終Phase: [完了確認/統合処理]

作業ディレクトリ: [パス]
```

---

## Step 1〜N: フェーズごとの3並列バッチ実行

各フェーズで以下を繰り返す。

### バッチ実行ルール

1. **未完了項目を3つずつグループ化**（残り3未満の場合はそのまま）
2. **1メッセージで Task tool を並列起動**（3つ同時 or それ未満）
3. **全サブエージェントの完了を待機**してから次バッチへ
4. **"DONE: {項目ID}"** を確認して完了判定
5. エラーが発生した項目は個別リトライキューに追加

### サブエージェント指示テンプレート

```
[タスク固有の作業指示をここに記載]
- 対象: {項目ID}
- 作業内容: {このフェーズで行う具体的な作業}

完了したら "DONE: {項目ID}" とだけ出力してください。

作業ディレクトリ: {ベースパス}
```

**Task tool パラメータ**:
- `subagent_type`: `general-purpose`
- 3項目分を同一メッセージ内で並列起動（依存関係がない場合）
- フェーズ内で順次処理が必要な場合は同一サブエージェント内で順次実行

### フェーズ完了判定

全項目の "DONE" 確認後、Glob/Read で完了条件ファイルを検証してからフェーズを閉じ、次フェーズへ進む。

---

## 最終Step: 完了確認とリトライ

### 完了サマリー出力

```
【完了サマリー】
Phase 1: XX/XX 完了
Phase 2: XX/XX 完了
...
エラー項目: XX件（リトライ対象）
```

### エラー項目のリトライ

エラーが発生した項目は個別にサブエージェントを起動してリトライする。
複数エラーがある場合も3並列バッチルールを適用する。

### 最終確認

すべての完了条件ファイルが存在することを Glob で確認し、
問題なければ全体完了を宣言する。

---

## オーケストレーターの絶対ルール

| ルール | 内容 |
|--------|------|
| **メインは調整専任** | メインセッションで実際の作業ファイルを生成・編集しない |
| **3並列固定** | 1バッチは最大3項目。依存関係がある場合はサブエージェント内で直列実行 |
| **完了待機必須** | 全エージェント完了前に次バッチを起動しない |
| **スキップ判定** | 完了済み項目を再実行しない（冪等性の確保） |
| **DONE確認** | サブエージェントの出力から "DONE: {項目ID}" を確認して完了記録 |
| **エラー分離** | 1項目の失敗で全体を止めず、エラーを記録して継続 |
