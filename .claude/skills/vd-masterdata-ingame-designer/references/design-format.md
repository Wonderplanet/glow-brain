# design.md フォーマット定義

vd-masterdata-ingame-design-creator スキルが Step 2 で生成する設計書（design.md）のフォーマットテンプレート。

## フォーマット指針

- インゲーム要件テキストは**散文形式**で記述すること。以下の3点を無駄のない情報で説明する：
  1. **敵構成・ステータス・出現パターン**: どんな敵がどんなステータスで、どのように出現してくるか
  2. **コマ設計**: 何行構成で各行のコマ数・幅パターンはどうか。コマアセットキーも記載
  3. **対抗キャラと狙い（対抗キャラがある場合）**: どのURキャラの対抗として設計され、どんな遊びを狙っているか
- **インゲーム要件テキストに記載してはいけないこと**:
  - 作品世界観の説明（「〇〇の世界観を反映した」等）— 自明のため不要
  - 不要な前置き（「難易度設計としては」等）— 直接内容を書く
  - 実装上の技術的制約の説明（e_プレフィックス制約、c_キャラ制約等）
  - 「〜を保証しています」「〜を目指しています」等の過剰な表現
- コマ設計は **mermaid block-beta 図**で可視化すること
  - **`columns` は1つだけ宣言する**（複数の `columns` 宣言は行崩れの原因になる）
  - `columns N` の N は全行のコマ幅をスパン整数で表現できる値を選ぶ（通常は 4）
  - 各ブロックのスパン = `N × コマ幅`（例: columns 4 のとき 幅0.25→`:1`、幅0.50→`:2`）
  - 各行のスパン合計が N になることを確認する
- シーケンスは **mermaid flowchart 図**で可視化すること

> **フォーマット参考実例**: `domain/knowledge/masterdata/in-game/guides/raid_jig1_00001_v2.md`
