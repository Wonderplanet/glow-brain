# CSVファイル識別ルール

スプシのローカルコピーフォルダ内のCSVファイルを「設計書CSV（読み込み対象）」と「ノイズCSV（スキップ）」に分類するためのルール。

---

## 分類ルール一覧

### 設計書CSV（必読）

AIが必ず読み込むべきCSV。マスタデータ生成に必要な情報が直接記載されている。

| パターン | 機能 | 読み込み優先度 |
|---------|------|-------------|
| `01_概要.csv` | イベント基本情報（期間・シリーズ・概要） | **最高** |
| `02_施策.csv` | 施策内容（コンテンツ種別・概要） | **最高** |
| `@*.csv` | ガシャ・施策設計書（`@`で始まるもの） | **高** |
| `@_*.csv` | ガシャ・施策設計書（`@_`で始まるもの） | **高** |
| `03_降臨バトル.csv` | 降臨バトル設定 | **高** |
| `04_ミッション.csv` | ミッション設定 | **高** |
| `05_報酬一覧.csv` | ミッション報酬詳細 | **中** |
| `06_ガシャ基本仕様.csv` | ガシャ確率・消費リソース・天井設定 | **中** |
| `07_ショップ_要件書.csv` | ショップ要件 | **中** |
| `07_*設計書.csv` | パック設計書（`07_`で始まり`設計書`で終わるもの） | **中** |
| `08_*.csv` | ショップ・パック追加設計書 | **中** |
| `09_*.csv` | その他設計書 | **中** |

### 設計書CSV（任意）

含まれる場合は読み込む。含まれない場合はその機能がないと判断する。

| パターン | 機能 |
|---------|------|
| `*交換所*.csv` | 交換所設定 |
| `*バナー設計*.csv` | UIバナー設計書（バナー「作成依頼」は除く） |

---

### ノイズCSV（スキップ）

AIが読み込む必要のないCSV。デザイン依頼・テキスト原稿・計算補助・ナビゲーション用。

| パターン | 分類理由 | 例 |
|---------|---------|-----|
| `案_*.csv` | 過去案・ボツ案 | `案_旧設計書.csv` |
| `*案_*.csv` | 中間部分に「案_」を含むもの | `@案_ピックアップ.csv` |
| `バナー作成依頼*.csv` | デザイン依頼（マスタデータと無関係） | `バナー作成依頼.csv` |
| `バナー申請*.csv` | デザイン申請 | `バナー申請書.csv` |
| `クリエイティブ*.csv` | デザイン・アセット管理 | `クリエイティブ一覧.csv` |
| `アセット一覧*.csv` | アセット管理 | `アセット一覧.csv` |
| `告知*.csv` | 告知テキスト・スケジュール | `告知スケジュールNEO.csv` |
| `お知らせ案_*.csv` | お知らせ原稿 | `お知らせ案_202602.csv` |
| `memo用*.csv` | 内部計算・メモ用 | `memo用計算シート.csv` |
| `景品単価*.csv` | 景品価格計算補助 | `景品単価管理.csv` |
| `*目次.csv` | ナビゲーション用 | `スプシ目次.csv` |
| `ロードマップ*.csv` | 施策ロードマップ | `ロードマップ2026.csv` |
| `*スケジュール*.csv` | 告知・運用スケジュール（設計書でないもの） | `リリーススケジュール.csv` |
| `*作業*.csv` | 作業管理・チェックリスト | `作業チェックリスト.csv` |
| `*確認*.csv` | 確認用シート | `QA確認シート.csv` |

## 判定フローチャート

```
CSVファイル名を確認
         |
         ↓
「案_」「バナー作成依頼」「クリエイティブ」「アセット一覧」
「告知」「お知らせ案_」「memo用」「景品単価」「目次」「ロードマップ」
「スケジュール」「作業」「確認」のいずれかで始まる or 含まれる
         |
        YES → ノイズCSV（スキップ）
         |
        NO
         ↓
`@`で始まる or `01_`〜`09_`で始まる
         |
        YES → 設計書CSV（読み込み）
         |
        NO
         ↓
ファイル名に「設計書」「仕様書」「要件書」を含む
         |
        YES → 設計書CSV（読み込み）
         |
        NO
         ↓
内容を確認してデータが含まれるか判断
（ガシャID・期間・報酬など具体的な数値・設定値がある → 設計書）
```

## 実例

### いいジャン祭（施策）の例

```
domain/raw-data/google-drive/spread-sheet/GLOW/080_運営/いいジャン祭（施策）/
  運営_仕様書/20260202_幼稚園WARS いいジャン祭_仕様書/
```

| ファイル | 分類 | 理由 |
|---------|------|------|
| `@幼稚園WARS いいジャン祭ピックアップガシャ_設計書.csv` | 設計書（必読） | `@`で始まる設計書 |
| `01_概要.csv` | 設計書（必読） | 概要シート |
| `02_施策.csv` | 設計書（必読） | 施策内容シート |
| `03_降臨バトル.csv` | 設計書（必読） | 降臨バトル設計 |
| `04_ミッション.csv` | 設計書（必読） | ミッション設計 |
| `05_報酬一覧.csv` | 設計書（任意） | 報酬詳細 |
| `06_ガシャ基本仕様.csv` | 設計書（任意） | ガシャ仕様 |
| `案_@旧ガシャ設計書.csv` | ノイズ（スキップ） | 過去案 |
| `バナー作成依頼.csv` | ノイズ（スキップ） | デザイン依頼 |
| `告知スケジュールNEO.csv` | ノイズ（スキップ） | 告知スケジュール |
| `memo用景品計算.csv` | ノイズ（スキップ） | 計算補助 |

## 判断に迷う場合

ファイル名だけで判断できない場合は：
1. CSVの1行目（ヘッダー）を確認する
2. マスタデータのカラム名らしいヘッダーがある → 設計書
3. 「日付」「担当者」「コメント」などの管理用カラムのみ → ノイズ
4. 判断がつかない場合は、ユーザーに確認する
