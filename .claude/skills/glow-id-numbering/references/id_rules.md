# GLOW ID採番ルール

GLOWゲームプロジェクトにおける各種リソースのID命名規則と採番ルールの詳細。

## 基本原則

- **作品ID**: 3文字の小文字英字（作品のローマ字表記の先頭3文字）
- **汎用リソース**: 作品ID = `glo`
- **番号**: ゼロ埋め（例: `00001`, `00100`）

## カテゴリー別ID規則

### 作品（IP）

| フィールド | 値 |
|-----------|-----|
| フォーマット | `{3文字英字}` |
| 例 | `spy`, `dan`, `aka` |
| 桁数 | 3桁 |

**ルール**: 作品のローマ字表記の先頭3文字（小文字）

### キャラクター

| フィールド | 値 |
|-----------|-----|
| フォーマット | `chara_{作品ID}_{5桁番号}` |
| 例 | `chara_spy_00001`, `chara_spy_00002` |
| 桁数 | 15桁 |

**ルール**:
- プレイヤーキャラ: 接頭語 `chara_`
- 敵キャラ: 接頭語 `enemy_`
- 1キャラにつき99番まで使用可能（例: アーニャ `00001`〜`00099`）
- 別キャラは100番単位で管理（例: ロイド `00100`〜`00199`）
- 同名キャラ（別バージョン）は連番でID取得

**例**:
```
chara_spy_00001  # アーニャ（1体目）
chara_spy_00002  # アーニャ（別バージョン）
chara_spy_00100  # ロイド（1体目）
enemy_spy_00001  # 敵キャラ
```

### アバターアイコン

| フィールド | 値 |
|-----------|-----|
| フォーマット | `unit_icon_chara_{作品ID}_{5桁番号}` |
| 例 | `unit_icon_chara_spy_00001` |
| 桁数 | 25桁 |

**ルール**: キャラIDに `unit_icon_` を前置

### キャラアイコン

| フィールド | 値 |
|-----------|-----|
| フォーマット | `picon_chara_{作品ID}_{5桁番号}` または `eicon_chara_{作品ID}_{5桁番号}` |
| 例 | `picon_chara_spy_00001`, `eicon_chara_spy_00001` |
| 桁数 | 21桁 |

**ルール**:
- プレイヤーキャラ: `picon_` または `eicon_`
- 敵onlyキャラ: `eicon_enemy_{作品ID}_{5桁番号}`

### クエスト

| フィールド | 値 |
|-----------|-----|
| フォーマット | `quest_{カテゴリー}_{難易度}_{作品ID}_{5桁番号}` |
| 例 | `quest_main_normal_dan_00001` |
| 桁数 | 32桁（最大） |

**ルール**:
- カテゴリー: `main`, `event`, `training`など
- 難易度: `normal`, `hard`, `expert`など（設定がない場合は`null`）
- 最大99999番まで割り振り可能

### アイテム

| フィールド | 値 |
|-----------|-----|
| フォーマット | `{接頭語}_{作品ID}_{5桁番号}` |
| 例 | `prism_glo_00001` |
| 桁数 | 19桁（最大） |

**ルール**:
- 接頭語はカテゴリー別に決定（例: `prism`, `ticket`, `material`）
- 汎用アイテムは作品ID = `glo`

### 図鑑

| フィールド | 値 |
|-----------|-----|
| フォーマット | `book_{作品ID}_{5桁番号}` |
| 例 | `book_spy_00001` |
| 桁数 | 14桁 |

### エンブレム

| フィールド | 値 |
|-----------|-----|
| フォーマット | `emblem_{カテゴリー}_{作品ID}_{5桁番号}` |
| 例 | `emblem_event_spy_00001` |
| 桁数 | 24桁（最大） |

**ルール**:
- カテゴリー: `event`, `achievement`, `season`など

### 背景

| フィールド | 値 |
|-----------|-----|
| フォーマット | `background_{作品ID}_{5桁番号}` |
| 例 | `background_spy_00001`, `background_glo_00001`（汎用） |
| 桁数 | 20桁 |

**ルール**: 汎用背景は作品ID = `glo`

### コンテンツ

| フィールド | 値 |
|-----------|-----|
| フォーマット | `contents_{カテゴリー}_{作品ID}_{5桁番号}` |
| 例 | `contents_event_aha_00001` |
| 桁数 | 24桁（最大） |

**ルール**:
- カテゴリー: `event`, `training`, `duel`, `raid`, `limited`

### BGM

| フィールド | 値 |
|-----------|-----|
| フォーマット | `SBG_{画面ID3桁}_{連番3桁}` |
| 例 | `SBG_011_001` |
| 桁数 | 11桁 |

**ルール**:
- 画面ID: 画面遷移図の大分類番号（例: `011` = タイトル、`012` = ホーム）
- 連番: その画面内でのBGM番号

**画面ID一覧**:
- `011`: タイトル
- `012`: ホーム
- `031`: キャラ・編成

### ゲート

| フィールド | 値 |
|-----------|-----|
| フォーマット | `outpost_{作品ID}_{5桁番号}` |
| 例 | `outpost_gom_00001` |
| 桁数 | 17桁 |

**ルール**:
- 実装順に連番
- 敵ゲート: `enemy_{作品ID}_{5桁番号}`から開始

## 検証チェックリスト

IDを検証する際は以下を確認:

- [ ] 接頭語が正しいか
- [ ] 作品IDが3文字の小文字英字か
- [ ] 番号がゼロ埋めされているか
- [ ] 桁数が規定内か
- [ ] アンダースコア `_` が正しく配置されているか

## 注意事項

- 同名キャラ（別バージョン）は別IDを取得する
- 削除予定ファイルは使用しない
- イレギュラーな場合は関係各所に共有してルール追加
