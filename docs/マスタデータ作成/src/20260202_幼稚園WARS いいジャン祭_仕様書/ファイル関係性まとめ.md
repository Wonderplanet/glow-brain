## 主要データの流れ

### 1. **企画立案フェーズ**
   - **企画仕様書_目次.html** で全体構造を把握
   - **01_概要.html** でイベント概要、新キャラ、新アイテムを定義

### 2. **コンテンツ設計フェーズ**
   - **02_施策.html** でクエスト仕様を設計
   - **03_降臨バトル.html** でランキングイベント仕様を設計
   - **04_ミッション.html** でミッション仕様を設計

### 3. **報酬バランス調整フェーズ**
   - **05_報酬一覧.html** で全施策の報酬合計を計算
   - 過去イベント（2.5次元の誘惑、SPY×FAMILY、怪獣８号）との比較
   - 配布上限チェック（PDM承認が必要な場合を判定）

### 4. **マネタイゼーション設計フェーズ**
   - **06_ガシャ基本仕様.html** でガシャの期待値を計算
   - **幼稚園WARS いいジャン祭ピックアップガシャ_設計書.html** で具体的なガシャを設計
   - **07_いいジャン祭パック_設計書.html** で課金パックを設計

### 5. **制作依頼フェーズ**
   - **03_制作依頼一覧.html** で制作物をリストアップ（バナー、アセット、文言など）
   - **04_監修に依頼したいもの.html** で監修項目をリストアップ
   - **アセット一覧.html** / **バナー一覧.html** / **背景一覧.html** で具体的な素材を管理

### 6. **告知・スケジュール管理フェーズ**
   - **告知スケジュールNEO.html** で告知タイミングを管理
   - **00_ロードマップ転記用.html** でロードマップへ転記

## 注意事項

### 命名規則
- **番号プレフィックス**: `00_`、`01_`、`02_` などで優先順位と読む順番を明示
- **カテゴリ分類**: `【デイリー】`、`【ストーリー】` などで施策種別を明示
- **ステータス表記**: `(作成中)` で開発中のファイルを明示
- **用途表記**: `のテンプレ`、`のコピー` でファイルの役割を明示
- **未使用表記**: `使用しない_` で不要ファイルを明示

### ファイル管理のベストプラクティス
1. **目次ファイルから開始**: `企画仕様書_目次.html` から全体構造を把握
2. **番号順に参照**: 00番から順に読むことで段階的に理解できる設計
3. **テンプレートの活用**: 新規イベント作成時はテンプレートファイルをコピーして使用
4. **バージョン管理**: 「のコピー」や「REBORN」などで複数バージョンを管理
5. **作成中の明示**: (作成中)表記で未完成ファイルを明確化

### データフロー
1. 企画書で全体像を定義 → 各施策の詳細設計 → 報酬バランス調整 → 制作依頼 → 告知スケジュール作成
2. 報酬一覧で他イベントとの比較を行い、配布バランスを調整
3. ガシャ基本仕様で期待値を計算し、課金設計の妥当性を検証

### リソースフォルダ
- `resources/sheet.css` はすべてのHTMLファイルから参照される共通スタイルシート
- HTMLファイルの見た目を統一的に管理

## 使用例

### 新規イベント企画時
1. `01_概要 のテンプレ .html` をコピーして新規イベント概要を作成
2. `企画仕様書_目次.html` を更新して新規ファイルへのリンクを追加
3. 番号順に各カテゴリの仕様書を作成

### 報酬バランス確認時
1. `05_報酬一覧.html` を開く
2. 過去イベントとの比較表を参照
3. 1ヶ月の配布上限と照らし合わせて調整

### 制作進行管理時
1. `03_制作依頼一覧.html` で制作物の進捗を確認
2. `04_監修に依頼したいもの.html` で監修スケジュールを確認
3. アセット一覧ファイルで具体的な素材を管理

### ガシャ設計時
1. `06_ガシャ目次.html` で関連ファイルを把握
2. `06_ガシャ基本仕様.html` で期待値を計算
3. 具体的なピックアップガシャ設計書で詳細を詰める

## まとめ

この仕様書群は、イベント企画から実装、告知まで一貫して管理できる構造になっています。番号プレフィックスによる順序付けと、カテゴリごとの分類により、初見でも理解しやすい設計です。テンプレートファイルの存在により、新規イベント作成時の効率化も図られています。

主要なワークフローは「企画 → 設計 → 報酬調整 → 制作依頼 → 告知」という流れで、各フェーズに対応するファイルが整備されています。
