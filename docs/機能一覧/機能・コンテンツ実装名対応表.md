このドキュメントは、GLOWゲームの各機能・コンテンツについて、アプリ上での表示名、glow-clientでの実装名、glow-serverでの実装名の対応関係をまとめたものです。

## 補足情報

### glow-clientの実装構造

glow-clientはUnity/C#で実装されており、以下のようなディレクトリ構造を持ちます：

```
Assets/GLOW/Scripts/Runtime/Scenes/
├── InGame/          # バトル画面
├── QuestSelect/     # クエスト選択
├── AdventBattle/    # 降臨バトル
├── Pvp/             # PvP
├── Gacha/           # ガチャ
├── Unit/            # ユニット管理
├── Mission/         # ミッション
├── Encyclopedia/    # 図鑑
├── Home/            # ホーム画面
├── Shop/            # ショップ
└── ...
```

各Sceneディレクトリには、以下のレイヤーが含まれます：
- **Presentation**: View、ViewController、ViewModel
- **Domain**: ユースケース、モデル、ファクトリ
- **Application**: インストーラー、デリゲート
- **Data**: リポジトリ、データストア

### glow-serverの実装構造

glow-serverはLaravel/PHPで実装されており、以下のような構造を持ちます：

```

├── StageController.php           # クエスト・ステージ全般
```

### マスターデータの管理

マスターデータは、`projects/glow-masterdata/`ディレクトリで管理されており、約165個のCSVファイルが存在します。各テーブルは`Mst`プレフィックスを持ちます（例：`MstQuest`, `MstUnit`, `MstGacha`）。
