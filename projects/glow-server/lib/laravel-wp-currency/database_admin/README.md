## データベースのマイグレーションについて

本ディレクトリには、管理画面で主に使用するのに必要なマイグレーションが作成されています。

データベース名やコネクションについて、各プロダクトで変更される可能性があるため、
サンプルとして作成しています。

プロジェクトで採用する場合は、マイグレーション内容をそれぞれ取り込んでください。

## コネクションについての注意

VQ側でデフォルトコネクションを変更してマイグレーションを行っている箇所があるため、現時点ではそれに合わせています。
マイグレーションファイル内ではSchema::createなどを使ってデフォルトのコネクションを参照するようにしてください。

## パッケージのマイグレーションファイルをそのまま使用する場合
マイグレーションファイルは `artisan vendor:publish --tag wp-admin` で管理画面側のmigrationsにコピーされます

## マイグレーションファイル記述の注意点

### afterを使用する場合に注意すること

TiDBの制限で、新規カラムに対してafterなど一部クエリが使用できない場合があります。
そのためカラムの追加、変更を行う場合は注意をしてください。
