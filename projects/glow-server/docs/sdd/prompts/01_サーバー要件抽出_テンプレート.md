あなたはバックエンドエンジニアの API 開発スペシャリストです。
これから渡す PDF（ゲーム体験仕様書）から、「サーバー側で考慮すべき要件（サーバー関与が必要となる項目）」のみを抽出してください。

※注意：実装方式（API が必要 / DB 更新が必要 等）の断定はしない。

@docs/sdd/features/{FEATURE_NAME}/ゲーム体験仕様書.pdf

以下のルールに従って出力してください：

## 【重要】出力ファイル名について

**必ず以下のファイル名で保存してください（番号プレフィックス "01_" を含めること）：**

```
docs/sdd/features/{FEATURE_NAME}/01_サーバー要件抽出.md
```

ファイル名は **`01_サーバー要件抽出.md`** です。番号を省略しないでください。

=================
【抽出ルール】
1. PDF の「どのページ」「どの箇所」に書かれていたかを明記する
   - 例: Page 12「ゲーム内通貨の獲得フロー」内、右上の箇条書き

2. サーバー要件に該当するテキストを **仕様書からそのまま抜粋する**

3. 抜粋したテキストに対して、
   バックエンド/API の観点から
   「サーバー側で考慮が必要となるポイント」を簡潔に1行で補足する
   - ※実装方針の断定（「API が必要」「DB 更新が必須」等）は禁止
   - ※あくまで「サーバーが関与しそうな領域」「データ管理が必要になる点」を中立的に述べる

4. 出力形式は以下に統一する：

=================
【出力フォーマット】

## サーバー要件一覧

### 要件X
- **元ページ:** Page X, 「〜〜」のセクション
- **抜粋:**
  > （仕様書に書かれていた原文そのまま）
- **サーバー観点での補足:**
  - （データ管理 / 状態管理 / 検証 / 記録など、サーバー側で考慮すべき点を記載）

### 要件Y
（同様に続けて列挙）

=================

【注意】
- サーバー非依存のUI仕様は除外
- アニメーション、画面遷移、演出内容は除外
- ゲーム内ロジックの記述でも、サーバーが関与しないものは除外
- 実装方針の断定はしないこと
- 「サーバーの関与する内容」だけ抽出すること

=================

PDF の解析が完了したら、上記フォーマットで全て列挙してください。
