========================
【目的】

- サーバー仕様レビューで洗い出された不明点・曖昧点に対するゲーム企画担当者の回答を整理する
- 各不明点がどのように解決されたかを明確に記録する
- 不明瞭だった部分が完全に補えたか、まだ不明な点が残っているかを評価する
- サーバー実装に必要な仕様が全て確定したかを判断する

※ 注意: 「サーバー仕様レビュー.md」自体は変更しません。新しいドキュメントとして出力します。

========================
【作業手順】

1. **サーバー仕様レビューの確認**
   - `@docs/sdd/features/{FEATURE_NAME}/03_サーバー仕様レビュー.md` を読み込む
   - 「3. 不明点・曖昧点・判断が必要な項目」セクションを特定する
   - 「4. プランナーへ確認が必要な項目まとめ」セクションを特定する

2. **ユーザーへの確認結果提供依頼**
   - ユーザーに対して、ゲーム体験仕様の確認結果を提供するよう依頼する
   - 提供してほしい内容を明確に伝える：
     - どの不明点について確認したか
     - ゲーム企画担当者からの回答内容
     - 確認日時や確認方法（分かれば）

3. **ゲーム体験仕様確認結果の受け取りと整理**
   - ユーザーから提供された確認結果を読み取る
   - サーバー仕様レビューの各不明点と照らし合わせる
   - 各質問項目に対する回答を抽出する
   - 回答が不明確な場合や追加確認が必要な場合は明記する

4. **解決状況の評価**
   - 各不明点が「完全に解決」「部分的に解決」「未解決」のいずれかを判定する
   - 解決によって明確になった仕様を整理する
   - まだ不明な点が残っている場合は、何が足りないかを明記する
   - **重要**: サーバー仕様レビューで挙げられた全ての不明点について、確認結果で言及されているかをチェックする

5. **出力ドキュメントの作成**
   - 以下のフォーマットで Markdown を生成する
