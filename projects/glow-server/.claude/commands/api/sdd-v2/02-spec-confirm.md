# SDD v2 仕様確認

要件調査で洗い出された不明点・曖昧点をレビューし、プランナーへの確認項目を整理します。
プランナー確認後は、確認結果を統合して確定仕様を整理します。

## 使用方法

```
/api:sdd-v2:02-spec-confirm {機能名}
```

例: `/api:sdd-v2:02-spec-confirm 交換所`

## 実行内容

引数: $ARGUMENTS

専門エージェント「**sdd-v2-spec-confirmation**」に委譲して実行します。

エージェントは `api-sdd-v2-spec-confirmation` スキルを使用して作業を実施します。

## 前提条件

- `docs/sdd-v2/features/{機能名}/01_要件調査.md` が存在すること

## 2段階のフロー

### Phase A: 確認項目の洗い出し

1. このコマンドを実行
2. `02_仕様確認.md` が生成される
3. プランナーへ確認項目を連絡

### Phase B: 確認結果の統合

1. プランナーから回答を受け取る
2. 再度このコマンドを実行し、確認結果を提供
3. `02_2_プランナー確認結果.md` が生成される

## 次のステップ

プランナー確認結果を統合後、以下のコマンドでAPI設計書を作成:

```
/api:sdd-v2:03-api-design {機能名}
```
