# SDD v2 API設計書作成

仕様確認結果を基に、実装可能な具体的なAPI設計書を作成します。
API概要、詳細設計、DB設計、実装設計を統合した完全な設計書を出力します。

## 使用方法

```
/api:sdd-v2:03-api-design {機能名}
```

例: `/api:sdd-v2:03-api-design 交換所`

## 実行内容

引数: $ARGUMENTS

専門エージェント「**sdd-v2-api-design**」に委譲して実行します。

エージェントは `api-sdd-v2-api-design` スキルを使用して作業を実施します。

## 前提条件

- `docs/sdd-v2/features/{機能名}/01_要件調査.md` が存在すること
- `docs/sdd-v2/features/{機能名}/02_仕様確認.md` が存在すること
- `docs/sdd-v2/features/{機能名}/02_2_プランナー確認結果.md` が存在すること

## 次のステップ

出力された `03_API設計書.md` を確認後、以下のコマンドで品質レビューを実行:

```
/api:sdd-v2:03-2-api-review {機能名}
```

## 完了後のアクション

API設計書レビュー完了後、以下の実装作業に進めます:

1. マイグレーション作成（DB設計に基づく）
2. ドメイン層実装（実装ガイドに基づく）
3. API層実装（詳細仕様に基づく）
4. テスト実装（テスト観点に基づく）
