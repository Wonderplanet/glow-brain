# SDD ゲーム体験仕様確認結果まとめ

人間が別途行ったゲーム体験仕様の確認結果を、サーバー仕様レビューと照らし合わせて、
不明点が全て解決されたかを評価し、整理したドキュメントを作成します。

## 使用方法

```
/sdd:confirm-game-experience-spec {機能名}
```

例: `/sdd:confirm-game-experience-spec スタミナブースト`

**重要**: このコマンドは機能名のみを引数に取ります。
ゲーム体験仕様の確認結果は、コマンド実行後にAIが依頼するので、その時に提供してください。

## 実行内容

引数: $ARGUMENTS

### 処理ステップ

1. **機能名の取得**
   - 第1引数から機能名を取得

2. **テンプレートファイルの読み込み**
   - `docs/sdd/prompts/04_ゲーム体験仕様確認結果まとめ_テンプレート.md` を読み込む

3. **テンプレートの置換**
   - テンプレート内の全ての `{FEATURE_NAME}` を第1引数の機能名に置換

4. **プロンプトの実行**
   - 置換後の内容をプロンプトとして実行
   - ユーザーからゲーム体験仕様の確認結果を受け取り、テンプレートファイルに記載されたルールと手順に従って確認結果を整理

## 前提条件

- `docs/sdd/features/{機能名}/03_サーバー仕様レビュー.md` が存在すること（先に `/sdd:review-server-spec` コマンドを実行済みであること）
- ゲーム企画担当者への確認が完了していること（確認結果のメモ、チャットログ、回答テキストなど）

## 出力

- `docs/sdd/features/{機能名}/04_ゲーム体験仕様確認結果まとめ.md` にゲーム体験仕様の確認結果整理レポートが出力されます
- 詳細なフォーマットはテンプレートファイルで定義されています
