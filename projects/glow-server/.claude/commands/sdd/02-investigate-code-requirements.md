# SDD コード調査によるサーバー要件追記

既存の「サーバー要件抽出.md」を起点に、コードベースを調査して仕様書に書かれていない追加のサーバー要件を洗い出します。

## 使用方法

```
/sdd:investigate-code-requirements {機能名}
```

例: `/sdd:investigate-code-requirements スタミナブースト`

## 実行内容

引数: $ARGUMENTS

### 処理ステップ

1. **機能名の取得**
   - 第1引数から機能名を取得

2. **テンプレートファイルの読み込み**
   - `docs/sdd/prompts/02_サーバー要件_コード調査追記テンプレート.md` を読み込む

3. **テンプレートの置換**
   - テンプレート内の全ての `{FEATURE_NAME}` を第1引数の機能名に置換

4. **プロンプトの実行**
   - 置換後の内容をプロンプトとして実行
   - テンプレートファイルに記載されたルールと手順に従ってコード調査を実施し、追加のサーバー要件を洗い出す

## 前提条件

- `docs/sdd/features/{機能名}/01_サーバー要件抽出.md` が存在すること（先に `/sdd:extract-server-requirements` コマンドを実行済みであること）
- 対象機能に関連するコードがリポジトリ内に存在すること

## 出力

- `docs/sdd/features/{機能名}/02_サーバー要件_コード調査追記.md` にコード調査で判明した追加のサーバー要件が出力されます
- 詳細な調査方法とフォーマットはテンプレートファイルで定義されています
