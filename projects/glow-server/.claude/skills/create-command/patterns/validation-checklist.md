# 検証チェックリスト

コマンド作成後に確認すべき項目です。

## 構文チェック

- [ ] YAMLフロントマターが正しく閉じられている（`---`で開始・終了）
- [ ] フィールド名のスペルが正しい（description, allowed-tools, model, argument-hint）
- [ ] インデントが正しい（YAML標準に従う）

## description

- [ ] 何ができるか明記されている
- [ ] いつ使うか示されている
- [ ] 具体的なキーワードを含んでいる
- [ ] 長すぎない（1-2文程度）

## allowed-tools

- [ ] 必要最小限のツールのみ許可している
- [ ] 構文が正しい（カンマ区切り）
- [ ] ワイルドカードの使用が適切（`Bash(git:*)`など）
- [ ] 省略した場合、会話設定から継承される

## argument-hint

- [ ] 引数が必要な場合、ヒントが設定されている
- [ ] ヒントが分かりやすい（[必須] や オプション|形式）
- [ ] $ARGUMENTS や $1 と一致している

## コマンド本文

- [ ] 目的が明確に記述されている
- [ ] 変数の使用が適切（$ARGUMENTS vs $1）
- [ ] Bash実行（!`cmd`）の出力量が適切
- [ ] ファイル参照（@path）のパスが正しい

## 配置

- [ ] 正しいディレクトリに配置されている
- [ ] ファイル名がkebab-caseになっている
- [ ] 既存コマンドと名前が競合していない

## 動作確認

- [ ] `/help`でコマンドが表示される
- [ ] 自動補完でdescriptionが表示される
- [ ] 引数ありの場合、ヒントが表示される
- [ ] 実際に実行して期待通り動作する

## セキュリティ

- [ ] 破壊的な操作には適切な制限がある
- [ ] 機密情報を含むファイル/コマンドを参照していない
- [ ] 必要に応じて`disable-model-invocation: true`を設定

## ドキュメント

- [ ] 複雑なコマンドには使用例を含める
- [ ] 前提条件があれば明記する
- [ ] 関連するスキルやコマンドへの参照を含める
