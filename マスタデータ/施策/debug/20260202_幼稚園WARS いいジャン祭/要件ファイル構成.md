## 主要データの流れ

### 1. **企画フェーズ**
`企画仕様書_目次.html` → `01_概要.html`
- 施策の目的、期間、主要コンテンツを定義
- 新キャラクター（リタ、ルーク、ダグ、ハナ）のID割り当て
- 新アイテム（かけら、メモリー、エンブレム、原画）の定義

### 2. **施策設計フェーズ**
`01_概要.html` → `02_施策.html` / `03_降臨バトル.html`
- イベントクエストの実装仕様
- クエスト種類: デイリー、ストーリー、チャレンジ、高難易度、降臨バトル

### 3. **報酬設計フェーズ**
`02_施策.html` + `04_ミッション.html` → `05_報酬一覧.html`
- ミッション報酬の集計
- イベント全体で獲得可能なアイテム総数の算出

### 4. **収益施策フェーズ**
`01_概要.html` → `06_ガシャ_設計書` / `07_ショップ_設計書`
- ガシャ: リタ（UR）、ルーク（SSR）のピックアップ
- パック: いいジャン祭パックの再販

### 5. **クリエイティブ制作フェーズ**
`01_概要.html` → `クリエイティブ一覧.html` / `バナー一覧.html`
- キャラクタービジュアル、UIアセット、バナーの制作依頼

## 命名規則

### ファイル命名規則

1. **番号付き正式版**: `00_`, `01_`, `02_`, `03_`, `04_`, `05_`, `06_`, `07_` で始まる
2. **テンプレート**: ファイル名に「テンプレ」が含まれる
3. **バックアップ**: 「のコピー」が含まれる
4. **廃棄予定**: 「使用しない_」で始まる
5. **作成中**: 「(作成中)」が含まれる

### アセットID命名規則

- **キャラクター**: `chara_you_xxxxx` (プレイアブル), `enemy_you_xxxxx` (エネミー)
- **かけら**: `piece_you_xxxxx`
- **メモリー**: `memory_chara_you_xxxxx`
- **エンブレム**: `emblem_event_you_xxxxx`, `emblem_adventbattle_you_seasonxx_xxxxx`
- **原画**: `artwork_event_you_xxxx`

## 注意事項

### ファイル使用上の注意

1. **目次ファイルの重要性**: `企画仕様書_目次.html`が全体のナビゲーションハブとなっているため、新規ファイル追加時は目次も更新する
2. **テンプレートファイルは編集禁止**: 「テンプレ」ファイルは複製して使用する
3. **バージョン管理**: 「のコピー」「_REBORN」などのファイル名で改訂版を管理
4. **廃棄予定ファイル**: 「使用しない_」プレフィックスのファイルは削除前の一時保管

### ID管理の注意

- 全てのキャラクターIDとアイテムIDは外部の「GLOW_ID 管理」スプレッドシートで一元管理
- 重複割り当てを避けるため、新規ID追加前に必ず管理シートを確認

### 施策スケジュールの注意

- イベント期間: 2026/02/02 15:00 〜 2026/03/02 10:59
- 降臨バトル: 2026/02/09 15:00 〜 2026/02/15 14:59（イベント期間中の限定開催）
- ログインボーナス: 2026/02/02 15:00 〜 2026/02/16 3:59
