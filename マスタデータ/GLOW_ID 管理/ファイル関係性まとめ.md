# GLOW_ID 管理 ファイル構造

## 概要

このディレクトリには、GLOWゲームプロジェクトにおける各種リソースやエンティティのID管理に関する仕様書・管理表が格納されています。全てHTMLファイル形式で、Googleスプレッドシートからエクスポートされたものと思われます。

**目的**: ゲーム内の各種要素（作品、キャラ、アイテム、クエスト、BGM、バナーなど）に対して一意のIDを割り当てるためのルールと管理表を提供

**管理対象**: 約40種類のリソースカテゴリー

## ファイル構造

### 1. コアルールファイル（必読）

- **ID割り振りルール.html** - ID命名規則の基本ルールを定義（最重要ドキュメント）
  - 各カテゴリーごとのID付与ルール
  - 最大ID桁数の定義
  - 命名規則（接頭語、作品ID、番号体系）
  
- **プルダウンメニュー (SE用).html** - SE（効果音）分類用のプルダウンメニュー定義
  - SEステータス（未発注、見積り依頼中、発注済など）
  - 使用画面ID（011: タイトル、012: ホーム、031: キャラ・編成など）
  - BGM発注ステータス管理

### 2. 作品・IPコンテンツ管理

- **作品.html** - 作品（IP）のマスター管理表
  - 作品ID（3文字の英字コード: spy, aka, dan など）
  - 作品バナーID、ロゴID
  - 連載曜日（月曜日〜日曜日）
  - 表示順序
  
### 3. キャラクター関連

- **キャラ.html** - キャラクターID管理
  - プレイヤーキャラ: `chara_{作品ID}_{5桁番号}`
  - 敵キャラ: `enemy_{作品ID}_{5桁番号}`
  
- **アバターアイコン.html** - アバター用アイコンID
  - 形式: `unit_icon_chara_{作品ID}_{5桁番号}`
  
- **キャラpng素材.html** - キャラクター画像素材管理
  
- **キャラアイコン（削除予定ファイル参照）** - 古い仕様のアイコン

### 4. クエスト・ステージ関連

- **クエスト.html** - クエストID管理
  - 形式: `quest_{カテゴリー}_{難易度}_{作品ID}_{5桁番号}`
  - 例: `quest_main_normal_dan_00001`
  
- **クエストカテゴリー.html** - クエスト種別の定義
  
- **クエストサムネ.html** - クエストサムネイル画像ID
  
- **イベント用ステージサムネ.html** - イベント専用ステージ画像

### 5. アイテム・リソース管理

- **アイテム.html** - アイテムID管理
  - 形式: `{接頭語}_{作品ID}_{5桁番号}`
  - 接頭語例: `prism` (プリズム)
  
- **原画&セリフ演出.html** - 原画演出リソース
  
- **原画のかけら 原画.html** - 原画かけらアイテム

### 6. バナー・UI素材

- **ガシャバナー.html** - ガシャ（ガチャ）バナーID
  
- **キャンペーンバナー.html** - キャンペーン告知バナー
  
- **パックバナー.html** - パック販売バナー
  
- **パスバナー.html** - パス（シーズンパス等）バナー
  
- **ショップバナー（交換所バナー.html）** - ショップ・交換所表示用
  
- **お知らせバナー.html** - お知らせ画面用バナー
  
- **IGNバナー.html** - IGN（インゲーム？）関連バナー
  
- **ランクマッチバナー.html** - ランクマッチモード用

### 7. ゲームシステム要素

- **ゲート.html** - ゲートシステムID
  - 形式: `outpost_{作品ID}_{5桁番号}`
  
- **コンテンツ.html** - コンテンツID管理
  - 形式: `contents_{カテゴリー}_{作品ID}_{5桁番号}`
  - カテゴリー: event, training, duel, raid, limited
  
- **図鑑.html** - 図鑑エントリーID
  - 形式: `book_{作品ID}_{5桁番号}`

### 8. アイコン・UI部品

- **エンブレム.html** - エンブレムID
  - 形式: `emblem_{カテゴリー}_{作品ID}_{5桁番号}`
  
- **パスアイコン.html** - パス用アイコン
  
- **特性アイコン.html** - キャラ特性表示アイコン
  
- **状態変化アイコン.html** - バトル中の状態異常等
  
- **敵情報アイコン.html** - 敵キャラ情報表示用

### 9. サウンド・BGM

- **BGM.html** - BGM管理
  - 形式: `SBG_{画面分類3桁}_{連番3桁}`
  - 例: `SBG_011_001` (タイトル画面)
  
- **02_SEファイル.html** - SE（効果音）ファイル管理

### 10. その他

- **背景.html** - 背景画像ID
  - 形式: `background_{作品ID}_{5桁番号}`
  - 汎用背景は作品ID = `glo`
  
- **ショップ.html** - ショップ関連ID
  
- **ログインボーナス.html** - ログインボーナス管理
  
- **いいジャン祭ID.html** - 「いいジャン祭」イベント専用ID

### 11. 削除予定ファイル（使用禁止）

- **削除予定_敵情報アイコン（ノーマル）.html**
- **削除予定_敵情報アイコン（ボス(強敵)）.html**
- **削除予定　→.html**

⚠️ これらのファイルは旧仕様のため使用しないこと

## 主要データの流れ

### 1. 新規作品追加時のID割り当てフロー

```
1. 作品.html に新規作品登録
   ↓ 作品ID（3文字）を決定（例: spy, dan, aka）
   
2. 作品IDを使用してリソースID生成
   ↓
   ├── キャラ.html: chara_spy_00001, chara_spy_00002...
   ├── クエスト.html: quest_main_normal_spy_00001...
   ├── アイテム.html: prism_spy_00001...
   └── 図鑑.html: book_spy_00001...
```

### 2. BGM・SE管理フロー

```
1. プルダウンメニュー (SE用).html で画面IDを確認
   ↓ 例: タイトル画面 = 011, ホーム = 012
   
2. BGM.html で該当画面のBGM ID割り当て
   ↓ SBG_011_001 (タイトル画面BGM 1曲目)
   
3. 02_SEファイル.html でSE管理
```

### 3. バナーリソース管理フロー

```
イベント・キャンペーン企画
├── ガシャバナー.html ← ガシャ開催用
├── キャンペーンバナー.html ← キャンペーン告知
├── お知らせバナー.html ← お知らせ画面表示
└── 各種専用バナー（パス、パック、ランクマッチ等）
```

## ID命名規則の詳細

### 基本パターン

| カテゴリー | 接頭語 | 作品ID | 番号桁数 | 形式例 |
|-----------|--------|--------|---------|--------|
| 作品 | なし | 3桁 | - | `spy`, `dan`, `aka` |
| キャラ | `chara_` | 3桁 | 5桁 | `chara_spy_00001` |
| 敵キャラ | `enemy_` | 3桁 | 5桁 | `enemy_spy_00001` |
| クエスト | `quest_` | 3桁 | 5桁 | `quest_main_normal_dan_00001` |
| アイテム | `{カテゴリー}_` | 3桁 | 5桁 | `prism_glo_00001` |
| ゲート | `outpost_` | 3桁 | 5桁 | `outpost_gom_00001` |
| 図鑑 | `book_` | 3桁 | 5桁 | `book_spy_00001` |
| エンブレム | `emblem_` | 3桁 | 5桁 | `emblem_event_spy_00001` |
| 背景 | `background_` | 3桁 | 5桁 | `background_spy_00001` |
| コンテンツ | `contents_` | 3桁 | 5桁 | `contents_event_aha_00001` |
| BGM | `SBG_` | 画面3桁 | 3桁 | `SBG_011_001` |

### 汎用リソースの作品ID

汎用的なリソース（全作品共通）には作品ID `glo` を使用：
- `prism_glo_00001` - GLOWゲーム全体の共通プリズム
- `background_glo_00001` - 汎用背景

### 番号割り当てルール

- **キャラクター**: 1IP（作品）につき1キャラで99番まで使用可能
  - 例: アーニャ `chara_spy_00001`, `chara_spy_00002` (別バージョン)
  - ロイド `chara_spy_00100`, `chara_spy_00101`
  
- **最大連番**: ほとんどのカテゴリーで99999番まで割り振り可能

## 使用時の注意事項

### ✅ ベストプラクティス

1. **新規ID割り当て前に必ず `ID割り振りルール.html` を確認**
2. **作品IDは `作品.html` で既存IDと重複しないか確認**
3. **削除予定ファイルは使用しない**
4. **同名キャラで別IDが必要な場合は連番で管理**（例: 極楽蝶（遠）、極楽蝶（近））
5. **BGM/SEは画面IDと紐付けて管理**（`プルダウンメニュー (SE用).html` 参照）

### ⚠️ よくある注意点

- **作品IDは必ず3文字の小文字英字**（例: ○ `spy` ✗ `SPY`, ✗ `spy1`）
- **IDの桁数を厳守**（5桁の場合は `00001` のように0埋め）
- **接頭語のアンダースコア `_` を忘れない**（`chara_spy` ○, `charaspy` ✗）
- **同時期・同名キャラは別IDを取得する**
- **force pushされたリモートの変更には注意**（別プロジェクトだが参考情報）

### 🔧 トラブルシューティング

**Q: 新しい作品のIDをどう決める？**
A: `作品.html` を参照し、作品名のローマ字表記の先頭3文字を使用（小文字）

**Q: BGMのID体系がわからない**
A: `BGM.html` と `プルダウンメニュー (SE用).html` を参照。画面分類ID（3桁）+ 連番（3桁）

**Q: 削除予定ファイルを見つけた**
A: 使用禁止。該当機能は `敵情報アイコン.html` など統合版を使用

**Q: ID桁数を間違えた**
A: マスターデータ全体に影響するため、設計段階で `ID割り振りルール.html` を必ず確認
