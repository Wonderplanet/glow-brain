# 運営仕様書とマスタデータのマッピング解析レポート

**リリース**: 202512020
**イベント**: 正月特別号！【推しの子】 いいジャン祭＋メインクエスト
**作成日**: 2026-01-16

---

## 概要

このレポートは、運営仕様書CSVファイルで定義された各施策が、実際にどのマスタデータCSVファイルに投入されたかを網羅的にまとめたものです。このマッピング情報を元に、各運営仕様書から対応するマスタデータを作成するための手順書を作成することができます。

---

## 7. まとめ：運営仕様書→マスタデータの変換フロー

各運営仕様書CSVからマスタデータCSVを作成する際の基本的なフローは以下のとおりです：

### 7.1 クエスト系施策（収集、強化、チャレンジ、高難易度）

1. **MstQuest.csv** にクエスト定義を作成
   - quest_type, mst_event_id, asset_key, start_date, end_date等を設定

2. **MstStage.csv** に各ステージを作成
   - mst_quest_id, stage_number, recommended_level, cost_stamina, exp等を設定
   - prev_mst_stage_idで前ステージを参照（順次開放）
   - auto_lap_type, max_auto_lap_countでスタミナブースト設定

3. **MstStageReward.csv** に報酬を作成
   - mst_stage_id, reward_category（FirstClear/Clear/Random）, resource_type, resource_id, resource_amount, percentageを設定

4. **MstStageClearTimeReward.csv** にクリアタイム報酬を作成（該当する場合）
   - mst_stage_id, upper_clear_time_ms, resource_type, resource_amountを設定

### 7.2 ミッション系施策

1. **MstMissionEvent.csv** にミッション定義を作成
   - mst_event_id, criterion_type, criterion_value, criterion_count等を設定

2. **MstMissionReward.csv** に報酬を作成
   - group_id, resource_type, resource_id, resource_amountを設定

3. **特定種類のミッション**は専用テーブルへ
   - ログインボーナス → MstMissionEventDailyBonus.csv, MstMissionEventDailyBonusSchedule.csv
   - 期間限定 → MstMissionLimitedTerm.csv
   - 恒常 → MstMissionAchievement.csv

### 7.3 降臨バトル

1. **MstAdventBattle.csv** に降臨バトル定義を作成
   - mst_event_id, challengeable_count, ad_challengeable_count, exp, coin等を設定

2. **MstAdventBattleRank.csv** にランク定義を作成
   - rank_type, rank_level, required_lower_scoreを設定

3. **MstAdventBattleRewardGroup.csv** に報酬グループを作成
   - reward_category（MaxScore/RaidTotalScore/Rank/Ranking）, condition_valueを設定

4. **MstAdventBattleReward.csv** に具体的な報酬を作成
   - mst_advent_battle_reward_group_id, resource_type, resource_id, resource_amountを設定

5. **MstAdventBattleClearReward.csv** にクリア時ランダム報酬を作成
   - reward_category（Random）, resource_type, resource_id, resource_amount, percentageを設定

### 7.4 交換所

1. **MstExchange.csv** に交換所定義を作成
   - mst_event_id, exchange_trade_type, start_at, end_atを設定

2. **MstExchangeLineup.csv** に商品ラインナップを作成
   - group_id, tradable_count, display_orderを設定

3. **MstExchangeReward.csv** に交換報酬を作成
   - mst_exchange_lineup_id, resource_type, resource_id, resource_amountを設定

4. **MstExchangeCost.csv** に交換コストを作成
   - mst_exchange_lineup_id, cost_type, cost_id, cost_amountを設定

## 8. 今後の手順書作成に向けた提言

このマッピング解析レポートを元に、以下のような手順書を作成することを推奨します：

### 8.1 各施策タイプ別の詳細手順書

- **収集クエスト作成手順書**: ステージ構成、ドロップ設計、原画のかけら設定等
- **強化クエスト作成手順書**: メモリーフラグメントの配分、難易度調整等
- **チャレンジ/高難易度クエスト作成手順書**: クリアタイム報酬、特別ルール設定等
- **降臨バトル作成手順書**: ランク設定、報酬バランス、ランキング報酬等
- **ミッション作成手順書**: 条件設定、報酬設計等
- **交換所作成手順書**: 商品設定、レート調整等

### 8.2 チェックリスト

各手順書には、以下のようなチェックリストを含めることを推奨します：

- [ ] 必須カラムがすべて埋まっているか
- [ ] 期間設定が仕様書と一致しているか
- [ ] 報酬IDが正しく参照されているか
- [ ] 前提クエストの依存関係が正しいか
- [ ] スタミナ、EXP等の数値が仕様書と一致しているか

### 8.3 自動化の検討

このマッピング情報を活用し、運営仕様書CSVからマスタデータCSVを自動生成するスクリプトの作成も検討できます。
