# MstSpecialAttackI18n 詳細説明

> CSVパス: `projects/glow-masterdata/MstSpecialAttackI18n.csv`

---

## 概要

`MstSpecialAttackI18n` は**ユニット（キャラ）の必殺技（スペシャルアタック）の名前を多言語設定するi18n専用テーブル**。

メインテーブルとなる `mst_special_attacks` は存在せず、必殺技の名前は `mst_units`（ユニット）に紐付いて管理される。1ユニットにつき1言語1レコードが対応する。

### ゲームへの影響

- **スペシャルアタック演出**: 必殺技発動時のUI（技名テキスト）に表示される名前がこのテーブルから取得される。
- **キャラ詳細画面**: 各キャラの必殺技名として表示される。
- 名前は各キャラの個性・作品観を反映したものになっている（例: `決して屈しない！`、`"拷問"の時間です`）。

### テーブル連携図

```
（ペアとなるmst_special_attacksテーブルは存在しない）
MstUnit（ユニット）
  └─ id → MstSpecialAttackI18n.mst_unit_id（必殺技名）
```

## 全カラム一覧

| カラム名 | 型 | NULL許容 | デフォルト値 | 説明 |
|---------|----|---------|-----------|----|
| `ENABLE` | varchar | - | - | CSVの有効フラグ。`e` = 有効 |
| `id` | varchar(255) | 不可 | - | 主キー（連番整数） |
| `release_key` | bigint | 不可 | 1 | リリースキー |
| `mst_unit_id` | varchar(255) | 不可 | - | 対応するユニットID（`mst_units.id`） |
| `language` | enum | 不可 | `ja` | 言語設定（現状 `ja` のみ） |
| `name` | varchar(255) | 不可 | - | 必殺技名 |

---

## Language（enum）

| 値 | 説明 |
|----|------|
| `ja` | 日本語（現在唯一の対応言語） |

## 命名規則 / IDの生成ルール

`id` は連番整数で管理する（既存の最大IDの次の番号を使用）。

## 設定時のポイント

1. **`mst_unit_id` は `MstUnit` に存在するIDを使用する**。先にユニットを作成してから必殺技名を設定する。
2. **必殺技名は原作の台詞や技名を参考に設定する**。作品の世界観・キャラクター性を尊重した名前をコンテンツ担当者と連携して決定する。
3. **現状は `language = ja`（日本語）のみ対応**。多言語対応時は同じ `mst_unit_id` で他言語レコードを追加する。
4. **1ユニットにつき1言語1レコード**。同じユニットの同一言語レコードを複数作成しないように注意する。
5. **新キャラ追加時は必ず本テーブルにも必殺技名レコードを追加する**。設定漏れがあると技名テキストが空表示になる。
6. **`id` は連番整数で管理する**。既存レコードの最大IDを確認して次の番号を採番する。
7. **特殊文字（記号・感嘆符・括弧等）が含まれる場合はCSV上のエスケープに注意する**。ダブルクォート含む文字列はCSVの仕様に従ってエスケープする。
