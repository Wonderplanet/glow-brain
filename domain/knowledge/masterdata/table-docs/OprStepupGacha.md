# OprStepupGacha 詳細説明

> CSVパス: `projects/glow-masterdata/OprStepupGacha.csv`

---

## 概要

ステップアップガシャの全体設定を管理するテーブル。`opr_gachas` の `gacha_type=Stepup` に対応する設定として、最大ステップ数と最大周回数を定義する。

各ステップの詳細設定（コスト・排出設定）は `opr_stepup_gacha_steps` テーブルで管理し、このテーブルはステップアップガシャの「親」設定として機能する。

クライアントクラス: `OprStepupGachaData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

---

## 全カラム一覧

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|-----|------|-----------|------|
| id | varchar(255) | NO | - | UUID（整数連番で管理されている） |
| release_key | bigint | NO | 1 | リリースキー |
| opr_gacha_id | varchar(255) | NO | - | opr_gachas.id（UNIQUE制約: 1ガシャに1設定のみ） |
| max_step_number | tinyint unsigned | NO | - | 最大ステップ数（例: 5 → ステップ1〜5を繰り返す） |
| max_loop_count | int unsigned | YES | - | 最大周回数（NULLで無制限） |
| created_at | timestamp | YES | - | 作成日時 |
| updated_at | timestamp | YES | - | 更新日時 |

---

## 他テーブルとの連携

| 関連テーブル | 関係 | 内容 |
|------------|------|------|
| opr_gachas | 1:1 (opr_gacha_id, UNIQUE) | opr_gacha_id でステップアップガシャ本体と紐付く |
| opr_stepup_gacha_steps | 1:N (opr_gacha_id経由) | 各ステップの詳細設定（コスト・排出） |
| opr_stepup_gacha_step_rewards | 1:N (opr_gacha_id経由) | ステップごとのおまけ報酬設定 |
| opr_master_releases | 参照 | release_key によるリリース制御 |

## 実データ例

**例1: 5ステップ・実質無制限周回のステップアップガシャ**
```
id: 1
release_key: 999999999
opr_gacha_id: step_1
max_step_number: 5
max_loop_count: 99999
```

**例2: 5ステップ・1周限定のステップアップガシャ**
```
id: 2
release_key: 999999999
opr_gacha_id: step_2
max_step_number: 5
max_loop_count: 1
```

**例3: 2ステップ・2周限定の短期ステップアップガシャ**
```
id: 3
release_key: 999999999
opr_gacha_id: step_3
max_step_number: 2
max_loop_count: 2
```

## 設定時のポイント

1. **opr_gachas の gacha_type=Stepup のガシャに対して必ず1レコード設定が必要**: このレコードがないとステップアップガシャとして機能しない。
2. **opr_gacha_id は UNIQUE 制約**: 1つのガシャに対してステップアップ設定は1つのみ。
3. **max_step_number は opr_stepup_gacha_steps のステップ数と一致させる**: ステップが足りない場合は動作に不整合が生じる可能性がある。
4. **max_loop_count=NULL は無制限周回**: NULLを設定すると何度でもステップ1から繰り返す。
5. **max_loop_count=1 は1周限定**: ステップ5まで引いたら終了する（再挑戦不可）。
6. **max_loop_count と opr_stepup_gacha_step_rewards の loop_count_target との連携**: 特定周回数だけのおまけ報酬を設定する場合に参照される。
