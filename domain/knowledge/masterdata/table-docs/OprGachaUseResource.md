# OprGachaUseResource 詳細説明

> CSVパス: `projects/glow-masterdata/OprGachaUseResource.csv`

---

## 概要

ガシャを引くために必要な消費リソース（コスト）の設定を管理するテーブル。1つのガシャに対して複数のコスト種別（ダイヤ・チケット・広告など）を設定でき、それぞれの消費量と何連分に対応するかを定義する。

`cost_priority` により優先度が設定され、クライアント側でどのコストを優先的に使用するかの順序制御に使用される。

クライアントクラス: `OprGachaUseResourceData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

---

## 全カラム一覧

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|-----|------|-----------|------|
| id | varchar(255) | NO | - | UUID（整数連番で管理されている） |
| opr_gacha_id | varchar(255) | NO | - | opr_gachas.id |
| cost_type | enum | NO | Diamond | 使用するコストのタイプ |
| cost_id | varchar(255) | YES | - | 消費リソースのID（Item の場合のみ指定） |
| cost_num | int unsigned | NO | 1 | 1回分消費する数量 |
| draw_count | int unsigned | NO | 1 | cost_num を消費して引ける回数 |
| cost_priority | int unsigned | NO | 1 | コスト使用優先度（数値が小さいほど優先） |
| release_key | bigint | NO | 1 | リリースキー |
| created_at | timestamp | YES | - | 作成日時 |
| updated_at | timestamp | YES | - | 更新日時 |

## CostType（コストタイプ enum）

| 値 | 説明 |
|----|------|
| Diamond | ダイヤ（有償・無償混合） |
| PaidDiamond | 有償ダイヤ限定 |
| Free | 無料（コスト不要） |
| Item | アイテム（チケット等）（cost_id に mst_items.id を指定） |
| Ad | 広告視聴 |

## 設定時のポイント

1. **1つのガシャに複数コストを設定可能**: 広告→チケット→ダイヤの順で優先度を設定することで、所持状況に応じた最適な消費が行われる。
2. **cost_priority は数値が小さいほど優先される**: 同じ優先度を持つコストが複数ある場合はシステム側のルールに従う。
3. **cost_type=Item の場合は cost_id（チケットIDなど）が必須**: Diamond や Ad の場合は cost_id を NULL にする。
4. **draw_count で単発と多連を区別**: 同一 cost_type で draw_count=1（単発）と draw_count=10（10連）を別レコードで設定することで異なる料金設定が可能。
5. **UNIQUE制約により (opr_gacha_id, cost_type, draw_count) の組み合わせは一意**: 同じコストタイプ・同じ連数の設定を重複して追加することはできない。
6. **Free タイプはコスト不要ガシャ**: 1日1回無料ガシャなど、フリーコストでガシャを提供する場合に使用する。
7. **Ad タイプは広告視聴で引けるガシャ**: opr_gachas.enable_ad_play=1 と組み合わせて使用する。
