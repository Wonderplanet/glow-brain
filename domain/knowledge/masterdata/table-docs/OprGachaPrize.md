# OprGachaPrize 詳細説明

> CSVパス: `projects/glow-masterdata/OprGachaPrize.csv`

---

## 概要

ガシャの排出物（景品）設定を管理するテーブル。ユニット・アイテム・コインなどの排出リソースと、その出現比重（weight）を定義する。

複数のレコードが同じ `group_id` を持つことで1つの「抽選テーブル」を形成し、`opr_gachas.prize_group_id` および `opr_gachas.fixed_prize_group_id` からグループIDで参照される。ピックアップ対象か否かも本テーブルで管理する。

クライアントクラス: `GachaPrizeData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)
※ OprGachaPrize専用のクライアントクラスは存在せず、スキーマ上のGachaPrizeDataが共通利用される。

---

## 全カラム一覧

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|-----|------|-----------|------|
| id | varchar(255) | NO | - | UUID（排出物レコード識別子） |
| release_key | bigint | NO | 1 | リリースキー |
| group_id | varchar(255) | NO | - | 同一抽選テーブルをまとめるグループID |
| resource_type | enum | NO | - | 排出リソースのタイプ |
| resource_id | varchar(255) | YES | - | 排出リソースのID（コインの場合はNULL） |
| resource_amount | int unsigned | YES | - | 排出リソースの数量 |
| weight | bigint unsigned | NO | 1 | 出現比重（相対値） |
| pickup | tinyint | NO | 0 | ピックアップ対象か（0=非ピックアップ、1=ピックアップ） |
| created_at | timestamp | YES | - | 作成日時 |
| updated_at | timestamp | YES | - | 更新日時 |

---

## ResourceType（排出リソースタイプ enum）

| 値 | 説明 |
|----|------|
| Coin | コイン（resource_id=NULL） |
| Unit | ユニット（キャラクター）（resource_id=mst_units.id） |
| Item | アイテム（resource_id=mst_items.id） |

---

## 命名規則 / IDの生成ルール

- group_id は `opr_gachas.prize_group_id` に設定した値と同名
  - 例: 通常排出グループ `Special_001`、確定枠グループ `fixd_Special_001`
- レコードIDは `{group_id}_{連番}` のパターンが多い
  - 例: `Tutorial_001_1`、`Tutorial_001_2`

## 他テーブルとの連携

| 関連テーブル | 関係 | 内容 |
|------------|------|------|
| opr_gachas | N:1 (group_id経由) | prize_group_id / fixed_prize_group_id でこのテーブルを参照 |
| opr_stepup_gacha_steps | N:1 (prize_group_id経由) | ステップガシャのステップごとの排出テーブル |
| mst_units | 参照 | resource_type=Unit の場合の resource_id |
| mst_items | 参照 | resource_type=Item の場合の resource_id |
| opr_master_releases | 参照 | release_key によるリリース制御 |

UNIQUE制約: `(group_id, resource_type, resource_id)` の組み合わせで一意

```

resource_amount: 1
```

**例2: ピックアップ対象ユニット（pickup=1）**
```

resource_amount: 1
weight: 990
pickup: 1
```

## 設定時のポイント

1. **同じ group_id のレコード群が1つの抽選テーブルを形成する**: weight の総和に対する各 weight の比率が確率となる。
2. **ピックアップ対象は pickup=1 に設定**: ピックアップ天井（OprGachaUpper の Pickup タイプ）と組み合わせて動作する。
3. **resource_type=Coin の場合は resource_id が NULL**: コインはIDなしで量のみ指定する。
4. **確定枠用グループと通常排出グループは別々の group_id で管理**: opr_gachas で `prize_group_id` と `fixed_prize_group_id` に別々の group_id を指定する。
5. **UNIQUE制約により同一グループ内で同一リソースの重複設定は不可**: 同じキャラを複数のweightで登録したい場合は resource_id を変えるかグループを分ける必要がある。
6. **weight は相対比重なので絶対値は問わない**: 運営の確率計算に基づいた weight を設定すること。
7. **ステップアップガシャでも同テーブルを参照**: opr_stepup_gacha_steps の prize_group_id / fixed_prize_group_id もこのテーブルを参照する。
