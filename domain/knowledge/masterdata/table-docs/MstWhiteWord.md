# MstWhiteWord 詳細説明

> CSVパス: `projects/glow-masterdata/MstWhiteWord.csv`

---

## 概要

NGワードフィルタリングから除外するホワイトワード（許可単語）を定義するテーブル。
通常のNGワードチェックでは禁止されてしまうが、ゲーム内での使用を許可したい単語（固有名詞・地名・人名など）を登録する。
プレイヤー名や自由入力欄のテキストフィルタリングで使用される。

---

## 全カラム一覧（テーブル形式）

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | varchar(255) | NOT NULL | - | ホワイトワードの識別ID |
| word | varchar(255) | NOT NULL | - | NGワードから除外するホワイトワード |
| release_key | bigint | NOT NULL | 1 | リリースキー |

---

## 命名規則 / IDの生成ルール

`id` は `wh_{連番4桁}` の形式で命名される。
例: `wh_0001`, `wh_0002`, `wh_0010`

|---|---|---|

## 設定時のポイント

1. **id は `wh_{4桁連番}` の命名規則を遵守する**: 既存の最大番号に +1 して新しいIDを付番する。
2. **NGワードに含まれる部分文字列も許可対象となる**: ホワイトワードはNGワードフィルタを通過させる単語であるため、NGワードリストとの整合性を確認して登録する。
3. **主に日本語の固有名詞を登録**: 人名・地名・漢字の固有名詞で、NGワードに誤検知される可能性があるものを対象とする。
4. **単語の重複登録は避ける**: 同じ単語を複数IDで登録しないように注意する。
5. **漢字・ひらがな・カタカナすべて登録対象**: 表記揺れがある場合は複数の表記をそれぞれ登録することを検討する。
6. **プレイヤーが不適切な単語を入力した際にホワイトワードが誤認識されないよう注意**: ホワイトワードに登録した単語が別の文脈でNGとなる組み合わせにならないか確認する。
7. **release_key はリリース管理に使用**: NGワード・ホワイトワードの追加・更新時は適切なリリースキーを設定する。
