# MstUnitRankCoefficient 詳細説明

> CSVパス: `projects/glow-masterdata/MstUnitRankCoefficient.csv`

---

## 概要

キャラクター（ユニット）のランクアップ後のステータス係数を定義するテーブル。
ランク（1〜6）ごとに、通常ユニットおよびスペシャルユニット専用の2種類のステータス上昇係数を設定する。
ランクが高いほど係数が大きくなり、スペシャルユニットは通常ユニットより大幅に高い係数が設定されている。

---

## 全カラム一覧（テーブル形式）

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | varchar(255) | NOT NULL | - | UUID（CSVでは連番整数） |
| rank | int unsigned | NOT NULL | - | ユニットのランク（1〜6） |
| coefficient | int | NOT NULL | - | 通常ユニット用のランクステータス上昇係数 |
| special_unit_coefficient | int | NOT NULL | - | スペシャルキャラ用のランクステータス上昇係数 |
| release_key | bigint | NOT NULL | 1 | リリースキー |

---

## 他テーブルとの連携

| 関連テーブル | 連携カラム | 説明 |
|---|---|---|
| mst_unit_rank_ups | rank | ランクアップ条件の定義 |
| mst_unit_specific_rank_ups | rank | 個別ユニットのランクアップ条件 |
| mst_units | - | ユニット本体のスペシャル判定と連携 |

## 設定時のポイント

1. **ランク数は現在6段階**: ランク1〜6の6段階が定義されている。ランク追加時は新しいレコードを追加する。
2. **スペシャルユニット係数は通常の約7〜20倍**: スペシャルキャラクターは大幅に高い係数が設定されており、別系統のバランス調整が行われている。
3. **係数は単調増加で設定**: ランクが高いほど係数が大きくなるように設計する。逆転が生じないよう注意する。
4. **ランクアップテーブルと整合性を保つ**: mst_unit_rank_ups や mst_unit_specific_rank_ups に定義されているランク値と一致させる。
5. **通常ユニットとスペシャルユニットで別々の係数を管理**: ユニット種別に応じてどちらの係数を使用するかはサーバー側のロジックで判定される。
6. **全ランク分のレコードが揃っていること**: ランク1からランク最大値まで全てのレコードが存在しないとゲームロジックが正しく動作しない。
7. **release_key はリリース管理に使用**: ランク上限追加時は適切なリリースキーを設定する。
