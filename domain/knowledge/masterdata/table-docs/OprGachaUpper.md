# OprGachaUpper 詳細説明

> CSVパス: `projects/glow-masterdata/OprGachaUpper.csv`

---

## 概要

ガシャの天井設定を管理するテーブル。指定した回数ガシャを引いた際に特定レアリティまたはピックアップアイテムを必ず排出することを保証する「天井」機能の定義を行う。

`upper_group` をキーとして `opr_gachas.upper_group` から参照され、同一 `upper_group` に複数の天井タイプ（最高レアリティ天井・ピックアップ天井）を設定することができる。

クライアントクラス: `OprGachaUpperData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

---

## 全カラム一覧

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|-----|------|-----------|------|
| id | varchar(255) | NO | - | UUID（整数連番で管理されている） |
| upper_group | varchar(255) | NO | None | 天井設定区分（opr_gachas.upper_group と対応） |
| upper_type | enum | NO | MaxRarity | 天井タイプ |
| count | int unsigned | NO | - | 天井を保証する回数 |
| release_key | bigint | NO | 1 | リリースキー |
| created_at | timestamp | YES | - | 作成日時 |
| updated_at | timestamp | YES | - | 更新日時 |

---

## UpperType（天井タイプ enum）

| 値 | 説明 |
|----|------|
| MaxRarity | 最高レアリティ（UR）の排出を保証する天井 |
| Pickup | ピックアップ対象の排出を保証する天井 |

---

## 命名規則 / IDの生成ルール

- `upper_group` は `opr_gachas.upper_group` と同名で設定する
  - 例: スペシャルガシャ → `Special_001`、ピックアップガシャ → `Pickup_kai_001`
- `opr_gachas.upper_group` が "None" または NULL のガシャは天井なし
- UNIQUE制約: `(upper_group, upper_type)` の組み合わせで一意（1グループに同タイプの天井は1つのみ）

## 実データ例

**例1: スペシャルガシャの最高レアリティ天井（100連）**


**例2: ピックアップガシャのピックアップ天井（100連）**
```

```

## 設定時のポイント

1. **天井は upper_group 名で opr_gachas と紐付く**: opr_gachas.upper_group と同名の upper_group を本テーブルに登録する。
2. **1つの upper_group に MaxRarity と Pickup の両方を設定できる**: その場合、両方の天井カウントが並行して動作する。
3. **天井なしにする場合は opr_gachas.upper_group を "None" または NULL に設定**: このテーブルへのレコード追加は不要。
4. **count の値は引いた回数の合計**: 多連ガシャの場合は1回のN連で count が N 増加するのが基本。
5. **ピックアップ天井は opr_gacha_prizes.pickup=1 と連携**: pickup フラグが立ったレコードが天井保証の対象となる。
6. **UNIQUE制約により同一グループに同一タイプの天井は1つのみ**: MaxRarity と Pickup は同時に設定可能だが、同タイプの重複設定は不可。
7. **広告での天井カウント制御は opr_gachas.enable_add_ad_play_upper で制御**: このフラグが 0 の場合、広告でガシャを引いても天井カウントが増加しない。
