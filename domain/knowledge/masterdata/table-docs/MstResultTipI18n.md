# MstResultTipI18n 詳細説明

> CSVパス: `projects/glow-masterdata/MstResultTipI18n.csv`

---

## 概要

`MstResultTipI18n` は**バトル敗北時のリザルト画面に表示する汎用Tipsテキストの多言語設定テーブル**（i18n専用テーブル）。

プレイヤーのユーザーレベルに応じて異なるTipsを表示する。レベル帯ごとに最適なアドバイスを出し分けることで、初心者から上級者まで有益な情報を提供できる。

### ゲームへの影響

- バトルで敗北した際のリザルト画面で、プレイヤーのレベルに対応したTipsが1件表示される。
- `user_level` の値を上限として、プレイヤーのレベルに最も近いレコードのTipsが採用される。
- Tipsは `language` によって多言語表示が可能だが、現状は `ja`（日本語）のみ定義されている。

### テーブル連携図

```
（ペアとなるメインテーブルなし）
MstResultTipI18n（単独のi18nテーブル）
  └─ user_level → プレイヤーのレベルと照合
```

## 全カラム一覧

| カラム名 | 型 | NULL許容 | デフォルト値 | 説明 |
|---------|----|---------|-----------|----|
| `ENABLE` | varchar | - | - | CSVの有効フラグ。`e` = 有効 |
| `id` | varchar(255) | 不可 | - | 主キー（連番整数） |
| `language` | enum | 不可 | `ja` | 言語コード。現状は `ja` のみ |
| `user_level` | int unsigned | 不可 | - | このTipsが表示されるユーザーレベルの上限値 |
| `result_tips` | varchar(255) | 不可 | - | Tipsのテキスト本文（改行は `\n` で表現） |
| `release_key` | bigint | 不可 | 1 | リリースキー |

---

## Language（enum）

| 値 | 説明 |
|----|------|
| `ja` | 日本語（現在唯一の対応言語） |

---

## 命名規則 / IDの生成ルール

`id` は連番整数で管理する。

`user_level` はTipsが切り替わる**レベルの閾値**を表す。例えば `user_level = 5` のレコードは、プレイヤーレベルが1〜5のときに表示対象となる候補。

---

## 実データ例

### パターン1: 初級者向けTips（レベル1〜10）

```csv
ENABLE,id,language,user_level,result_tips,release_key
e,1,ja,1,キャラのレベルを強化しよう!\nコインが足りない場合は、\nコイン獲得クエストを遊んでみよう!,202509010
e,2,ja,5,キャラのレベルを強化しよう!\nコインが足りない場合は、\nコイン獲得クエストを遊んでみよう!,202509010
e,3,ja,10,キャラのレベルを強化しよう!\nコインが足りない場合は、\nコイン獲得クエストを遊んでみよう!,202509010
```

- レベル1・5・10は同じTipsを表示（キャラ強化の促し）

### パターン2: 中級者向けTips（レベル20以降）

```csv
ENABLE,id,language,user_level,result_tips,release_key
e,5,ja,20,キャラをランクアップをしてみよう!\n探索で各種カラーメモリーを\n獲得しよう!,202509010
e,6,ja,30,キャラをランクアップをしてみよう!\n探索で各種カラーメモリーを\n獲得しよう!,202509010
```

- レベル20・30ではランクアップを促すTipsに切り替わる

## 設定時のポイント

1. **`user_level` はTipsが表示される上限レベルとして機能する**。プレイヤーのレベルが `user_level` 以下のときに対象候補になる。
2. **レベル帯ごとに段階的なTipsを設定する**。初心者はキャラ強化・育成の基本、上級者はランクアップやゲートなどの高度なコンテンツを促すTipsを用意するのが理想。
3. **テキストの改行は `\n` で記述する**。CSV上の実際の改行文字は使用しない。
4. **現状は `language = ja` のみ対応**。多言語展開する際は同じ `user_level` で他言語レコードを追加する。
5. **同一レベルに複数のTipsを設定することも可能**（ランダム表示の実装による）。レベル帯ごとに複数の候補があると、表示の多様性が生まれる。
6. **`id` は連番整数で管理する**。既存レコードの最大IDを確認して次の番号を採番する。
7. **敗北理由に応じた具体的なアドバイスを書く**。「強くなる方法」だけでなく、「なぜ負けたか」に対する解決策を示すと効果的。
