# OprStepupGachaStepReward 詳細説明

> CSVパス: `projects/glow-masterdata/OprStepupGachaStepReward.csv`

---

## 概要

ステップアップガシャのステップごとに付与される「おまけ報酬」を管理するテーブル。特定のステップを引いた際に、ガシャの排出物とは別に追加で付与されるボーナス報酬（無償ダイヤ・コイン・スタミナアイテム・ユニットなど）を定義する。

`loop_count_target` により、特定周回数のみに付与する報酬と全周回共通で付与する報酬を区別して設定できる。

クライアントクラス: `OprStepupGachaStepRewardData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

## loop_count_target の動作

| 値 | 説明 |
|----|------|
| NULL | 全周回で付与（毎ステップ引くたびに付与） |
| 0 | 最初の周のみ付与（初回限定報酬） |
| 1 | 1周目（最初の周）のみに付与 |
| N（N>1） | N周目のみに付与（特定周回限定） |

## 設定時のポイント

1. **同一ステップに複数の報酬を設定可能**: step_number が同じレコードを複数追加することで、1ステップで複数種類のおまけを付与できる。
2. **loop_count_target=NULL は毎周回付与**: ステップアップガシャを引くたびに付与されるので設定量に注意が必要。
3. **loop_count_target=0 は初回のみ付与**: max_loop_count が 1 のガシャでは loop_count_target=NULL と同じ効果になる。
4. **resource_type によって resource_id の要否が異なる**: FreeDiamond・Coin・Stamina・Exp は resource_id=NULL で問題ない。Item・Unit・Artwork などは対応するIDが必要。
5. **opr_stepup_gachas.max_loop_count と loop_count_target の整合性確認**: max_loop_count を超えた loop_count_target を設定しても報酬は付与されない。
6. **報酬量（resource_amount）は慎重に設定**: 全周回共通で設定するとゲームバランスに大きく影響するため、周回数上限と報酬量を合わせて設計する。
7. **idx_opr_gacha_id_step インデックス**: ガシャIDとステップ番号での検索が高速化されている。
