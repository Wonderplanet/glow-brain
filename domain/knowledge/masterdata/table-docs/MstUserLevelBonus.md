# MstUserLevelBonus 詳細説明

> CSVパス: `projects/glow-masterdata/MstUserLevelBonus.csv`

---

## 概要

プレイヤーが特定のレベルに到達したときに付与される報酬グループを設定するテーブル。
どのレベルに到達した際にどの報酬グループを付与するかをマッピングし、mst_user_level_bonus_groups の報酬詳細と組み合わせて動作する。
level のユニーク制約により、1つのレベルに対して1つの報酬グループのみを設定できる。

---

## 全カラム一覧（テーブル形式）

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | varchar(255) | NOT NULL | - | UUID（CSVでは連番整数） |
| level | int | NOT NULL | - | 報酬が付与されるプレイヤーレベル |
| mst_user_level_bonus_group_id | varchar(255) | NOT NULL | - | 付与する報酬グループのID |
| release_key | bigint | NOT NULL | 1 | リリースキー |

---

## 命名規則 / IDの生成ルール

CSVの `id` は連番整数で管理される。`level` 列には `mst_user_level_bonuses_level_unique` ユニーク制約があるため、各レベルに対して1レコードのみ登録可能。

---

## 他テーブルとの連携

| 関連テーブル | 連携カラム | 説明 |
|---|---|---|
| mst_user_level_bonus_groups | mst_user_level_bonus_group_id | 報酬グループの詳細（アイテム・数量） |
| mst_user_levels | level | プレイヤーレベルの定義（レベルが存在すること） |

## 設定時のポイント

1. **level のユニーク制約**: 1つのレベルに対して1つの報酬グループしか設定できない（`mst_user_level_bonuses_level_unique` 制約）。
2. **5レベル刻みで報酬を設定する設計**: 現行データでは5の倍数レベル（10・15・20...）に報酬が設定されている。
3. **グループIDの使い回しが可能**: 基本的な報酬（グループ1）は多くのレベルで共有されており、特別なレベル（20・30・40...）に別グループを割り当てる設計。
4. **報酬グループは事前にmst_user_level_bonus_groupsに作成しておく**: 参照するグループIDが存在しないと整合性エラーになる。
5. **level はmst_user_levelsに存在するレベルであること**: 存在しないレベルへの設定は無意味になる。
6. **レベル上限追加時は本テーブルとmst_user_level_bonus_groupsを同時に更新する**: 新しいレベルに報酬を設定する場合は両テーブルの更新が必要。
7. **release_key はリリース管理に使用**: 新しいレベル上限追加時は適切なリリースキーを設定する。
