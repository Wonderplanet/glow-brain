# MstPartyUnitCount 詳細説明

> CSVパス: `projects/glow-masterdata/MstPartyUnitCount.csv`

---

## 概要

`MstPartyUnitCount` は**メインクエストの進捗に応じて、プレイヤーがパーティに編成できるキャラクター数の上限を定義するテーブル**。特定のステージをクリアしていることを条件として、パーティ編成可能なユニット数の上限（`max_count`）が変化する仕組みを実現する。

現在の実データでは `mst_stage_id` が NULL のレコード（最大10人）が1件のみ存在しており、初期状態または汎用のパーティ上限として機能している。

### ゲームプレイへの影響

- `mst_stage_id` に対応するステージをクリアしたプレイヤーは `max_count` 人までパーティにユニットを編成できる
- 段階的にパーティ枠を解放することで、ゲーム進行に応じた成長を演出できる

---

## 全カラム一覧

| カラム名 | 型 | NULL許容 | デフォルト値 | 説明 |
|---------|----|---------|-----------|----|
| `ENABLE` | varchar | - | - | CSVの有効フラグ。`e` = 有効 |
| `id` | varchar(255) | 不可 | - | 主キー |
| `mst_stage_id` | varchar(255) | 不可 | - | 参照先ステージID（`mst_stages.id`）。このステージ以降に適用 |
| `max_count` | int unsigned | 不可 | - | 設定可能なパーティキャラ数の上限 |
| `release_key` | bigint | 不可 | 1 | リリースキー |

---

## 命名規則 / IDの生成ルール

- `id`: `party_unit_count_{max_count}` 形式が多い（例: `party_unit_count_10`）

---

## 他テーブルとの連携

```
MstPartyUnitCount
  └─ mst_stage_id → MstStage.id（進捗判定に使用するステージ）
```

`mst_stage_id` に一意制約（`mst_party_unit_counts_mst_stage_id_unique`）が設定されており、同じステージIDに対して複数の上限設定は不可。

|--------|-----|-----------|---------|-------------|

---

## 設定時のポイント

1. **mst_stage_idに一意制約あり**: 同じステージIDに複数レコードは作成できない。1ステージにつき1つのパーティ上限を設定する
2. **段階的解放の設計**: ゲーム序盤は少ない人数からスタートし、特定ステージクリアで拡張するような設計ができる
3. **mst_stage_idがNULLの意味**: 現状のデータでは「ステージ進捗に関係なく常に適用される」デフォルト上限として機能している
4. **max_countはゲームデザインと整合させる**: UIのパーティ編成画面で表示できる最大スロット数と一致させる必要がある
5. **運用変更に注意**: パーティ上限を変更する場合は既存プレイヤーへの影響を確認する
