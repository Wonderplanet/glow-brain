# MstQuestBonusUnit 詳細説明

> CSVパス: `projects/glow-masterdata/MstQuestBonusUnit.csv`

---

## 概要

`MstQuestBonusUnit` は**特定のクエストにおいて、特定のユニット（キャラ）をパーティに編成するとコイン獲得量がボーナスアップする設定テーブル**。

1レコードが「特定のクエストで特定のユニットを使ったときのコインボーナス倍率と有効期間」を表す。運営施策として期間限定でボーナスを設定するケースが多い。

### ゲームへの影響

- 対象ユニットをパーティに編成してクエストをクリアすると、通常の獲得コイン量に `coin_bonus_rate` を加算した倍率でコインを獲得できる。
- 複数のユニットにボーナスが設定されている場合、全ユニットのボーナスが加算される（スタック制）。
- **有効期間** (`start_at` / `end_at`) で期間限定ボーナスを制御する。期間外のレコードは無効。

### テーブル連携図

```
MstQuest（クエスト）
  └─ id → MstQuestBonusUnit.mst_quest_id（1:N）
              └─ mst_unit_id → MstUnit.id（ボーナス対象ユニット）
```

---

## 全カラム一覧

| カラム名 | 型 | NULL許容 | デフォルト値 | 説明 |
|---------|----|---------|-----------|----|
| `ENABLE` | varchar | - | - | CSVの有効フラグ。`e` = 有効 |
| `id` | varchar(255) | 不可 | - | 主キー（連番整数）|
| `release_key` | bigint | 不可 | 1 | リリースキー |
| `mst_quest_id` | varchar(255) | 不可 | - | 対象クエストID（`mst_quests.id`） |
| `mst_unit_id` | varchar(255) | 不可 | - | ボーナス対象ユニットID（`mst_units.id`） |
| `coin_bonus_rate` | double | 不可 | - | コイン獲得量の上昇倍率（例: `0.1` = +10%） |
| `start_at` | timestamp | 不可 | - | ボーナス有効開始日時 |
| `end_at` | timestamp | 不可 | - | ボーナス有効終了日時 |

---

## 命名規則 / IDの生成ルール

`id` は連番整数で管理する（既存の最大IDの次の番号を使用）。

インデックス: `(mst_quest_id, mst_unit_id)` の複合インデックスが設定されており、クエスト×ユニットの組み合わせで高速検索できる。

| `MstQuest` | `MstQuestBonusUnit.mst_quest_id = MstQuest.id` | クエスト情報を取得 |
| `MstUnit` | `MstQuestBonusUnit.mst_unit_id = MstUnit.id` | ボーナス対象ユニット情報を取得 |

## 実データ例

### パターン1: 強化クエストへの初期ボーナス設定

```csv
ENABLE,id,mst_quest_id,mst_unit_id,coin_bonus_rate,start_at,end_at,release_key
e,1,quest_enhance_00001,chara_kai_00101,0.1,2025-09-22 11:00:00,2025-10-22 11:59:59,202509010
e,2,quest_enhance_00001,chara_kai_00301,0.1,2025-09-22 11:00:00,2025-10-22 11:59:59,202509010
e,3,quest_enhance_00001,chara_kai_00001,0.1,2025-09-22 11:00:00,2025-10-22 11:59:59,202509010
```

- `coin_bonus_rate = 0.1` は +10% のボーナスを意味する
- `quest_enhance_00001` に `kai` シリーズの複数キャラが設定されている

### パターン2: 期間を変えて追加ボーナスを設定

```csv
ENABLE,id,mst_quest_id,mst_unit_id,coin_bonus_rate,start_at,end_at,release_key
e,9,quest_enhance_00001,chara_spy_00001,0.1,2025-10-06 12:00:00,2025-11-06 14:59:59,202510010
e,10,quest_enhance_00001,chara_spy_00101,0.1,2025-10-06 12:00:00,2025-11-06 14:59:59,202510010
```

- 翌リリース（202510010）で `spy` シリーズのキャラボーナスが追加
- 期間が重ならない場合はリリースキーを変えて追加していく

## 設定時のポイント

1. **`mst_quest_id` は強化クエスト（`Enhance`種別）に設定するのが主なユースケース**。メインクエストへの設定も可能だが、運用上は強化クエストが中心。
2. **`coin_bonus_rate` は加算倍率**。`0.1` が +10% を意味し、複数ユニットのボーナスは加算される。
3. **同一クエスト・ユニットの組み合わせで有効期間が重複しないように注意する**。期間が被ると意図しない二重ボーナスが発生する可能性がある。
4. **`start_at` / `end_at` は期間限定ボーナスとして運用する**。恒常化する場合は `end_at` に遠い未来の日時を設定する。
5. **新キャラ追加時は強化クエストへのボーナス設定もセットで行う**。キャラ育成促進のため、同一作品のキャラへのボーナスを追加するのが慣例。
6. **`id` は連番整数で管理する**。既存の最大IDを確認してから次の番号を採番する。
