# OprStepupGachaStep 詳細説明

> CSVパス: `projects/glow-masterdata/OprStepupGachaStep.csv`
> i18n CSVパス: `projects/glow-masterdata/OprStepupGachaStepI18n.csv`

---

## 概要

ステップアップガシャの各ステップ詳細設定を管理するテーブル。コスト種別・コスト量・排出回数・確定枠設定など、ステップごとに異なる条件を定義する。

多言語テーブル `opr_stepup_gacha_steps_i18n` とペアで使用し、確定枠に関する表示テキスト（例:「XXキャラ1体以上確定！」）を言語別に管理する。

クライアントクラス: `OprStepupGachaStepData.cs` / `OprStepupGachaStepI18nData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

## 設定時のポイント

1. **ステップ番号は step_number=1 から開始**: max_step_number まで連番で定義する必要がある。
2. **UNIQUE制約により (opr_gacha_id, step_number) の組み合わせは一意**: 同一ガシャで同じステップ番号は設定不可。
3. **fixed_prize_count=0 は確定枠なし**: 確定枠がある場合は 1 以上の値を設定し、fixed_prize_group_id も必須となる。
4. **fixed_prize_rarity_threshold_type でレアリティ閾値を指定**: "UR" などのレアリティを指定すると、確定枠でそのレアリティ以上が保証される。
5. **prize_group_id が NULL のステップはデフォルト排出テーブルを使用**: 通常は opr_gachas.prize_group_id が使われる。
6. **is_first_free=1 は最初の周のみ無料**: 2周目以降は通常コストが発生する。
7. **i18n の fixed_prize_description はリッチテキスト対応**: `<color>` タグで強調表示が可能。
8. **i18n の UNIQUE制約**: (opr_stepup_gacha_step_id, language) の組み合わせで一意。
