# OprGachaDisplayUnitI18n 詳細説明

> CSVパス: `projects/glow-masterdata/OprGachaDisplayUnitI18n.csv`

---

## 概要

ガシャ画面に表示するピックアップユニット（キャラクター）情報の多言語テーブル。ガシャ詳細UI上でどのキャラクターを何番目に表示するか、また各キャラクターに対してどのような説明テキストを表示するかを定義する。

このテーブルはメインテーブル（`opr_gacha_display_units`）が存在せず、i18n専用テーブルとして独立している。ガシャ（`opr_gachas`）と直接 `opr_gacha_id` で紐付く。

クライアントクラス: `OprGachaDisplayUnitI18nData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

---

## 全カラム一覧

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---------|-----|------|-----------|------|
| id | varchar(255) | NO | - | UUID（整数連番で管理されている） |
| release_key | bigint | NO | 1 | リリースキー |
| opr_gacha_id | varchar(255) | NO | "" | 表示対象のガシャ（opr_gachas.id） |
| mst_unit_id | varchar(255) | NO | "" | 表示するキャラ（mst_units.id） |
| language | enum('ja') | NO | - | 言語情報（現在は日本語のみ） |
| sort_order | int | NO | 0 | キャラの表示順（昇順） |
| description | varchar(255) | NO | "" | 表示キャラごとの説明文言 |

---

## 他テーブルとの連携

| 関連テーブル | 関係 | 内容 |
|------------|------|------|
| opr_gachas | N:1 | opr_gacha_id でガシャと紐付く |
| mst_units | N:1 | mst_unit_id でキャラクターのマスタデータを参照 |
| opr_master_releases | 参照 | release_key によるリリース制御 |

UNIQUE制約: `(opr_gacha_id, mst_unit_id, language)` の組み合わせで一意

## 実データ例

**例1: ピックアップガシャのキャラ表示（1番目）**
```
id: 1
release_key: 202509010
opr_gacha_id: Pickup_kai_001
mst_unit_id: chara_kai_00201
language: ja
sort_order: 1
description: 遠距離から単体と範囲攻撃を使い分ける器用なアタッカー！
```

**例2: スタートダッシュガシャのキャラ表示（複数設定）**
```
id: 3
release_key: 202509010
opr_gacha_id: StartDash_001
mst_unit_id: chara_gom_00001
language: ja
sort_order: 1
description: 必殺ワザと特性で\n耐久性抜群の\nディフェンスキャラ!

id: 4
release_key: 202509010
opr_gacha_id: StartDash_001
mst_unit_id: chara_jig_00001
language: ja
sort_order: 2
description: 火傷付与と\n状態異常軽減を\n兼ね揃えている\nテクニカルキャラ!
```

## 設定時のポイント

1. **sort_order で表示順を制御**: 数値が小さいほど先に表示される（昇順）。
2. **description は改行コード（\n）対応**: ガシャ画面のUIレイアウトに合わせて改行を含めることが多い。
3. **ピックアップガシャ以外にも設定可能**: スタートダッシュや各種チケットガシャにもキャラ説明を付与できる。
4. **1ガシャに複数のキャラを設定できる**: sort_order を異なる値にして複数レコードを追加する。
5. **UNIQUE制約に注意**: 同一ガシャ・同一ユニット・同一言語の組み合わせは1レコードのみ設定可能。
6. **opr_gachas.display_mst_unit_id との関係**: `display_mst_unit_id` はサマリ的な参照で、詳細な説明文はこのテーブルで管理する。
