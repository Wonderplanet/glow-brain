# MstUserLevel 詳細説明

> CSVパス: `projects/glow-masterdata/MstUserLevel.csv`

---

## 概要

プレイヤー（ユーザー）のレベルに関する設定を定義するテーブル。
各レベルに対して、時間回復するスタミナの最大値とそのレベルに到達するために必要な累計経験値を設定する。
プレイヤーレベルが上がるにつれてスタミナ上限が増加し、累計経験値の要求量も増加する。

---

## 全カラム一覧（テーブル形式）

| カラム名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | varchar(255) | NOT NULL | - | プレイヤーレベルID（CSVでは連番整数） |
| level | int | NOT NULL | - | プレイヤーレベル |
| stamina | int | NOT NULL | - | 時間回復最大値のスタミナ |
| exp | bigint | NOT NULL | - | レベルアップに必要な累計経験値 |
| release_key | bigint | NOT NULL | 1 | リリースキー |

---

## 他テーブルとの連携

| 関連テーブル | 連携カラム | 説明 |
|---|---|---|
| mst_user_level_bonuses | level | 特定レベル到達時の報酬定義 |
| mst_user_level_bonus_groups | - | 報酬グループの詳細定義 |

## 設定時のポイント

1. **レベル1の累計経験値は0**: レベル1がスタート地点のため必要経験値は0。全レベル分の累計経験値は単調増加で設定する。
2. **exp は累計経験値**: そのレベルに到達するための総合経験値であり、差分（次レベルまでの必要EXP）ではない点に注意。
3. **スタミナは段階的に増加する**: 毎レベルごとに増加するわけでなく、数レベルごとに +1〜2 ずつ増加するパターン。
4. **レベル上限を追加する際は連続した値で設定**: 既存データの最大レベルより1高いレベルから追加し、スタミナ・経験値の連続性を保つ。
5. **mst_user_level_bonuses との整合性を保つ**: 報酬が設定されているレベルは本テーブルに必ず存在していること。
6. **経験値曲線の設計に注意**: 高レベルになるほど必要経験値の増加率が急激に大きくなるように設計することでゲームバランスを調整する。
7. **release_key はリリース管理に使用**: レベル上限解放時は新しいリリースキーを設定する。
