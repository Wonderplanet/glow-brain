# OprCampaign 詳細説明

> CSVパス: `projects/glow-masterdata/OprCampaign.csv`
> i18n CSVパス: `projects/glow-masterdata/OprCampaignI18n.csv`

---

## 概要

インゲームコンテンツ（クエスト・PvP・アドベントバトル等）に対するキャンペーン設定を管理するテーブル。スタミナ消費半減・経験値倍増・アイテムドロップ率アップ・チャレンジ回数増加など、各種ボーナス施策を期間と対象ごとに定義する。

多言語テーブル `opr_campaigns_i18n` とペアで使用し、UIに表示するキャンペーン説明文（リッチテキスト含む）を管理する。

クライアントクラス: `OprCampaignData.cs` / `OprCampaignI18nData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

## CampaignType（キャンペーンタイプ enum）

| 値 | 説明 |
|----|------|
| Stamina | スタミナ消費量の変更（effect_value=50 → 消費1/2） |
| Exp | 経験値獲得量の変更（effect_value=200 → 2倍） |
| ArtworkFragment | アートワークフラグメントドロップ率変更 |
| ItemDrop | アイテムドロップ率変更 |
| CoinDrop | コインドロップ量変更 |
| ChallengeCount | アドベントバトルのチャレンジ回数変更（effect_value=3 → 3回） |

## 実データ例

**例1: スタミナ消費1/2キャンペーン**
```
id: Campaign1_1
release_key: 999999999
campaign_type: Stamina
target_type: NormalQuest
difficulty: Normal
target_id_type: Quest
target_id: quest_main_spy_normal_1
effect_value: 50
start_at: 2024-10-10 09:00:00
end_at: 2025-04-14 15:00:00
```
i18n (ja): `消費量  <color=#C81919><size=22>1/2</size></color>`

**例2: チャレンジ回数アップキャンペーン（シリーズ指定なし）**
```
id: Campaign1_6
release_key: 999999999

target_type: AdventBattle
difficulty: Normal
target_id_type: Quest
target_id: NULL
effect_value: 3
start_at: 2024-10-10 09:00:00
end_at: 2025-04-14 15:00:00
```
i18n (ja): `<color=#C81919><size=22>53</size><size=17>回</size></color>にUP中!!`

## 設定時のポイント

1. **effect_value の解釈はcampaign_typeに依存する**: Stamina の場合は消費量のパーセント（50=1/2）、Exp/ItemDrop/ArtworkFragment/CoinDrop の場合は倍率パーセント（200=2倍）、ChallengeCount の場合は回数そのものを表す。
2. **target_id はNULL許容**: アドベントバトルの ChallengeCount など、対象を全体に適用する場合は target_id を NULL にする。
3. **target_id_type で参照先テーブルが変わる**: Quest 指定時は mst_quests.id、Series 指定時は mst_series.id を参照する。
4. **i18nのdescriptionはリッチテキスト対応**: `<color>` タグや `<size>` タグを用いてUI上で強調表示が可能。
5. **期間は start_at / end_at で管理**: 終了日時を過ぎると自動的にキャンペーンが無効となる。
6. **release_key で配信制御**: opr_master_releases と連携してリリースタイミングを管理する。
7. **同一クエストに複数のキャンペーンを設定可能**: 例えば Exp と ArtworkFragment を同じ期間・同じクエストに重複設定できる。
8. **i18n の UNIQUE 制約**: opr_campaign_id と language の組み合わせで一意制約が設定されており、1キャンペーン=1言語=1レコードとなる。
