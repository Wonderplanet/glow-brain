# OprProduct 詳細説明

> CSVパス: `projects/glow-masterdata/OprProduct.csv`
> i18n CSVパス: `projects/glow-masterdata/OprProductI18n.csv`

---

## 概要

ユーザーに販売する実際の商品（課金アイテム）を管理するテーブル。1つの `mst_store_products` に対して販売期間・販売回数制限・配布量が異なる複数の `opr_products` レコードを設定できる（1対多の関係）。

多言語テーブル `opr_products_i18n` とペアで使用し、商品ごとのアセットキー（画像など）を言語別に管理する。

クライアントクラス: `OprProductData.cs` / `OprProductI18nData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

## 実データ例

**例1: 通常ダイヤ商品（購入制限なし・常時販売）**
```
id: 1
mst_store_product_id: 1
product_type: Diamond
purchasable_count: NULL
paid_amount: 80
display_priority: 6
start_date: 2024-09-22 12:00:00
end_date: 2034-01-01 00:00:00
release_key: 202509010
```
i18n (ja): asset_key=NULL（汎用デザイン）

**例2: 期間限定お得なダイヤ商品（購入制限あり）**
```
id: 7
mst_store_product_id: 7
product_type: Diamond
purchasable_count: 3
paid_amount: 160
display_priority: 106
start_date: 2025-09-24 14:00:00
end_date: 2025-10-31 11:59:59
release_key: 202509010
```
i18n (ja): asset_key=`increase_prism_00001`（期間限定デザイン）

## 設定時のポイント

1. **mst_store_products との 1:多 関係**: 同じストア商品に対して常時販売版と期間限定版（お得版）を別々の opr_products として設定できる。
2. **purchasable_count=NULL は無制限購入可能**: 通常のダイヤ商品は NULL（無制限）が基本、期間限定特典付き商品は制限数（例: 3回）を設定する。
3. **display_priority は数値が大きいほど上位表示**: 期間限定商品（display_priority=106）が通常商品（6）より上位に表示されるよう設計する。
4. **期間限定商品は start_date / end_date で販売期間を制御**: 販売期間外はクライアントに表示されない。
5. **paid_amount は有償ダイヤの付与量**: 実際の課金額は mst_store_products 側で管理される。
6. **i18n の asset_key は期間限定商品のみ使用することが多い**: 通常商品は NULL で汎用デザインが使用される。
7. **UNIQUE制約 (opr_product_id, language)**: 1商品・1言語につき1つのi18nレコードのみ設定可能。
