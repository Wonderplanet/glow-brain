# OprGacha 詳細説明

> CSVパス: `projects/glow-masterdata/OprGacha.csv`
> i18n CSVパス: `projects/glow-masterdata/OprGachaI18n.csv`

---

## 概要

ガシャの基本設定を管理するテーブル。ガシャタイプ・天井設定・広告連携・排出設定・表示制御・開放条件など、1つのガシャを定義するために必要な全パラメータを保持する。

多言語テーブル `opr_gachas_i18n` とペアで使用し、ガシャ名・説明文・バナー画像URL・背景色などUI表示に関わる情報を管理する。

排出物は `opr_gacha_prizes`、消費リソースは `opr_gacha_use_resources`、天井は `opr_gacha_uppers` と連携して完全なガシャを構成する。

クライアントクラス: `OprGachaData.cs` / `OprGachaI18nData.cs`
(`projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/`)

## 命名規則 / IDの生成ルール

- 通常ガシャ: `Special_001`、`Tutorial_001`
- ピックアップガシャ: `Pickup_{シリーズ略称}_{連番3桁}` 例: `Pickup_kai_001`
- 有償限定: `StartDash_001`
- メダル: `Premiummedal_001`
- チケット限定: `UR_10Ticket_001`
- prize_group_id は id と同じ名称（`Special_001`）、確定枠は `fixd_{id}` プレフィックス（`fixd_Special_001`）
- upper_group は prize_group_id と同名（`Special_001`、`Pickup_kai_001`）

## 設定時のポイント

1. **gacha_type=Stepup の場合は opr_stepup_gachas との連携が必須**: ステップ設定がないと動作しない。
2. **upper_group が "None" または NULL の場合は天井なし**: 天井を設定する場合は opr_gacha_uppers に同名の upper_group レコードを追加する。
3. **prize_group_id と fixed_prize_group_id は opr_gacha_prizes.group_id を参照**: 同じ group_id を持つ複数レコードが1つの抽選テーブルを構成する。
4. **appearance_condition=HasTicket のガシャはチケット所持者のみに表示**: チケットガシャでよく使用する。
5. **unlock_condition_type=MainPartTutorialComplete を使う場合は unlock_duration_hours も必須**: チュートリアル完了後から指定時間が経過するまでガシャが開放されない。
6. **gacha_priority は小さい数値ほど上位に表示**: 同一時期に複数のガシャがある場合の並び順制御に使用する。
7. **display_mst_unit_id は複数のユニットIDをカンマ区切りで格納可能**: より詳細な表示制御は opr_gacha_display_units_i18n で行う。
8. **i18n の opr_gacha_id_unique 制約**: 1ガシャにつき1言語で1レコードのみ設定可能。
