# イベントクエスト・ステージ 要件テキストフォーマット

> **用途**: プランナーがヒアリング結果を記入し、Claudeに渡すことでマスタデータCSVを一括生成するための要件テキスト。
>
> **生成されるCSV**:
> - `MstQuest.csv`（クエスト定義・クエスト数分）
> - `MstQuestI18n.csv`（クエスト名・多言語、クエスト数分）
> - `MstStage.csv`（ステージ定義・ステージ数分）
> - `MstStageI18n.csv`（ステージ名・多言語、ステージ数分）
> - `MstStageEventSetting.csv`（イベント設定・ステージ数分）
> - `MstStageEventReward.csv`（ステージ報酬・報酬行数分）
> - `MstStageClearTimeReward.csv`（タイムクリア報酬・チャレンジ／サベージのステージ数×3行）
> - `MstQuestBonusUnit.csv`（コインボーナスユニット設定・ユニット数分）
> - `MstStageEndCondition.csv`（特殊終了条件・サベージのみ、必要な場合）

## 報酬アイテム名の選択肢と対応ID

Claudeがこのテキストを解釈してIDに変換します。以下の表記を使ってください。

### チケット類

| 表記 | アイテムID | 備考 |
|------|-----------|------|
| ピックアップガシャチケット | `ticket_glo_00003` | ピックアップ対象を引ける |
| スペシャルガシャチケット | `ticket_glo_00002` | どのガシャでも使える |

### 通貨・汎用アイテム

| 表記 | 報酬タイプ | 備考 |
|------|-----------|------|
| コイン | `Coin` | resource_id 不要 |
| プリズム | `FreeDiamond` | 無償ダイヤ。resource_id = `prism_glo_00001` |

### キャラかけら

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| {キャラ名}のかけら | `piece_{作品略称}_{キャラ連番}` | 例: `piece_hut_00201` |

### キャラメモリー（Lv上限開放素材）

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| {キャラ名}のメモリー | `memory_chara_{作品略称}_{キャラ連番}` | 例: `memory_chara_hut_00201` |

### メモリーフラグメント（Lv上限開放素材・汎用）

| 表記 | アイテムID | 備考 |
|------|-----------|------|
| メモリーフラグメント・初級 | `memoryfragment_glo_00001` | SR相当 |
| メモリーフラグメント・中級 | `memoryfragment_glo_00002` | SSR相当 |
| メモリーフラグメント・上級 | `memoryfragment_glo_00003` | UR相当 |

### エンブレム

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| イベントエンブレム（ステージ1） | `emblem_event_{作品略称}_00001` | サベージST1クリア報酬 |
| イベントエンブレム（ステージ2） | `emblem_event_{作品略称}_00002` | サベージST2クリア報酬 |

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| イベントアイテム | `item_{作品略称}_00001` | イベントポイント等の特殊アイテム。作品によって異なる |

> **その他アイテム**: 上記にない場合はアイテムIDを直接記入するか、「アイテム名（item_xxx_xxxxx）× N個」と書く。

## 補足: テーブル間の関係

```
MstEvent（イベント親レコード、別途作成済み）
  └─ id → MstQuest.mst_event_id（1:N）
              ├─ id → MstQuestI18n.mst_quest_id（クエスト表示名）
              ├─ id → MstStage.mst_quest_id（1:N、ステージ一覧）
              │           ├─ id → MstStageI18n.mst_stage_id（ステージ名）
              │           ├─ id → MstStageEventSetting.mst_stage_id（期間・クリア制限）
              │           ├─ id → MstStageEventReward.mst_stage_id（報酬設定）
              │           ├─ id → MstStageClearTimeReward.mst_stage_id（タイム報酬）
              │           └─ prev_mst_stage_id → MstStage.id（ステージ解放連鎖）
              └─ id → MstQuestBonusUnit.mst_quest_id（強化クエスト：コインボーナス）
```

### MstQuest ID 命名規則

```
quest_event_{作品略称}{回次}_{クエスト種別}
```

例:
- `quest_event_hut1_charaget01` → hutイベント第1回・キャラ獲得クエスト01
- `quest_event_hut1_charaget02` → hutイベント第1回・キャラ獲得クエスト02
- `quest_event_hut1_challenge`  → hutイベント第1回・チャレンジクエスト
- `quest_event_hut1_savage`     → hutイベント第1回・サベージクエスト
- `quest_event_hut1_1day`       → hutイベント第1回・1日限定クエスト

### MstStage ID 命名規則

```
event_{作品略称}{回次}_{クエスト種別}_{ステージ連番5桁}
```

例:
- `event_hut1_charaget01_00001` → charaget01の1ステージ目
- `event_hut1_challenge_00004`  → challengeの4ステージ目

### MstQuestI18n ID 命名規則

```
{mst_quest_id}_{language}
```

例: `quest_event_hut1_charaget01_ja`

### MstStageI18n ID 命名規則

```
{mst_stage_id}_{language}
```

例: `event_hut1_charaget01_00001_ja`

### MstQuestBonusUnit の対象クエスト

コインボーナスは強化クエスト（`quest_enhance_00001`）のみに設定する。
イベントクエスト自体には設定しない。

### ステージ間の解放連鎖（prev_mst_stage_id）

- **charaget**: ステージ1は NULL（最初から解放）、ステージ2以降は前ステージIDを設定
- **challenge**: ステージ1の `prev_mst_stage_id` = `charaget01` の最終ステージID（解放条件）
- **savage**: ステージ1の `prev_mst_stage_id` = `charaget01` の最終ステージID（解放条件）
- **1day**: NULL（最初から解放）

### MstStageEventSetting の設定パターン

| クエスト種別 | reset_type | clearable_count | 備考 |
|------------|-----------|----------------|------|
| charaget   | NULL       | NULL（無制限） | 何度でもプレイ可 |
| challenge  | NULL       | NULL（無制限） | 何度でもプレイ可 |
| savage     | NULL       | NULL（無制限） | 何度でもプレイ可 |
| 1day       | Daily      | 1              | 1日1回のみ |

### MstStageClearTimeReward の標準設定

チャレンジ・サベージの全ステージに3段階のタイム目標を設定する。

| 段階 | upper_clear_time_ms | 報酬 | 備考 |
|-----|---------------------|------|------|
| 1（最速） | 140000 | FreeDiamond × 20 | 140秒以内 |
| 2（標準） | 200000 | FreeDiamond × 20 | 200秒以内 |
| 3（ゆっくり） | 300000 | FreeDiamond × 20 | 300秒以内 |

例外: サベージ最終ステージは難易度が高いため、240秒・300秒・400秒の設定が多い。

## 注意事項

- **イベントID（mst_event_id）**: `MstQuest` に設定する `mst_event_id` は、事前に `MstEvent` テーブルにレコードが存在していること。存在しない場合は別途 MstEvent の作成を依頼する。
- **インゲームID（mst_in_game_id）**: `MstStage.mst_in_game_id` は `MstInGame` テーブルのIDを参照する。このIDはエンジニア／インゲーム担当が作成するため、事前に確認・共有を受けること。フォーマット上は `{stage_id}` と同じIDを仮記入し、実際の値に差し替える。
- **アセットキー**: `MstStage.asset_key` および `MstStageEventSetting.background_asset_key` はアート担当に確認する。
- **日時はJST前提**: このフォーマットに記入する日時はすべてJST。Claudeへ渡す際も UTC 変換は不要（依頼文に明記済み）。
- **ステージ数の変更**: 202511010（mag イベント）は charaget が8ステージ構成だったが、202601010（jig イベント）以降は6ステージが標準になっている。特別な事情がなければ6ステージで設計する。
- **charaget01 の解放条件**: charaget01 は1ステージ目を NULL（最初から解放）にし、後続のチャレンジ・サベージは charaget01 の最終ステージを `prev_mst_stage_id` に設定して解放連鎖を作る。charaget02 は独立して公開日が来たら解放される（prev_mst_stage_id = NULL）。
- **MstQuestI18n の `flavor_text`**: 実データではほぼ空文字が設定されている。特別な説明文がない場合は空で問題ない。
- **MstStageI18n の言語**: `ja` のみ生成が基本。`en` / `zh-Hant` は別途多言語対応の指示がある場合に追加する。
- **MstStageClearTimeReward の対象**: キャラ獲得クエスト（charaget）には設定しない。チャレンジ・サベージのみに設定する。
- **背景アセットキーの差異**: charaget01 と charaget02 で背景が異なる場合がある（実データでは `mag_00002` と `glo_00011` のように別々のキーが使われることがある）。アート担当に確認すること。
