# イベントミッション 要件テキストフォーマット

> **用途**: プランナーがヒアリング結果を記入し、Claudeに渡すことでマスタデータCSVを一括生成するための要件テキスト。
>
> **生成されるCSV**:
> - `MstMissionEvent.csv`（ミッション定義レコード・N行、既存CSVへの追記）
> - `MstMissionEventDependency.csv`（依存関係レコード・M行、既存CSVへの追記）
> - `MstMissionReward.csv`（報酬レコード・N行、既存CSVへの追記）
> - `MstMissionEventI18n.csv`（多言語説明文レコード・N行、既存CSVへの追記）

## ミッションタイプ一覧（criterion_type）

実データから確認された全タイプ。プランナーは通常意識不要（Claudeが補完）。

| criterion_type | 説明 | criterion_value | 遷移先 (destination_scene) |
|---------------|------|----------------|--------------------------|
| `SpecificUnitGradeUpCount` | 特定キャラのグレードアップ回数 | キャラID（例: `chara_mag_00501`） | `UnitList` |
| `SpecificUnitLevel` | 特定キャラのレベル | キャラID（例: `chara_mag_00501`） | `UnitList` |
| `SpecificQuestClear` | 特定クエストのクリア | クエストID（例: `quest_event_mag1_charaget01`） | `Event` |
| `DefeatEnemyCount` | 敵の撃破数（通常敵） | 空（設定なし） | `Event` |
| `DefeatBossEnemyCount` | 強敵（ボス）の撃破数 | 空（設定なし） | `Event` |

> **criterion_count の意味**:
> - `SpecificUnitGradeUpCount` の場合: グレード段数（2, 3, 4, 5）
> - `SpecificUnitLevel` の場合: レベル数値（20, 30, 40, 50, 60, 70, 80）
> - `SpecificQuestClear` の場合: 常に 1（1回クリア）
> - `DefeatEnemyCount` / `DefeatBossEnemyCount` の場合: 撃破体数

## 報酬アイテム名の選択肢と対応ID

Claudeがこのテキストを解釈してIDに変換します。以下の表記を使ってください。

### チケット類

| 表記 | アイテムID | 備考 |
|------|-----------|------|
| ピックアップガシャチケット | `ticket_glo_00003` | ピックアップ対象を引ける |
| スペシャルガシャチケット | `ticket_glo_00002` | どのガシャでも使える |
| イベント専用チケット | `ticket_{イベントコード}_{連番}` | イベント固有（例: `ticket_kim_00001`） |

### 通貨・汎用アイテム

| 表記 | 報酬タイプ | 備考 |
|------|-----------|------|
| コイン | `Coin` | resource_id 不要 |
| プリズム | `FreeDiamond` | 無償ダイヤ。resource_id 不要 |

### キャラ専用メモリー（グレードアップ素材）

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| {キャラ名} のメモリー | `memory_chara_{キャラID}` | 例: `memory_chara_mag_00501` |

### キャラフラグメント（ピース）

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| {キャラ名} のピース | `piece_{キャラコード}_{番号}` | 例: `piece_mag_00501`、`piece_sur_00801` |

### カラーメモリー（Lv上限開放素材）

| 表記 | アイテムID | 属性 |
|------|-----------|------|
| カラーメモリー・グレー | `memory_glo_00001` | 無属性 |
| カラーメモリー・レッド | `memory_glo_00002` | 赤 |
| カラーメモリー・ブルー | `memory_glo_00003` | 青 |
| カラーメモリー・イエロー | `memory_glo_00004` | 黄 |
| カラーメモリー・グリーン | `memory_glo_00005` | 緑 |

### メモリーフラグメント（Lv上限開放素材・色共通）

| 表記 | アイテムID | レアリティ |
|------|-----------|-----------|
| メモリーフラグメント・初級 | `memoryfragment_glo_00001` | SR相当 |
| メモリーフラグメント・中級 | `memoryfragment_glo_00002` | SSR相当 |
| メモリーフラグメント・上級 | `memoryfragment_glo_00003` | UR相当 |

### エンブレム（称号）

| 表記 | アイテムID形式 | 備考 |
|------|-------------|------|
| イベントエンブレム | `emblem_event_{イベントコード}` | 例: `emblem_event_sur_00001`、`emblem_event_mag_00001` |

> **その他アイテム**: 上記にない場合はアイテムIDを直接記入するか、「アイテム名（item_xxx_xxxxx）× N個」と書く。

## 補足: テーブル間の関係

```
MstEvent（イベント定義）
  └─ mst_event_id
       ↓
MstMissionEvent（ミッション本体 / N行）
  ├─ id: {event_id}_{連番}
  ├─ mst_event_id: イベントID
  ├─ criterion_type: ミッション達成条件タイプ
  ├─ criterion_value: 達成対象（キャラID・クエストID・空）
  ├─ criterion_count: 達成必要数
  ├─ unlock_criterion_type: __NULL__（常時表示）固定
  ├─ mst_mission_reward_group_id: {イベントコード}_event_reward_{ゼロ埋め2桁}
  ├─ sort_order: 表示順連番
  └─ destination_scene: UnitList（育成系）/ Event（クエスト・撃破系）
       ↓（依存関係がある系列のみ）
MstMissionEventDependency（開放順制御 / M行）
  ├─ id: 連番整数（既存の続き）
  ├─ group_id: シリーズ先頭ミッションのID（例: event_mag_00001_1）
  ├─ mst_mission_event_id: シリーズ内の各ミッションID
  └─ unlock_order: 1から連番（先頭が1）
       ↓
MstMissionReward（報酬定義 / N行）
  ├─ id: mission_reward_{連番}
  ├─ group_id: {イベントコード}_event_reward_{ゼロ埋め2桁}
  ├─ resource_type: Coin / FreeDiamond / Item / Emblem
  ├─ resource_id: アイテムID（Coin/FreeDiamond は空）
  ├─ resource_amount: 数量
  ├─ sort_order: 1（固定）
  └─ 備考: 備考欄ラベル
       ↓
MstMissionEventI18n（多言語説明文 / N行）
  ├─ id: {ミッションID}_ja
  ├─ mst_mission_event_id: ミッションID
  ├─ language: ja（固定）
  └─ description: ミッション説明文（例: 重本 浩司 をグレード2まで強化しよう）
```

### 依存関係（MstMissionEventDependency）の設定ルール

育成ミッションは必ずシリーズ単位で依存関係を設定する。独立ミッション（クエストクリア系、単発撃破など）には設定しない。

| ミッション種別 | 依存関係設定 | group_id の命名 |
|-------------|------------|----------------|
| グレードアップ（グレード2〜5） | あり（順番開放） | シリーズ先頭ミッションのID |
| レベルアップ（Lv.20〜80） | あり（順番開放） | シリーズ先頭ミッションのID |
| クエストクリア（各クエスト） | なし | 設定不要 |
| 敵撃破数（全段階） | あり（順番開放） | 撃破系先頭ミッションのID |
| ボス敵撃破数（全段階） | あり（順番開放） | 撃破系先頭ミッションのID |

## 注意事項

- **unlock_criterion_type**: 常に文字列 `__NULL__` を設定する（NULL とは区別）
- **criterion_value の空**: `DefeatEnemyCount` / `DefeatBossEnemyCount` は criterion_value を空にする
- **reward_group_id の命名**: `{イベントコード}_event_reward_{ゼロ埋め2桁}` が標準（例: `mag_00001_event_reward_01`）
- **MstMissionEventDependency の id**: CSVでは連番整数（1, 2, 3...）。既存の最大値+1から採番すること
- **ミッション説明文（I18n）**: キャラ育成系はキャラ名フルネームを含める（例: `重本 浩司 をグレード2まで強化しよう`）
- **sort_order（MstMissionEvent）**: ミッションの表示順。育成→クエスト→撃破の順で採番するのが慣例
- **release_key**: すべてのテーブルで同一のリリースキーを設定する
- **event_category**: 通常イベントは NULL（空）。降臨バトル系のみ `AdventBattle` を設定する
- **group_key**: 通常は空（設定なし）
- **destination_scene**: `UnitList`（育成系ミッション）/ `Event`（クエスト・撃破系ミッション）の2値のみ使用する
