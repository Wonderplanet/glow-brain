# 降臨バトル 要件テキストフォーマット

> **用途**: プランナーがヒアリング結果を記入し、Claudeに渡すことでマスタデータCSVを一括生成するための要件テキスト。
>
> **生成されるCSV**:
> - `MstAdventBattle.csv`（降臨バトル基本定義・1行）
> - `MstAdventBattleI18n.csv`（表示名・1行）
> - `MstAdventBattleRank.csv`（ランク定義・16行固定）
> - `MstAdventBattleRewardGroup.csv`（報酬グループ定義・N行）
> - `MstAdventBattleReward.csv`（スコア報酬・N行）
> - `MstAdventBattleClearReward.csv`（クリア報酬・N行）

---

## テンプレート

```
# 降臨バトル 要件テキスト

## 基本情報

- 降臨バトルID: {quest_raid_{シリーズ略称}_{連番5桁} の形式で記入}
  例: quest_raid_abc1_00001
- イベントID（連動するイベントがある場合）: {mst_events.id。単独開催の場合は「なし」と記入}
  例: event_abc_00001 または なし
- インゲームID: {mst_in_games.id。通常は quest_raid の部分を raid に置き換えた値}
  例: raid_abc1_00001
- アセットキー: {バナー等のアセットキー}
  例: abc_00001
- リリースキー: {このリリースのリリースキーを記入}
  例: 202601010

## バトル設定

- バトルタイプ: {ScoreChallenge または Raid}
  ※ ScoreChallenge: 個人スコアを競う通常形式。Raid: 全プレイヤー合計スコアを競う大型レイド形式。
- 挑戦回数（1日）: {無料で挑戦できる回数。標準は 3}
- 広告視聴で追加される挑戦回数（1日）: {標準は 2}
- バトルクリア時の獲得EXP: {標準は 100}
- バトルクリア時の獲得コイン: {標準は 300}

## 開催期間

- 開始: YYYY-MM-DD HH:MM:SS
- 終了: YYYY-MM-DD HH:MM:SS

  ※ 開始時刻の慣例: 15:00:00（JSTの当日15時）
  ※ 終了時刻の慣例: 14:59:59（JST。1週間後の14:59:59が標準）

## 表示設定

- 降臨バトル表示名（日本語）: {ゲーム内に表示されるタイトル}
  例: まるで悪夢を見ているようだ
- ボス説明文（日本語）: {ボスの説明。通常は「ボスを倒して高スコア獲得!!」}
- 表示キャラID1（任意）: {トップ画面表示位置1のキャラまたは敵のID}
  例: enemy_abc_00001
- 表示キャラID2（任意）: {表示位置2。なければ「なし」}
- 表示キャラID3（任意）: {表示位置3。なければ「なし」}
- イベントボーナスグループID: {通常はインゲームIDと同じ値}
  例: raid_abc1_00001

## ランク区分（スコアしきい値）

全降臨バトルで16段階固定（Bronze×4 + Silver×4 + Gold×4 + Master×4）。
スコアしきい値のみ記入してください（標準値からの変更がある場合のみ）。

  ※ 特に変更なければ「標準」と記入（Claudeが下記標準値を使用）:
     Bronze1:    1,000 / Bronze2:    5,000 / Bronze3:   10,000 / Bronze4:   15,000
     Silver1:   30,000 / Silver2:   50,000 / Silver3:   75,000 / Silver4:  100,000
     Gold1:    150,000 / Gold2:    200,000 / Gold3:    250,000 / Gold4:    300,000
     Master1:  500,000 / Master2: 1,000,000 / Master3: 1,500,000 / Master4: 2,000,000

- ランクしきい値: {標準 または 各レベルのスコアを記入}

## スコア達成報酬（MaxScore）

プレイヤーが最高スコアで達成した場合に付与される報酬。段階ごとに設定する。

{スコアしきい値}: {報酬アイテム名} × {数量}
（スコアが低い→高い順に記入。スコアの奇数段はプリズム、偶数段はコインが標準パターン）

## ランク到達報酬（Rank）

各ランクレベルに到達した時に付与される報酬。rank_01〜rank_16（16段階固定）。

{ランクID}: {報酬1}: {アイテム名} × {数量} / {報酬2}: ...
（複数報酬はスラッシュ区切りで記入）

  ※ 特に変更なければ「標準」と記入（Claudeが下記標準パターンを使用）:
     rank_01〜04 (Bronze): プリズム10 + コイン1,000 + メモリーフラグメント・初級×1
     rank_05〜08 (Silver): プリズム20 + コイン2,000 + メモリーフラグメント・初級×1〜2
     rank_09〜12 (Gold):   プリズム30 + コイン3,000 + 初級×2 + 中級×1〜2
     rank_13〜15 (Master): プリズム40 + コイン4,000 + 初級×3 + 中級×2
     rank_16 (Master4):   プリズム40 + コイン4,000 + 初級×3 + 中級×2 + 上級×1

- ランク到達報酬: {標準 または 各ランクの報酬を記入}

## ランキング報酬（Ranking）

シーズン終了後の最終順位に応じて付与される報酬。

{順位（またはParticipation）}: {報酬アイテム名} × {数量} / ...
（Participationは参加者全員に付与）

  ※ 特に変更なければ「標準」と記入（Claudeが下記標準順位区分を使用）:
     1位 / 2位 / 3位 / 50位 / 300位 / 1000位 / 5000位 / 10000位 / 参加賞（Participation）

  ※ ランキング報酬でのエンブレムIDの命名規則: emblem_adventbattle_{シリーズ略称}_season{シーズン番号}_{00001〜}

【記入欄】

## クリア報酬

バトルをクリアした際に毎回または初回のみ付与される報酬。

【毎回クリア報酬（Random）】
{アイテム名} × {数量}（重み: {割合}）
（カラーメモリー5色を等確率で1種ランダム付与が標準）

【初回クリア報酬（FirstClear）】
{アイテム名} × {数量}
（初回クリア報酬がない場合は「なし」と記入）
```

## 報酬アイテム名の選択肢と対応ID

Claudeがこのテキストを解釈してIDに変換します。以下の表記を使ってください。

### チケット類

| 表記 | アイテムID | 備考 |
|------|-----------|------|
| ピックアップガシャチケット | `ticket_glo_00003` | ピックアップ対象を引ける |
| スペシャルガシャチケット | `ticket_glo_00002` | どのガシャでも使える |

### 通貨・汎用アイテム

| 表記 | resource_type / resource_id | 備考 |
|------|----------------------------|------|
| コイン | `Coin`（resource_id 不要） | |
| プリズム | `FreeDiamond` + `prism_glo_00001` | 無償ダイヤ。202603010以降は resource_id 空文字 |

> **注意**: プリズムの resource_id は古いリリース（〜202512010）では `prism_glo_00001`、202603010以降は空文字になっている。リリースキーに合わせてClaudeが判断する。

### メモリーフラグメント（Lv上限開放素材・色共通）

| 表記 | アイテムID | レアリティ |
|------|-----------|-----------|
| メモリーフラグメント・初級 | `memoryfragment_glo_00001` | SR相当 |
| メモリーフラグメント・中級 | `memoryfragment_glo_00002` | SSR相当 |
| メモリーフラグメント・上級 | `memoryfragment_glo_00003` | UR相当 |

### カラーメモリー（クリア報酬に使用）

| 表記 | アイテムID | 属性 |
|------|-----------|------|
| カラーメモリー・グレー | `memory_glo_00001` | 無属性 |
| カラーメモリー・レッド | `memory_glo_00002` | 赤 |
| カラーメモリー・ブルー | `memory_glo_00003` | 青 |
| カラーメモリー・イエロー | `memory_glo_00004` | 黄 |
| カラーメモリー・グリーン | `memory_glo_00005` | 緑 |

### エンブレム（ランキング報酬に使用）

| 表記 | 命名規則 |
|------|---------|
| エンブレム（1位）| `emblem_adventbattle_{略称}_season{N}_00001` |
| エンブレム（2位）| `emblem_adventbattle_{略称}_season{N}_00002` |
| エンブレム（3位）| `emblem_adventbattle_{略称}_season{N}_00003` |
| エンブレム（50位）| `emblem_adventbattle_{略称}_season{N}_00004` |
| エンブレム（300位）| `emblem_adventbattle_{略称}_season{N}_00005` |
| エンブレム（1000位）| `emblem_adventbattle_{略称}_season{N}_00006` |

> **その他アイテム**: 上記にない場合はアイテムIDを直接記入するか、「アイテム名（item_xxx_xxxxx）× N個」と書く。

## 補足: テーブル間の関係

```
MstAdventBattle（1行）
  ├─ id: quest_raid_{略称}_{連番}
  ├─ mst_event_id: 紐付くイベントID（単独開催の場合は空文字）
  ├─ mst_in_game_id: raid_{略称}_{連番}
  ├─ advent_battle_type: ScoreChallenge | Raid
  ├─ challengeable_count: 1日の無料挑戦回数（標準3）
  ├─ ad_challengeable_count: 広告視聴で追加される回数（標準2）
  ├─ exp: クリア時EXP（標準100）
  ├─ coin: クリア時コイン（標準300）
  ├─ start_at / end_at: 開催期間（JST）
  └─ event_bonus_group_id: raid_{略称}_{連番}
    ↓
MstAdventBattleI18n（1行）
  ├─ id: {advent_battle_id}_ja
  ├─ mst_advent_battle_id: advent_battle_id
  ├─ language: ja
  ├─ name: 降臨バトル表示名
  └─ boss_description: ボス説明文
    ↓
MstAdventBattleRank（16行固定）
  ├─ id: {advent_battle_id}_rank_{01〜16}
  ├─ mst_advent_battle_id: advent_battle_id
  ├─ rank_type: Bronze（01〜04）/ Silver（05〜08）/ Gold（09〜12）/ Master（13〜16）
  ├─ rank_level: タイプ内の番号（1〜4）
  └─ required_lower_score: そのランクに必要な最低スコア
    ↓
MstAdventBattleRewardGroup（N行: MaxScore + RaidTotalScore + Rank×16 + Ranking）
  ├─ id: {略称}_reward_group_{バトル連番}_{グループ連番}
  ├─ mst_advent_battle_id: advent_battle_id
  ├─ reward_category: MaxScore | RaidTotalScore | Rank | Ranking
  └─ condition_value:
       MaxScore → スコア数値（文字列）
       RaidTotalScore → 累積スコア数値（文字列）
       Rank → {advent_battle_id}_rank_{01〜16}
       Ranking → 順位数値（文字列）または "Participation"
    ↓
MstAdventBattleReward（N行）
  ├─ id: {reward_group_id}_{内部連番（1始まり）}
  ├─ mst_advent_battle_reward_group_id: reward_group_id
  ├─ resource_type: Coin | FreeDiamond | Item | Emblem
  ├─ resource_id: アイテムID（Coinは空文字）
  └─ resource_amount: 数量

MstAdventBattleClearReward（N行）
  ├─ id: {advent_battle_id}_{連番（1始まり）}
  ├─ mst_advent_battle_id: advent_battle_id
  ├─ reward_category: Always | FirstClear | Random
  ├─ resource_type: Exp | Coin | FreeDiamond | Item | Emblem | Unit
  ├─ resource_id: アイテムID（Exp・Coinは空文字またはNULL）
  ├─ resource_amount: 数量
  ├─ percentage: 重み（Random時のみ有効。Always/FirstClearは1を設定）
  └─ sort_order: 表示順（1始まり）
```

## 注意事項

- **バトルタイプとRaidTotalScoreの関係**: ScoreChallengeの場合は個人累積スコア（万〜十数万程度）、Raidの場合は全プレイヤー合計スコア（億〜数十億規模）でしきい値が大きく異なる。
- **ランク16段階は固定**: MstAdventBattleRankは全バトルでBronze×4 + Silver×4 + Gold×4 + Master×4の計16行が必須。段階数の変更は不可。
- **Rankingグループのcondition_value**: 1位・2位・3位は数値（`"1"`, `"2"`, `"3"`）。それ以外の順位は「その順位まで（累積）」ではなく「その順位丁度」を表す。参加者全員は `"Participation"`。
- **報酬グループIDと報酬IDの採番規則**: 連番の末尾サフィックスが `_01` 〜 `_53` 等の場合、53番台はMaxScoreの追加段階（後から追加されたもの）として扱われる。既存データとの整合のためClaude任せで採番する。
- **エンブレムはランキング1〜1000位のみ**: 5000位以下・参加賞にはエンブレムが付与されないのが標準パターン。
- **イベントなし（単独開催）のパターン**: `mst_event_id` が空文字の場合も問題ない（quest_raid_kai_00002が該当）。
- **display_mst_unit_id**: キャラIDまたは敵IDを設定する。設定なしの場合はNULL（空文字）。
- **時刻はすべてJST前提**: UTC変換不要。start_at / end_at にJSTの時刻をそのまま記入する。
- **クリア報酬のデフォルト構成**: カラーメモリー5色を各重み20（計100）で等確率ランダム付与が全バトル共通の標準構成。
