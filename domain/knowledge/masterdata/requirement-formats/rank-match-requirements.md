# ランクマッチ（PvP）要件テキストフォーマット

> **用途**: プランナーがリリース計画を記入し、Claudeに渡すことでランクマッチ（PvP）のマスタデータCSVを一括生成するための要件テキスト。
>
> **生成されるCSV**:
> - `MstPvp.csv`（シーズン定義・1リリースにつき2行が標準）
> - `MstPvpI18n.csv`（説明文・MstPvp と同数行）
>
> **注意**: `MstPvpRewardGroup.csv` / `MstPvpReward.csv` は初回リリース（202512010）で設定済みの `default_pvp` 紐付けデータが継続利用されるため、**通常は追加不要**。報酬内容を変更する場合のみ別途依頼する。

---

```

  例: ○○コラボ

1シーズン = 1週間（月曜00:00〜日曜23:59 JST）。1リリースに2シーズン分を登録するのが標準。

シーズン1:
- PvP ID: {西暦4桁}{ISO週番号2桁ゼロ埋め}
  例: 2026016（2026年第16週 = 2026-04-13月〜）

  例: pvp_xxx_01

  例: 2026017（2026年第17週 = 2026-04-20月〜）
- 使用インゲームID: {mst_in_games.id に登録済みのID}
  例: pvp_xxx_02
- 初期バトルポイント: {200 または 1000}
- ステージ段数: {3段 または 4段}
- コマ効果: {コマ効果なし / 攻撃UPコマ / 攻撃DOWNコマ / ダメージコマ / 毒コマ / 突風コマ のいずれか}
- 特別ルール（任意）: {あれば記入}

（シーズン数に応じて増減してください。1リリースに1シーズンのみの場合はシーズン2を削除）
```

## コマ効果・特別ルールの説明文変換対応表

プランナーが「コマ効果」と「特別ルール」を記入すると、Claudeが以下の対応表を参照して `MstPvpI18n.description` の説明文を自動生成します。

### コマ効果

| 記入値 | コマ効果情報テキスト |
|--------|---------------------|
| コマ効果なし | 【コマ効果情報】\nコマ効果は登場しないぞ! |
| 攻撃UPコマ | 【コマ効果情報】\n攻撃UPコマが登場するぞ!\n特性で攻撃UPコマ効果UPを持っているキャラを編成しよう! |
| 攻撃DOWNコマ | 【コマ効果情報】\n攻撃DOWNコマが登場するぞ!\n特性で攻撃DOWNコマ無効化を持っているキャラを編成しよう! |
| ダメージコマ | 【コマ効果情報】\nダメージコマが登場するぞ!\n特性でダメージコマ無効化を持っているキャラを編成しよう! |
| 毒コマ | 【コマ効果情報】\n毒コマが登場するぞ!\n特性で毒ダメージ軽減を持っているキャラを編成しよう! |
| 突風コマ | 【コマ効果情報】\n突風コマが登場するぞ!\n特性で突風コマ無効化を持っているキャラを編成しよう! |

### 特別ルール

| 記入値 | 特別ルール情報テキスト |
|--------|----------------------|
| （なし） | セクション自体を省略 |
| リーダーP 1000開始 | 【特別ルール情報】\nリーダーPが1,000溜まった状態でバトルが開始されるぞ! |
| リーダーP 1000開始 + 体力2倍UP | 【特別ルール情報】\nリーダーPが1,000溜まった状態でバトルが開始されるぞ!\nさらに、全キャラの体力のステータス値が2倍UP! |
| リーダーP 1000開始 + 体力3倍UP | 【特別ルール情報】\nリーダーPが1,000溜まった状態でバトルが開始されるぞ!\nさらに、全キャラの体力のステータス値が3倍UP! |

### ステージ段数と基本情報テキスト

| ステージ段数 | 基本情報テキスト |
|------------|----------------|
| 3段 | 【基本情報】\n3段のステージで戦うぞ!\n相手との距離があるので、リーダーPのバランスを考えてパーティを編成しよう! |
| 4段 | 【基本情報】\n4段のステージで戦うぞ!\n相手との距離があるので、リーダーPのバランスを考えてパーティを編成しよう! |

> **完成する description の構成**: `【基本情報】` + `\n\n` + `【コマ効果情報】` + （あれば `\n\n` + `【特別ルール情報】`）

## 補足: テーブル間の関係

```
MstPvp（1シーズン = 1行）
  ├─ id: {年4桁}{週番号2桁}  例: 2026010
  ├─ release_key: リリースキー
  ├─ ranking_min_pvp_rank_class: Bronze（固定）
  ├─ max_daily_challenge_count: 10（固定）
  ├─ max_daily_item_challenge_count: 10（固定）
  ├─ item_challenge_cost_amount: 1（固定）
  ├─ mst_in_game_id: pvp_xxx_01 など（インゲーム設定）
  └─ initial_battle_point: 200 または 1000
    ↓ id が参照される
MstPvpI18n（MstPvpと1:1対応）
  ├─ id: MstPvp.id と同じ値
  ├─ mst_pvp_id: MstPvp.id と同じ値
  ├─ language: ja（固定）
  ├─ name: NULL（空のまま）
  └─ description: 基本情報 + コマ効果情報 + 特別ルール情報の連結テキスト

MstPvpRewardGroup（default_pvp に紐付く・通常追加不要）
  ├─ mst_pvp_id: default_pvp（固定）
  ├─ reward_category: Ranking / RankClass / TotalScore
  └─ condition_value: 順位閾値 / ランク名 / スコア値
    ↓
MstPvpReward（報酬内容・通常追加不要）
  ├─ mst_pvp_reward_group_id: MstPvpRewardGroup.id
  ├─ resource_type: Coin / FreeDiamond / Item / Emblem
  ├─ resource_id: アイテムIDまたはNULL
  └─ resource_amount: 数量
```

### 既存の固定報酬グループ（default_pvp）

| グループ種別 | 設定済みグループ数 | 内容概要 |
|------------|-----------------|---------|
| ランキング報酬（Ranking） | 8グループ（1位・2位・3位・50位以内・100位以内・1000位以内・5000位以内・10000位以内） | スペシャルガシャチケット + コイン（上位ほど多い） |
| ランク到達報酬（RankClass） | 16グループ（Bronze1〜4 / Silver1〜4 / Gold1〜4 / Platinum1〜4） | プリズム（到達ランクが高いほど多い） |
| 累計スコア報酬（TotalScore） | 15グループ（10〜2000スコア） | コイン・メモリーフラグメント・プリズム・スペシャルガシャチケット |

## 注意事項

- **IDはISO週番号ベース**: ISO週番号は月曜始まり。`{年4桁}{ISO週番号2桁ゼロ埋め}` で採番する。例: 2026年第16週 = `2026016`
- **1リリースに2シーズンが標準**: 通常は2週分（2行）をセットで登録する。1週のみのリリースも可
- **同一インゲームIDの再使用可**: 同じ `mst_in_game_id` を異なるシーズンで使用しても構わない（例: pvp_Xmas_01 を2週連続で使用）
- **MstPvpRewardGroup / MstPvpReward は原則変更不要**: 報酬内容はシーズンをまたいで `default_pvp` の設定が引き継がれる。変更する場合はエンジニアへ確認のうえ別途CSVを作成する
- **initial_battle_point の選択**: 特殊ルールなし（コマ効果のみ）の場合は `200`、特殊ルール（リーダーP 1000開始など）がある場合は `1000` が標準
- **MstInGame の事前作成が必要**: `mst_in_game_id` に指定するIDは、インゲームマスタデータ（MstInGame等）として事前に別リリースで登録されている必要がある。IDが未登録の場合は別途 `masterdata-ingame-creator` スキルで作成する
- **時刻はすべてJST前提**: UTC変換不要。サーバー側でJSTとして処理される
