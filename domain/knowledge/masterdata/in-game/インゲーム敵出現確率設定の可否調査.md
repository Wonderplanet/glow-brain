# インゲーム 敵出現確率設定の可否調査

**調査日**: 2026-03-01
**対象**: マスタデータ（MstAutoPlayerSequence）および glow-client のインゲーム実装

---

## 結論

**現状では「確率指定」による敵出現設定は不可能。**

敵の出現はすべてマスタデータで固定されたトリガー条件（時間・ゲームイベント）に基づく決定論的な仕様であり、確率によって出し分ける仕組みは存在しない。

---

## 敵出現を制御するマスタデータ

### MstAutoPlayerSequence（主制御テーブル）

インゲームにおける敵出現シーケンスを定義するテーブル。

**全カラム一覧**:

| カラム名 | 説明 |
|---|---|
| `id` | シーケンスID |
| `sequence_set_id` | シーケンスセットID |
| `sequence_group_id` | シーケンスグループID |
| `sequence_element_id` | シーケンス要素ID |
| `priority_sequence_element_id` | 優先シーケンス要素ID |
| `condition_type` | 出現トリガーの条件種別 |
| `condition_value` | 条件値（例：経過時間ミリ秒） |
| `deactivation_condition_type` | 無効化条件種別 |
| `deactivation_condition_value` | 無効化条件値 |
| `action_type` | アクション種別（SummonEnemy 等） |
| `action_value` | 出現させる敵ID |
| `action_value2` | アクション補助値 |
| `summon_count` | 敵の出現数 |
| `summon_interval` | 出現間隔（ミリ秒） |
| `summon_position` | 出現位置 |
| `summon_animation_type` | 出現アニメーション種別 |
| `aura_type` | オーラ種別 |
| `death_type` | 死亡種別 |
| `last_boss_trigger` | ラストボストリガーフラグ |
| `enemy_hp_coef` | 敵HPの倍率 |
| `enemy_attack_coef` | 敵攻撃力の倍率 |
| `enemy_speed_coef` | 敵移動速度の倍率 |
| `move_start_condition_type` | 移動開始条件種別 |
| `move_start_condition_value` | 移動開始条件値 |
| `move_stop_condition_type` | 移動停止条件種別 |
| `move_stop_condition_value` | 移動停止条件値 |
| `move_restart_condition_type` | 移動再開条件種別 |
| `move_restart_condition_value` | 移動再開条件値 |
| `move_loop_count` | 移動ループ回数 |
| `is_summon_unit_outpost_damage_invalidation` | アウトポストダメージ無効フラグ |
| `override_drop_battle_point` | ドロップバトルポイント上書き値 |
| `defeated_score` | 撃破スコア |
| `release_key` | リリースキー |

**`probability`・`weight`・`rate` 等の確率関連カラムは存在しない。**

### 関連テーブル

| テーブル | 役割 |
|---|---|
| `MstEnemyStageParameter` | 敵ごとのHP・攻撃力・移動速度等のパラメータ |
| `MstInGame` | ステージ全体の設定（シーケンスIDの参照先） |

## 敵出現の制御方式（現状）

敵の出現は `condition_type` / `condition_value` によるトリガーベースで制御される。

### 主な condition_type

| condition_type | 説明 |
|---|---|
| `ElapsedTime` | バトル開始からの経過時間（ミリ秒）で出現 |
| `FriendUnitDead` | 味方ユニットが撃破された時に出現 |
| `OutpostDamage` | アウトポストが被ダメージを受けた時に出現 |

すべて**確定的なトリガー**であり、ランダム性はない。

## クライアント実装の確認

### 敵出現アクション

**`AutoPlayerSummonEnemyAction.cs`**
パス: `projects/glow-client/Assets/GLOW/Scripts/Runtime/Scenes/InGame/Domain/Battle/AutoPlayer/`

シンプルな「敵ID + 出現モデル」のみを保持するレコード型。確率計算のロジックは存在しない。

### ランダム処理の有無

`DeckAutoPlayerSummonSelector.cs` に `Random.Range()` が存在するが、これは**プレイヤー側のデッキユニット選択**のためのものであり、敵出現とは無関係。

敵出現処理全体を通じて、確率計算のロジックはクライアント実装にも存在しない。

---

## 確率的な敵出現を実現するために必要な対応

マスタデータの変更だけでは対応できず、以下のセットで改修が必要になる。

1. **DBスキーマ変更**: `mst_auto_player_sequences` に `probability`（または `weight`）カラムを追加
2. **サーバー実装更新**: スキーマ変更に合わせたマスタデータ配信の更新
3. **クライアント実装更新**: `AutoPlayerUpdateProcess` 等で確率判定ロジックを追加
4. **マスタデータ作成**: 各シーケンスに確率値を設定
