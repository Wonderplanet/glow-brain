## 概要

インゲームで出現するエネミーのHP・ATK設定の基準値をまとめたシート。
マスタデータ作成時（`MstEnemyStageParameter`・`MstInGame`）の仮パラメータ投入に使用する。

---

## ①大まかなステージ作成の流れ

### ステージ設計に考慮すべきこと

1. **活躍させるキャラの性能・属性に合わせる** ※いわゆる運営における必要条件
2. **プレイヤーがギミックの解法の目安をつけられる** ※敵が使う特殊効果・コマ効果・ファントムの出現構成などの情報
3. **単調でなく、ファントムの出現に緩急や流れがある** ※キャラを召喚するタイミング・召喚Pの温存や勝ちを決めるタイミング・負け筋があるなど

---

## ②ステージの調整方法

- このゲームの調整は**ファントムのパラメータとSequenceのファントム出現タイミング、ドロップ召喚ポイント**のみでほぼ完結する
  - ※強力な状態異常などを使う場合は要調整
- Sequenceでファントムの出現タイミングや配置を決定してから、パラメータを調整する
  - ※両方同時に調整すると設計意図からずれたものが出来上がるため
- ドロップ召喚Pはデバッグ機能で召喚コスト0でお試しプレイしたり、多めに配布してから試して下げていって調整するとやりやすい

---

## ③エネミーステータスシートの使い方

### ステータスシートの見方

- HP・ATKの数値は**推奨PTで出撃した際に攻略可能な数値のおおよそ中央値**をとったもの
- Sequenceを設定した際に、ファントムのパラメータを仮で入力しておくための数値として最適
- ※サポートに関してはバトルの設計上、先に倒れてほしい・最後まで残ってほしいなど役割によって大きく変動するため、表の限りではない

**HP**: おおよそ中盤に召喚されるプレイキャラと対峙してちょうど良いHP。
中ボスの役割を持たせたりボスの取り巻きを想定する場合は中央値から大きく外れた数値になる。

**ATK**: おおよそ中盤に召喚されるプレイキャラと対峙してちょうど良いATK。
攻撃間隔50フレームのファントムが与えるダメージとして適量になっているため、攻撃頻度によっては中央値から大きく外れた数値になる。

> ※ファントムの過疎具合・密集具合によって数値は変わるので、Sequenceで単体稼働させる場合は数値が低めに感じる。集団で稼働させる場合は高く感じるといったことは発生する。

### クリアタイムの想定からボスのHPを調整する

- クリアタイムを **3分程度** とする場合 → Sequenceフレーム換算で **約9,000フレーム**
- ボスが1体の構成でエネミーステータスシートの数値を利用する場合：
  - 大体 **5,000〜6,000フレーム** でボスを登場させると、ボス討伐に **2,000〜3,000フレーム** 消費し、おおよそ **2分45秒〜3分超** のバトルになる
- ボスの数・取り巻きの数、活躍させたいプレイヤーキャラ（特攻や有利特性持ちなど）がいる場合はその都度調整する

## マスタデータとの対応関係

### MstEnemyStageParameter（エネミーパラメータ設定）

エネミーのHP・ATKを設定するメインテーブル。

| カラム | 説明 | ステータスシートとの対応 |
|--------|------|------------------------|
| `hp` | HP | ロール別の雑魚HP / ボスHP |
| `attack_power` | 攻撃力 | ロール別の雑魚ATK / ボスATK |
| `character_unit_kind` | 敵カテゴリ | `Normal`（雑魚）/ `Boss`（ボス）/ `AdventBattleBoss`（降臨ボス） |
| `role_type` | ロールタイプ | `Attack` / `Balance` / `Defense` / `Support` / `Technical` など |
| `color` | 影色 | `Colorless` / `Red` / `Blue` / `Yellow` / `Green` |
| `attack_combo_cycle` | 攻撃サイクル | ATK基準値は攻撃間隔50フレーム相当（cycle=1が最頻値） |
| `move_speed` | 移動速度 | 一般的な値: 25〜65（中央値: 35〜45） |
| `well_distance` | 索敵距離 | — |
| `drop_battle_point` | 被撃破時のリーダーP獲得量 | — |

**実データの分布（MstEnemyStageParameter.csv より / 2026-03-01 時点）**:
- 総レコード数: 1,061件
- `character_unit_kind` 分布: Normal=616, Boss=437, AdventBattleBoss=8
- `role_type` 分布: Attack=589, Defense=225, Technical=190, Support=57
- HPの中央値: 10,000（Normal）/ 10,000（Boss）
- ATKの中央値: 300（Normal Attack/Technical）/ 100（Normal Defense）

### MstInGame（インゲーム設定）

倍率係数でエネミー全体のHP・ATK・Speedをまとめて調整するテーブル。

| カラム | 説明 | 用途 |
|--------|------|------|
| `normal_enemy_hp_coef` | 通常敵HP倍率 | ステージ全体の雑魚HPを一括調整（通常は1.0） |
| `normal_enemy_attack_coef` | 通常敵ATK倍率 | ステージ全体の雑魚ATKを一括調整（通常は1.0） |
| `normal_enemy_speed_coef` | 通常敵Speed倍率 | ステージ全体の雑魚Speedを一括調整 |
| `boss_enemy_hp_coef` | ボスHP倍率 | ボスHPを一括調整（通常は1.0） |
| `boss_enemy_attack_coef` | ボスATK倍率 | ボスATKを一括調整（通常は1.0） |
| `boss_enemy_speed_coef` | ボスSpeed倍率 | ボスSpeedを一括調整 |

> 実データでは `normal_enemy_hp_coef=1.0` が約91%（509/560件）。
> 2.0, 2.5, 4.0 などの倍率を使って既存エネミーをハードな難易度に流用しているケースがある。

## マスタデータ設定時の注意点

1. **ステータスシートの値は「中央値」であり、調整の出発点**
   ボスの取り巻きや特殊役割のエネミーはシートの値から大きく外れることがある

2. **ATK値はバフ考慮外**
   作品バフや特攻補正がある場合は実効ダメージが変わるため、バフがある場合は低めに設定

3. **ロールタイプ別のHP傾向**
   - Def（ディフェンス）: HPが最も高い（Atk/Techの約5倍）
   - Sup（サポート）: HPはDefの約44%、ATKはDefの約1.5〜1.7倍
   - Atk/Tech: HPが最も低いが、ATKはDefの約3倍

4. **ゲートHPは各ステージ毎に個別調整が必要**
   シートの値はあくまで目安。プレイ体験に応じて増減させる

5. **倍率係数（MstInGame）を使ったバリエーション展開**
   同じエネミーキャラクターを複数難易度のステージに流用する場合、
   `MstEnemyStageParameter` の値をベースに `MstInGame` の倍率で調整する手法がある
