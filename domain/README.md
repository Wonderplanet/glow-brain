# domain フォルダ構成ガイド

この `domain/` フォルダは、  
**ゲーム開発における全職種・全タスクを対象とした AI 活用基盤**として設計されています。

本フォルダ配下には、

- AIに「何をやらせるか」
- AIが「何を事実として信じるか」
- AIが「どの視点で考えるか」
- AIが「どう実行されるか」

を明確に分離して管理するための構造を採用しています。

---

## 全体構成

```
domain/
  raw-data/
  knowledge/
  tasks/
  roles/
  tools/
  shared/
```

---

## 1. raw-data/

### 役割
未加工・未解釈の生データ置き場。

### 入れるもの
- ログ
- 外部システムからの取得データ
- 解析前の一次情報

---

## 2. knowledge/

### 役割
AIが判断材料として信じる知識。

### レイヤー
```
knowledge/
  derived/
  curated/
```

- derived: 分析・解析から得られた知見
- curated: 人が承認したルール・仕様

---

## 3. tasks/

### 役割
AIに何をやらせるかを定義する。

### 入れるもの
- プロンプト
- タスク説明
- 入出力例

---

## 4. roles/

### 役割
職種ごとの視点・判断軸をAIに与える。

---

## 5. tools/

### 役割
実行可能なAIツール・エージェント。

---

## 6. shared/

### 役割
複数領域で共通利用される資産。

---

## データと知識の流れ

```
raw-data
  ↓
knowledge/derived
  ↓
knowledge/curated
```

---

## 判断基準まとめ

- 判断材料 → knowledge
- 生データ → raw-data
- やらせ方 → tasks
- 視点 → roles
- 実行物 → tools
