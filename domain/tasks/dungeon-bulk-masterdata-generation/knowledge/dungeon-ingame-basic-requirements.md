## 1. フロア構成の基本構造（仕様書 p9）

- ゲーム画面に表示される段数 = **1フロア**
- 召喚されたキャラが次のフロアに進軍できたタイミングで進行フロア数が増える
- フロアの種類: **通常フロア** と **ボスフロア** の2種類
- フロアの構成はゲーム開始時に毎回変化する（ランダム生成）

### 難易度スケール

- 一定の深度を超えると、次のフロアから敵ステータス（HP/攻撃）に上昇係数がかかる
- フロアを進行すればするほど敵が強くなる

### ボスフロアのクリア判定

- ボスフロアのゲートを破壊 → クリア判定 → ボーナスカード効果を獲得（即時反映）
- クリア判定後、ボスフロアの敵ゲートHPを0にする
- 中断復帰データ保存後、次の通常フロアへ移動
- タイムカウントは静止し、引継ぎ要素は以下の通り:

| 引き継ぐもの | リセットされるもの |
|------------|-----------------|
| ローグライク効果 | 敵・味方から付与された状態効果 |
| 編成中キャラの残HP | — |

## 2. ブロック生成ロジック（仕様書 p9）

### ブロック定義

| ブロック種別 | 略称 | フロア数（行数） | MstKomaLine行数 | MstEnemyOutpost HP |
|------------|------|--------------|---------------|--------------------|
| 通常ブロック（Normal Block） | N-Block | ×3 | **3行**（固定） | **100**（固定） |
| レアブロック（Rare Block） | R-Block | ×3 | **3行**（固定） | 100（固定・推定） |
| ボスブロック（Boss Block） | B-Block | ×1 | **1行**（固定） | **1,000**（固定） |

### 生成パターン（抽選、通常ブロックの重複あり）

1. ボスブロック（B）を抽選
2. レアブロック（R）の出現有無を抽選
3. レアブロック出現時 → 抽選終了
4. レアブロックなしの場合 → 通常ブロックのパターンを以下3種から抽選:
   - `N → B`
   - `N → N → B`
   - `N → N → N → B`
5. 登場するブロックを抽選

## 3. バトルシステム（仕様書 p10）

### 3-1. 通常ブロック（Normal Block）

| 項目 | 内容 |
|------|------|
| ゲーム性 | アクションRPG（1体操作のみ） |
| 出撃可能数 | **1体のみ** |
| 召喚操作 | リーダーP消費でキャラ交代召喚。スペシャルキャラはリーダーP消費→スキル発動後消える |
| 基本敵シーケンス | フロア毎に配置された敵が、味方キャラが該当フロアに進行した際に動き出し、射程範囲内で会頭した場合に攻撃開始 |
| 動作特性 | **受動的構成**（味方の進行に反応して起動） |

**→ マスタデータ設計への示唆**:
- シーケンスは `ElapsedTime` による時間差配置が基本
- グループ切り替え（SwitchSequenceGroup）・ギミックオブジェクトは不要
- シーケンス行数の目安: **3〜8行**

---

### 3-2. ボスブロック（Boss Block）

| 項目 | 内容 |
|------|------|
| ゲーム性 | タワーオブフェンス（総力戦）= 既存インゲームルール |
| 出撃可能数 | **生存している全キャラ召喚可能** |
| 召喚操作 | 既存インゲームルールと同様 |
| 基本敵シーケンス | 敵ゲート前に召喚され、進軍してくる（**能動的構成**） |
| ボスのHP | インゲームで通常登場するよりは **低い** |
| ボスの攻撃力 | インゲームで通常登場するよりは **高い** |
| 敵ゲート | **ボスが撃破されるまでダメージ無効** |

**→ マスタデータ設計への示唆**:
- ボスは `InitialSummon` でゲート前に配置し、N秒おきに進軍
- ボスのHPは通常登場スペックより低め、攻撃力は高め設定
- 護衛雑魚は `ElapsedTime` で時間差出現
- シーケンス行数の目安: **2〜4行**

---

### 3-3. レアブロック（Rare Block）

| 項目 | 内容 |
|------|------|
| ゲーム性 | アクションRPG（1体操作のみ） |
| 出撃可能数 | 1体のみ |
| 登場する敵 | **レアファントム** |
| 特記事項 | 敵のシーケンスで「ゼロリセット（プラス・マイナス効果解除）」は行わない |

## 4. リレー・召喚・状態引継ぎ仕様（仕様書 p10）

### 再召喚（リレー）のルール

| 項目 | 内容 |
|------|------|
| タイミング | 前任者の戦闘不能時 / リーダーP消費による交代時 |
| 召喚位置（通常ブロック） | 前任者が到達したフロアの右端に召喚サークルを表示し、そこから召喚 |
| 召喚位置（ボスフロア） | フロア右端に固定で召喚サインを表示 |

### 状態異常・効果の保持/消滅ルール

| 効果の種類 | 生存時 | 死亡時 |
|----------|--------|--------|
| ルーレット効果（JUMBLE SPIN / カード選択） | デッキ戻り・ステージ跨ぎでも**保持** | 全て解除（蘇生後も再付与なし） |
| バトル効果（スキルバフ・敵デバフ等） | 交代または盤面離脱で**消滅** | — |

### HPの引継ぎ

- 盤面を離れた時点のHPを維持する

## 5. マスタデータ設計まとめ

### MstInGame / MstEnemyOutpost

| ブロック種別 | `mst_enemy_outpost.hp` | `boss_mst_enemy_stage_parameter_id` |
|------------|----------------------|--------------------------------------|
| dungeon_normal | **100**（固定） | 空（ボスなし） |
| dungeon_boss | **1,000**（固定） | 設定あり（URキャラ等） |

### MstKomaLine

| ブロック種別 | 行数 |
|------------|------|
| dungeon_normal | **3行**（固定） |
| dungeon_boss | **1行**（固定） |

### MstAutoPlayerSequence

| ブロック種別 | シーケンス行数目安 | 基本構成 |
|------------|-----------------|---------|
| dungeon_normal | **3〜8行** | `ElapsedTime` で雑魚を時間差配置。グループ切り替えなし |
| dungeon_boss | **2〜4行** | ボス1体 + 時間差雑魚1〜2パターン |

### インゲームID命名規則

```
dungeon_{シリーズID}_{ブロック種別}_{連番5桁}

例:
  dungeon_spy_normal_00001   ← 通常ブロック
  dungeon_spy_boss_00001     ← ボスブロック
```
