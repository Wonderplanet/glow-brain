# ネクストアクション

## 次にやること

1. **作品リストの整理** - 対象となるdungeon作品と設計テキストをまとめる
   - どの作品を対象にするか確定する
   - 各作品の設計思想（敵構成・ステージ種別・難易度）を用意する

2. **1作品分のプロトタイプ** - まず1作品分を `masterdata-ingame-creator` で生成・確認する
   - 生成 → 検証 → XLSX出力の一連の流れを確認

3. **一括実行の仕組みを検討** - 複数作品分を連続実行する方法を決める
   - 作品ごとに順番に実行するワークフロー設計
   - outputs/ フォルダへの整理方法

4. **全作品分を一括生成** - 確立したワークフローで全作品分を生成

5. **XLSX提出** - 全作品分のXLSXをまとめて提出

## 長期的なアクション

- バッチ実行のワークフローをスキル化・汎用化する
- 他のコンテンツタイプにも同様の一括生成を適用する

## 注意事項

- 各作品のID採番は `masterdata-id-numbering` スキルで確認すること
- 生成後は必ず `masterdata-ingame-verifier` で検証すること
- XLSX出力は `masterdata-csv-to-xlsx` スキルを使うこと
