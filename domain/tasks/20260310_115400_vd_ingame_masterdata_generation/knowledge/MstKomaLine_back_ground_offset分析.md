# MstKomaLine `koma<N>_back_ground_offset` 分析

調査日: 2026-03-11
調査対象: `projects/glow-masterdata/MstKomaLine.csv`（VD以外の既存全コンテンツ）

---

## 結論サマリ

| 質問 | 答え |
|------|------|
| 値は決められるか？ | **コマアセットキーが確定すれば、過去実績から仮値を決められる** |
| 決め方は？ | 同じコマアセットキーで最も多く使われている値を採用する |
| 値の意味は？ | コマ背景テクスチャの**横方向表示位置**（-1.0=右端寄り、0.0=中央、1.0=左端寄り） |
| 絶対に正確には決まらないのか？ | 最適値はアセット画像の被写体位置を実際に見ないと決まらない |
| コマアセットキーがまだ空の場合は？ | **`0.0`（中央表示、空欄nullと同等）を仮置きで使う** |

---

## 値の仕組み（クライアント実装より）

`KomaBackgroundComponent.cs` 内の計算：

```csharp
var sizeDiff = texture.width - komaSetWidth;   // テクスチャ幅 - コマセットUI幅
var offset = sizeDiff * 0.5f * back_ground_offset.Value;
center.x -= offset / texture.width;            // UV座標のX中心をずらす
```

| 値 | 意味 | 効果 |
|----|------|------|
| `-1.0` | 右端方向へオフセット | 背景の右寄り部分を表示 |
| `0.0` | オフセットなし（null時と同じ） | 背景の中央を表示 |
| `1.0` | 左端方向へオフセット | 背景の左寄り部分を表示 |
| `null`（空欄） | `KomaBackgroundOffset.Empty = 0f` | 背景の中央を表示 |

## 注意事項

### コマアセットキーが確定してから設定するのがベスト
`back_ground_offset` はコマ背景テクスチャ画像の「どの横位置を見せるか」を制御する値。
アセット画像の内容（キャラクターや被写体がどこにいるか）によって最適値が変わる。

### 0.0（空欄）でも動作する
null/空欄の場合は `KomaBackgroundOffset.Empty = 0f` として動作し、背景中央を表示する。
見た目の微調整が不要なら空欄のままでも問題ない。

### 同一アセットキーでも複数の値が使われている理由
同じ作品の同じコマアセットを使っても、コマのwidthや画面上の配置位置によって
「どの部分を見せたいか」が異なるため。仮置きであれば最頻値を使えばほぼ問題ない。

### chi / hut / tak は既存データなし
MstKomaLine.csvにこれら3作品のコマアセットキーは一切存在しない（2026-03-11時点）。
VD作成時はアセット担当者に確認するか、`0.0` で仮置きする。
