## D. 気づき・注意点

### D-1. バレンタインガチャ・キャラパックが 202602015 に含まれていない

`@_バレンタインガシャ_設計書.csv`（`Pickup_Valentine_001`、開催期間: 2/10〜3/2）と `@_2月追加の課金施策_設計書.csv`（バレンタインキャラパック各種）が仕様書フォルダ内に存在するが、**202602015 の OprGacha には `Pickup_you_001`（いいじゃん祭ガチャ）の1件しか含まれていない**。

バレンタインガチャ・バレンタインキャラパックは別のリリースキー（例: `202602010` 等）で管理されており、仕様書フォルダが同一施策フォルダで共有されているため混在している。

### D-2. プロンプトに記載のなかったシートが3枚存在

調査プロンプトの初期シート一覧に記載されていなかったが、実際には以下が存在した：
- `03_降臨バトル.csv` → 降臨バトル設計書（**マスタデータ生成あり**。`MstAdventBattle`系10テーブルに対応）
- `04_ミッション.csv` → ミッション設計書（**マスタデータ生成あり**。`MstMissionEvent`系6テーブルに対応）
- `05_報酬一覧.csv` → 報酬集計シート（参照用。マスタデータ生成なし）

### D-3. 仕様書に記載があるがマスタデータ対応シートが不明なもの

- **MstAutoPlayerSequence（202件）**: 敵AIのシーケンス定義。仕様書に専用シートがなく、ステージ仕様から間接的に生成される。
- **MstEnemyCharacter / MstEnemyOutpost / MstEnemyStageParameter**: 敵キャラ・拠点の詳細パラメータ。`03_降臨バトル.csv` に方針は記載されるが、詳細パラメータの専用シートはない。
- **MstKomaLine（58件）**: コマ（マス目）の配置と効果定義。仕様書に専用シートがない。
- **MstMangaAnimation（32件）**: ストーリー演出マンガアニメーション定義。仕様書に専用シートがない。
- **MstSpeechBalloonI18n（8件）**: キャラの吹き出しセリフ（召喚時・必殺技チャージ時）。仕様書に専用シートがない。
- **MstArtwork / MstArtworkFragment 系**: ストーリークエストクリアで獲得できる原画アートワーク定義。仕様書に専用シートがない。
- **MstUnitAbility**: キャラの特性（アビリティ）定義。仕様書に専用シートがなく、ガシャ設計書のキャラ説明から間接的に設計される。

### D-4. 1つのシートが複数テーブルにまたがるケース

- **`02_施策.csv`** は最も複雑で、以下の複数テーブルの情報を1シートに集約している:
  - MstSeries / MstSeriesI18n（シリーズ定義）
  - MstEvent / MstEventI18n（イベント定義）
  - MstQuest / MstQuestI18n（各クエスト定義）
  - MstStage / MstStageI18n / MstStageEventReward / MstStageEventSetting / MstStageClearTimeReward（ステージ定義・報酬）
  - MstMissionEventDailyBonus / MstMissionEventDailyBonusSchedule（ログインボーナス）
  - MstQuestBonusUnit（コイン獲得クエストのキャラボーナス）
  - MstEventDisplayUnit / MstEventDisplayUnitI18n（クエスト表示キャラ）

- **`03_降臨バトル.csv`** も複数テーブルを内包:
  - MstAdventBattle / MstAdventBattleI18n / MstAdventBattleClearReward / MstAdventBattleRank / MstAdventBattleReward / MstAdventBattleRewardGroup
  - MstEnemyCharacter / MstEnemyCharacterI18n / MstEnemyOutpost / MstEnemyStageParameter

- **`04_ミッション.csv`** が関係するテーブル:
  - MstMissionEvent / MstMissionEventI18n / MstMissionEventDependency
  - MstMissionLimitedTerm / MstMissionLimitedTermI18n
  - MstMissionReward（ミッション報酬部分）
  - MstUnitSpecificRankUp（DropSRキャラのグレードアップ設定はこのシートから読み取れる）

### D-5. 複数シートが1つのテーブルに集約されているケース

- **MstMissionReward（63件）**: `02_施策.csv`（ログインボーナス報酬）・`04_ミッション.csv`（ミッション報酬）・`03_降臨バトル.csv`（降臨バトル期間限定ミッション報酬）の3シートの報酬定義が1テーブルに集約されている。

### D-6. `@_` プレフィックスと番号付きシートの役割の違い

- **番号付きシート（`02_`, `03_`, `04_`等）**: 施策全体の構造定義・一覧（全体設計書）。1シートが複数テーブルに対応。
- **`@_` または `@` プレフィックス**: 個別コンテンツの詳細設計書（ガチャ・パック等）。実マスタデータに直接対応するデータが記載。
- マスタデータ生成に実際に使われるのは主に `@_` / `@` 設計書と `03_` / `04_` シート。

### D-7. `案_` プレフィックスのファイルはAI生成出力物

`案_@_バレンタインガシャ_設計書_260122_1756.csv` と `案_@幼稚園WARS いいジャン祭ピックアップガシャ_設計書_260122_1755.csv` は設計書の旧バージョンではなく、**AIがお知らせ文を生成した出力物**（生成元ファイル名・処理時間が記録されている）。命名が設計書と混同しやすいため注意が必要。
