### 設計書の品質と網羅性
- 全15件（local指定）の設計書が非常に詳細に作成されている
- 各クエストに20種類前後、各キャラに14種類前後のCSVシートが付属し、あらゆる設定が網羅されている
- ID管理、監修資料、クエスト設計、キャラ設計、運営仕様書が整然と構造化されている

### マスタデータ作成の難易度
- 運営仕様書「20260116_地獄楽 いいジャン祭_仕様書」が全体の「マスター」として機能しており、ここから多くの情報を取得できる

- ファイル数が膨大（合計200ファイル以上）であり、全てを人間が手作業で読み込むのは現実的ではない
- 各設計書の構造を理解し、必要な情報を抽出するロジックを構築する必要がある

### 推奨アプローチ
1. **運営仕様書を起点にする**: `20260116_地獄楽 いいジャン祭_仕様書` が全体像を把握するのに最適
2. **ID管理シートで紐付けを確認**: `GLOW_ID 管理` でID体系を確認
3. **各詳細設計書を参照**: クエスト設計書、キャラ設計書から詳細パラメータを取得
4. **監修資料でテキストを確認**: `GLOW_キャラ名称＆キャラセリフ資料` でテキスト系カラムを取得
