# イベント基本設定マスタデータ作成プロンプト

## 作業内容

手順書（イベント基本設定_マスタデータ設定手順書.md）に従って、
添付したイベント基本設定書からマスタデータを作成してください。
設計書: {ファイル名を指定}

新規リソース情報など、データ作成に必要な追加情報があれば、追加情報ファイルとして追加します。
追加情報ファイル:
- {追加ファイル1}
- {追加ファイル2}

## 入力パラメータ

- release_key: {release_keyを指定}
- mst_series_id: {シリーズIDを指定（例: jig、osh、kai）}
- event_id: {イベントIDを指定（例: event_jig01）}
- event_name: {イベント名を指定（例: 地獄楽いいジャン祭）}
- event_type: {イベントタイプを指定（例: SeriesEvent、CollaborationEvent）}

## 出力形式

### 1. マスタデータ

- 手順書の「データ整合性のチェック」を満たしたデータになっていることを必ずチェックし、問題があれば修正完了してから出力する
- Markdown表形式で出力（スプレッドシートへのエクスポート・コピーボタンが正常に表示される形式）
- 以下の3シートを作成:
  1. MstEvent
  2. MstEventI18n
  3. MstEventBonusUnit

## 出力形式（続き）

### 2. 推測値レポート（必須）

作成したデータのうち、以下に該当するものを必ずレポートしてください：

- **添付ファイルにも手順書にも記載がなく、推測で決定したID値やパラメータ値**
- 手順書通りに作成したID値は対象外

**レポート形式:**
```
## 推測値レポート

### [テーブル名].[カラム名]
- 値: [設定した値]
- 理由: [推測した根拠]
- 確認事項: [ユーザーが確認すべき内容]
```

**推測値の例**:
- MstEvent.banner_destination: 設計書に記載なく推測
- MstEvent.has_advent_battle: 設計書に記載なく推測
- MstEventBonusUnit.bonus_multiplier: 設計書に記載なく推測

**重要:** このレポートを怠ると、データインポートエラーや本番不具合のリスクが高まります。推測で決定した値は必ず報告してください。

## 注意事項

### イベント期間について

MstEventのstart_at/end_atは、イベントの開催期間を示します：
- フォーマット: `YYYY-MM-DD HH:MM:SS`
- 時刻はJST（日本標準時）で設定

### バナー遷移先について

MstEvent.banner_destinationは、ホームバナーをタップした際の遷移先を示します：
- `Quest`: クエスト画面
- `AdventBattle`: 降臨バトル画面
- `Gacha`: ガチャ画面
- `Mission`: ミッション画面
- `Shop`: ショップ画面
- `Artwork`: 原画一覧画面
- `EventTop`: イベントトップ画面
- （他7種類あり、詳細は手順書を参照）

### ボーナスユニットについて

MstEventBonusUnitは、イベント期間中に特定のユニットを使用した際のボーナスを設定します：
- bonus_multiplier: ボーナス倍率（例: 1.5 = 1.5倍、2.0 = 2倍）
- 複数のユニットに異なる倍率を設定可能

### I18n設定について

各I18nテーブルは、最低でも日本語（ja）を設定してください。余裕があれば英語（en）、中国語簡体字（zh-CN）、中国語繁体字（zh-TW）も設定してください。

### 外部キー整合性について

以下のリレーションが正しく設定されていることを必ず確認してください：
- MstEventBonusUnit.mst_event_id → MstEvent.id
- MstEventBonusUnit.mst_unit_id → MstUnit.id
- すべてのI18nテーブルの親ID → 親テーブルのid
