## 背景・目的

### 背景・経緯
限界チャレンジ(Dungeon)のインゲームマスタデータの設定量が膨大なため、AIを使ってマスタデータを生成し、プランナーさんの工数削減をしたい。

### 解決したい問題
インゲームマスタデータの設定量が膨大で、プランナーさんの手作業による入力に多大な工数がかかっている。

### 期待する成果
- AIがインゲーム設計要件テキストを解析し、XLSXテンプレートへ自動入力できる
- 既存のプランナーさんの作業フロー（GoogleSpreadSheet）を変えずに導入できる
- 16作品×2ブロック分のXLSXを一括生成できる

### ネクストアクション候補
- P1完了後: テンプレシートを使ったP2の実装に着手
- P2完了後: 1作品分のXLSXが正常生成できることを確認し、P3・P4へ
- P4完了後: 16作品分を一括生成し、P5でGoogleドライブへのアップロード
- P5完了後: プランナーさんへのレビュー依頼・フィードバック収集

### 解決アイデア
- **方針**: GoogleSpreadSheetでインゲーム設計書のテンプレを作り、xlsxファイルとして用意
- インゲーム設計要件テキストをコンテキストとして投げることで、xlsxファイルの指定セルへ自動入力したxlsxを作成
- これをGoogleドライブにアップロードして、既存通りの作業方法でインゲームマスタデータを投入
- **GoogleSpreadSheet採用理由**: 既存プランナーさんの作業方法を変えずにスムーズに作業でき、レベルデザイン調整のたびにAI生成を依頼するフローにすることで持続的に活用できる

### 追記: 汎用式の動的参照ロジック（重要）

| 動的参照 | MATCHパターン | 説明 |
|---------|-------------|------|
| page_id行 | `MATCH("敵ゲートID",E:E,0)+1` | E列の「敵ゲートID」ヘッダーの次行 |
| release_key行 | `MATCH("リリースキー",N:N,0)+1` | N列の「リリースキー」ヘッダーの次行 |
| コマデータ第i行 | `MATCH("■コマ設計",B:B,0)+2+i` | B列の「■コマ設計」から+3〜+7行目 |

シートタイプ別の実際の位置：

| シートタイプ | 「敵ゲートID」行 | page_id行 | 「■コマ設計」行 | データ1行目 |
|------------|---------------|----------|--------------|------------|
| ストーリー/チャレンジ1-2話/降臨バトル | E12 | E13 | B28 or B35 | 行31 or 行38 |
| チャレンジ3-4話/高難度 | E12 | E13 | B29 | 行32 |
| 通常/ボスブロック | E13 | E14 | B16 | 行19 |

### 追記: スプシ1話シートのセル構造（重要）

| セル | 内容 |
|------|------|
| B13 | ゲートタイプ (default) |
| E13 | **page_id** (例: event_jig1_charaget01_00001) |
| H13 | 敵ゲートasset |
| N13 | **release_key** (例: 202601010) |
| B28 | ■コマ設計 ヘッダー |
| D31〜D35 | 行パターンID (空なら行なし) |
| H31〜H35 | コマ高さ |
| J31〜J35 | コマ幅1 (VLOOKUP for layout) |
| L31〜L35 | コマ幅2 ("none"でコマなし) |
| N31〜N35 | コマ幅3 ("none"でコマなし) |
| P31〜P35 | コマ幅4 ("none"でコマなし) |
| R31〜R35 | コマ背景 (全コマ共通) |
| U31〜 | コマ効果1 (koma1), AD=コマ効果2, AM=コマ効果3, AV=コマ効果4 |
| EA28〜ED28 | ★MstPage投入用ヘッダー（EA=ラベル, EB=ENABLE, EC=id, ED=release_key） |
| EB29〜ED29 | MstPageデータ（動的式: MATCH("敵ゲートID",E:E,0)+1 で行参照） |
| EA30〜FR30 | ★MstKomaLine投入用ヘッダー (43列) |
| EB31〜FR35 | MstKomaLineデータ (各コマ行) (動的式: MATCH("■コマ設計",B:B,0)+3〜+7) |

### 追記: MstKomaLine CSV のコマなし時のパターン

| フィールド | コマ存在時 | コマ不在時 |
|-----------|-----------|----------|
| asset_key | 実値 | "" (空) |
| width | 実値 | "" (空) |
| back_ground_offset | IF(幅=1,0,-1) ※1 | "__NULL__" |
| effect_type | "None" or 実値 | "None" |
| effect_parameter1 | 0 or 実値 | "" (空) |
| effect_parameter2 | 0 or 実値 | "" (空) |
| effect_target_side/colors/roles | "All" | "__NULL__" |

※1 bg_offsetについて：ストーリーコンテンツはIF(幅=1,0,-1)で正解CSV一致。チャレンジ/高難度/降臨バトルの既存CSVには特殊値（0.2, 0.3等）があり近似値のみ生成可能。設計書にbg_offset入力列追加で将来対応可能。
