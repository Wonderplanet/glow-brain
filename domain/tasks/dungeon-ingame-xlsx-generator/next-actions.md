# ネクストアクション

## 次にやること

### P1（最優先）: vdブロックテンプレートシートを用意する

- [ ] 既存のインゲーム設計書の構造を調査する
- [ ] 通常ブロックのテンプレートシート構成を確定する（コマ設計の行数・カラム定義）
- [ ] GoogleSpreadSheetでテンプレートを作成する
- [ ] xlsxファイルとしてエクスポートして `outputs/` に保存する
- ※ ボスブロックは通常ブロックのコマ設計の行数が異なるだけなので、通常ブロック完成後に対応

### P2: テンプレシートへのデータ自動入力

- [ ] インゲーム要件テキストのサンプルを `inputs/` に用意する
- [ ] テンプレシートのセル構造を整理する（どのセルに何を入力するかのマッピング定義）
- [ ] AIがインゲーム要件テキストを解析してXLSXへ入力するスクリプトを作成する
- [ ] 1作品分で動作確認する

### P3: インゲーム要件テキストの転用生成

- [ ] 1作品の要件テキストのサンプルを用意する
- [ ] 他16作品への転用ルールを定義する
- [ ] AIによる一括生成・レビュー・調整フローを設計する

### P4〜P6（後続）

- P4: 16作品×2ブロックのXLSX一括生成
- P5: 1つのXLSXにまとめてGoogleドライブへアップロード
- P6: テーブル別シート・目次シートの追加

## 長期的なアクション

- プランナーさんへのレビュー依頼・フィードバック収集
- レベルデザイン調整フローの確立（調整→再生成→アップロードのサイクル化）

## 注意事項

- GoogleSpreadSheetの既存作業フローを壊さないことを最優先に設計する
- レベルデザイン調整が入るたびにAI再生成するフローを前提にする
- P1のテンプレ設計がP2以降の品質に直結するため、丁寧に進める
