## 作業手順

### Step 1: 対象ブロックの特定

`vd-ingame-design-creator/` 配下のフォルダを確認し、`design.md` が存在しないブロックをリストアップしてください。

- `vd_all/` は除外する
- 対象はブロック種別フォルダ（例: `vd_kai_normal_00001/`、`vd_kai_boss_00001/` など）のみ
- 上記16作品 × 2種別（Normal/Boss）= **最大32ブロック**が対象
- すでに `design.md` が存在するブロックはスキップする

### Step 2: サブエージェント並列起動（3ブロックずつ）

未作成ブロックを3つずつグループ化し、**1メッセージで3つのTask toolを並列起動**してください。
全サブエージェントの完了を待ってから、次のグループへ進んでください。

**各サブエージェントへの指示**:

```
以下のスキルをバッチモードで実行してください:
/vd-masterdata-ingame-design-creator 作品ID={作品ID} ブロック種別={Normal|Boss} --batch

完了したら "DONE: {ブロックID}" とだけ出力してください。

作業ディレクトリ:
/Users/junki.mizutani/Documents/workspace/glow/glow-brain-repos/glow-brain-shuna
```

**Task toolパラメータ**:
- `subagent_type`: `general-purpose`
- 3ブロック分を同一メッセージ内で並列起動（依存関係なし）

**引数の例**:
- `vd_kai_normal_00001` → 作品ID: `kai`、ブロック種別: `Normal`
- `vd_osh_boss_00001` → 作品ID: `osh`、ブロック種別: `Boss`

### Step 3: 完了確認

全対象ブロックについて `design.md` が生成されたことを確認して作業完了です。

```
vd-ingame-design-creator/ 配下の全32ブロックフォルダに design.md が存在するか確認
```

## 注意事項

- サブエージェントは `general-purpose` タイプを使用する

- エラーが発生したブロックは個別にリトライする
