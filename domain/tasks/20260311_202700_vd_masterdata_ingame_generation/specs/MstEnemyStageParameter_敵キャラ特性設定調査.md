# MstEnemyStageParameter 敵キャラ特性設定調査

**調査日**: 2026-03-15
**調査目的**: 敵キャラ（MstEnemyStageParameter等）に、味方ユニットと同様の特性（ability）設定が存在するかを明らかにする

---

## 結論

**敵キャラにも特性設定は存在する。** ただし使用率は低く（2.5%）、一部の強敵のみに設定される。

---

## 1. 敵関連テーブル一覧

| テーブル（CSV名） | DBスキーマ名 | 特性設定 |
|---|---|---|
| MstEnemyCharacter | mst_enemy_characters | **なし** |
| MstEnemyStageParameter | mst_enemy_stage_parameters | **あり**（`mst_unit_ability_id1`） |
| MstEnemyOutpost | mst_enemy_outposts | なし |

## 4. テーブル間リレーション

```
MstEnemyStageParameter
  └─ mst_unit_ability_id1 ──→ MstUnitAbility.id
                                  └─ mst_ability_id ──→ MstAbility.ability_type
```

味方ユニットと**同一の特性システムを共有**している。

## 5. 実際に設定されている敵特性（使用状況）

全1073件の MstEnemyStageParameter のうち、特性が設定されているのは **27件（2.5%）** のみ。

| 特性ID | ability_type | 効果 | 使用敵数 |
|---|---|---|---|
| enemy_ability_gust_1_zombie | Guts | 1回まで致命傷を耐える（確率100%） | 5 |
| enemy_ability_gust_3_zombie | Guts | 3回まで致命傷を耐える（確率100%） | 7 |
| enemy_ability_knockback_block | KnockBackBlock | ノックバックを完全無効化 | 15 |
| **合計** | | | **27** |

## 6. 敵が使用可能な特性タイプ

クライアントの `UnitAbilityModel.cs` には、敵が召喚時に状態効果として受け取る特性タイプのリストが定義されている。以下が**敵ユニットに設定可能な特性**の全種類。

### 常時発動型（召喚時付与）

| ability_type | 効果 |
|---|---|
| KnockBackBlock | ノックバック無効化 |
| Guts | 致命傷をParam1回まで耐える |
| PoisonDamageCut | 毒ダメージを Param1% 軽減 |
| BurnDamageCut | 火傷ダメージを Param1% 軽減 |
| StunBlock | スタンを Param1% の確率で無効化 |
| FreezeBlock | 氷結を Param1% の確率で無効化 |
| WeakeningBlock | 弱体化を完全無効化 |

### コマ効果連動型・条件付き発動型も技術上は設定可能

（ただし現状CSVでは未使用）

## 7. 味方ユニットとの比較

| 項目 | 敵ユニット | 味方ユニット |
|---|---|---|
| 特性設定カラム | `mst_unit_ability_id1`（1個のみ） | `mst_unit_ability_id1〜3`（最大3個） |
| 特性の参照先 | MstUnitAbility を直接参照 | MstUnitAbility を直接参照 |
| ランク解放 | なし（常に有効） | `ability_unlock_rank{N}` で段階解放 |
| 実装の共通化 | UnitAbilityModel を共有 | UnitAbilityModel を共有 |
| 設定率 | 2.5%（27/1073件） | 多くのキャラに設定あり |
| 典型的な用途 | 強敵の耐性設定 | キャラ個性の差別化 |

---

## 8. 敵の状態異常への対応

特性が**設定されていない敵**（97.5%）は全ての状態異常が等しく有効。

| 状態異常 | 特性なし敵への影響 | 特性あり敵への影響 |
|---|---|---|
| 毒（Poison） | ダメージ受け続ける | PoisonDamageCut で軽減可能 |
| 火傷（Burn） | ダメージ受け続ける | BurnDamageCut で軽減可能 |
| スタン（Stun） | 行動が一時停止 | StunBlock で無効化可能 |
| 凍結（Freeze） | 行動が一時停止 | FreezeBlock で無効化可能 |
| 弱体化（Weakening） | 攻撃力/防御力低下 | WeakeningBlock で無効化可能 |
| ノックバック | 後退する | KnockBackBlock で無効化可能 |

---

## 9. VD（限界チャレンジ）設計への示唆

- VDブロックでも強敵・ボス敵に特性を設定することが可能
- 現状の実績では `Guts` や `KnockBackBlock` が使われている
- 特性は `MstEnemyStageParameter.mst_unit_ability_id1` に MstUnitAbility の ID を設定するだけで適用される
- 新規特性が必要な場合は MstUnitAbility にレコードを追加する
