# MstAttack 設定パターン詳細

> 参照DBスキーマ: `projects/glow-server/api/database/schema/exports/master_tables_schema.json`（`mst_attacks`テーブル）
> C#データクラス: `projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Data/AutoGenerated/MstAttackData.cs`
> C#トランスレーター: `projects/glow-client/Assets/GLOW/Scripts/Runtime/Core/Data/Translators/AttackDataTranslator.cs`

## 1. 概要

`MstAttack` は**ユニットの攻撃アクション1つ分のメタ情報**を定義するテーブル。
ユニット（`MstUnit`）に対して `attack_kind`（通常攻撃 / 必殺技 / 登場演出）と `unit_grade`（強化段階 0〜5）の組み合わせで複数レコードが紐づく。

実際の攻撃の「何を・どの範囲に・どんな効果で」という詳細は `MstAttackElement`（子テーブル）が担う。

```
MstUnit（ユニット）
  └─(1:N) MstAttack（攻撃アクション）
               ├─ attack_kind: Normal（通常攻撃）
               │     └─(1:N) MstAttackElement（攻撃エレメント）
               ├─ attack_kind: Special（必殺技 grade 1〜5）
               │     └─(1:N) MstAttackElement
               └─ attack_kind: Appearance（登場演出）
                     └─(1:N) MstAttackElement
```

## 3. attack_kind の全 enum 値

| 値 | 意味 | unit_grade | next_attack_interval |
|---|---|---|---|
| `Normal` | 通常の近接攻撃。コンボアクション（連打）の1セット | `0` 固定 | `0` より大きい（例: 75〜100フレーム） |
| `Special` | 必殺技。グレードごとに1レコード | `1`〜`5` | `0` 固定 |
| `Appearance` | 登場演出アクション（召喚時の演出フレーム定義） | `0` 固定 | `0` 固定 |

**C# 実装から確認した内容**（`AttackDataTranslator.cs`）:

```csharp
var baseData = new AttackBaseData(
    EnumListTranslator.ToEnumList<CharacterColor>(mstAttack.KillerColors),
    new KillerPercentage(mstAttack.KillerPercentage),
    new TickCount(mstAttack.ActionFrames),
    new TickCount(mstAttack.NextAttackInterval));

var elements = ToAttackElements(mstElements, mstAttackHitEffectDataList);

return new AttackData(
    new TickCount(mstAttack.AttackDelay),
    baseData,
    elements);
```

`MstAttack` は `AttackBaseData`（フレーム数・インターバル・キラー設定）と複数の `AttackElement`（`MstAttackElement` から変換）を束ねる上位コンテナ。

## 4. killer_colors / killer_percentage の設定パターン

### キラーなし（大多数のケース）

```
killer_colors:     ""（空文字）
killer_percentage: 0
```

### キラーあり

```
killer_colors:     "Green"（単一属性）または "Green,Yellow"（複数属性カンマ区切り）
killer_percentage: 110〜218（属性有利時の追加ダメージ倍率）
```

**実データ分析（全 2,633 件）**:

| attack_kind | キラーなし | キラーあり |
|---|---|---|
| `Appearance` | 438件 | **0件** |
| `Normal` | 1,184件 | **0件** |
| `Special` | 966件 | **45件** |

**キラーは Special 攻撃にのみ設定される**（Appearance・Normal には設定されない）。

**実データ例**:

| id | killer_colors | killer_percentage |
|---|---|---|
| `chara_jig_00101_Special_00001` | `Green` | `110` |
| `chara_jig_00101_Special_00005` | `Green` | `150` |
| `chara_chi_00001_Special_00001` | `Yellow` | `105` |

**設定ポイント**:
- `killer_colors` が空の場合、`killer_percentage` は `0` にする（未使用を明示）
- `killer_percentage` は `100` 以上の値（100% = 等倍）。`110` = 対象属性に10%追加ダメージ
- `Normal` / `Appearance` にはキラーを設定しない（実績上も0件）

## 5. action_frames / attack_delay / next_attack_interval の実績値

`projects/glow-masterdata/MstAttack.csv` 全 2,633 件の統計。

### action_frames のパーセンタイル分布

| attack_kind | 最小 | P25 | 中央値 | P75 | 最大 | 件数 |
|---|---|---|---|---|---|---|
| `Appearance` | **50** | **50** | **50** | **50** | **50** | 438件（**全件50固定**） |
| `Normal` | 0 | 50 | 67 | 79 | 145 | 1,184件 |
| `Special` | 69 | 121 | 134 | 167 | 261 | 966件 |

- **Appearance は全件 50 frames 固定**（アニメーションが統一されている）
- **Normal の中央値は 67 frames**（約1.1秒）。50〜90 が最も多い帯域
- **Special の中央値は 134 frames**（約2.2秒）。Normal の約2倍

### attack_delay

| attack_kind | 実績値範囲 | 代表値 |
|---|---|---|
| `Normal` | 0〜175フレーム | 0, 25, 27, 33 |
| `Special` | 0〜175フレーム | 0, 50, 67, 83 |
| `Appearance` | 0 | 0（固定） |

### next_attack_interval の分布（Normal 限定・実績上位）

| 値 | 件数 | 意味（約秒数） |
|---|---|---|
| `100` | 255件（最多） | 約1.7秒 |
| `50` | 246件 | 約0.8秒 |
| `75` | 179件 | 約1.3秒 |
| `60` | 63件 | 約1.0秒 |
| `80` | 40件 | 約1.3秒 |
| `150` | 37件 | 約2.5秒 |

**推奨設定値**: `50` / `75` / `100` の3択が全体の約75%を占める標準帯域。

| attack_kind | next_attack_interval |
|---|---|
| `Normal` | 50〜1000フレーム（50 / 75 / 100 が最多） |
| `Special` | `0` 固定 |
| `Appearance` | `0` 固定 |

> **注意**: `action_frames` はアニメーション全体のフレーム長。`attack_delay` はアクション開始から最初のヒット判定までの遅延。`next_attack_interval` は攻撃完了後に次の攻撃サイクルを開始するまでの休止フレーム。

## 6. IDの命名規則

### 基本パターン

```
{mst_unit_id}_{attack_kind}_{unit_grade5桁}
```

| 構成要素 | 説明 | 例 |
|---|---|---|
| `{mst_unit_id}` | ユニットID | `chara_dan_00001`, `e_kai_00101` |
| `{attack_kind}` | `Normal`、`Special`、`Appearance` | `Normal`, `Special` |
| `{unit_grade5桁}` | グレードをゼロ埋め5桁 | `00000`（grade=0）〜`00005`（grade=5） |

### 具体例

| id | mst_unit_id | attack_kind | unit_grade |
|---|---|---|---|
| `chara_dan_00001_Normal_00000` | `chara_dan_00001` | Normal | 0 |
| `chara_dan_00001_Special_00001` | `chara_dan_00001` | Special | 1 |
| `chara_dan_00001_Special_00005` | `chara_dan_00001` | Special | 5 |
| `e_kai_00101_vd_Normal_Yellow_Appearance_00001` | `e_kai_00101_vd_Normal_Yellow` | Appearance | 0 |

> **VD敵キャラの命名**: 敵キャラ（`e_xxx` / `c_xxx`）の場合、`mst_unit_id` 部分が長くなる（例: `e_kai_00101_vd_Normal_Yellow`）。

## 7. MstAttack と MstAttackElement のリレーション

```
MstAttack.id（= MstAttackElement.mst_attack_id）
  └─(1:N) MstAttackElement（sort_order順に複数行）
               ├─ sort=1: メインヒット（attack_type=Direct）
               ├─ sort=2: サブエフェクト（attack_type=Direct）
               └─ sort=3: バフ付与（attack_type=Direct / None?）
```

**C#実装での変換ロジック**（`AttackDataTranslator.cs`）:

```csharp
// sort_order 順に並べて処理
var sortedMstElements = mstElements.OrderBy(mstElement => mstElement.SortOrder);
```

子テーブル `MstAttackElement` は `sort_order` 順に処理され、複数のヒット判定が逐次発動する（マルチヒット構成）。

## 8. ユニット1体あたりの MstAttack レコード構成

1ユニットに対して通常以下の構成:

```
chara_xxx_yyyyyyy_Normal_00000     ... 通常攻撃（1件）
chara_xxx_yyyyyyy_Special_00001    ... 必殺技 grade 1（1件）
chara_xxx_yyyyyyy_Special_00002    ... 必殺技 grade 2（1件）
chara_xxx_yyyyyyy_Special_00003    ... 必殺技 grade 3（1件）
chara_xxx_yyyyyyy_Special_00004    ... 必殺技 grade 4（1件）
chara_xxx_yyyyyyy_Special_00005    ... 必殺技 grade 5（1件）
```

Appearance は登場演出があるキャラのみ追加:
```
e_xxx_yyyyy_zzzzzz_Appearance_00001  ... 登場演出（1件）
```

## 9. 注意事項まとめ

| 番号 | 注意事項 |
|------|---------|
| ① | `Normal` の `unit_grade` は必ず `0`、`next_attack_interval` は `0` より大きい値を設定する |
| ② | `Special` の `next_attack_interval` は必ず `0`。必殺技発動後は自動的に終わるためインターバル不要 |
| ③ | `killer_colors` が空の場合は `killer_percentage` も `0` にする |
| ④ | `killer_percentage` は `100` = 等倍。キラー効果は `100` 超の値で有効（例: `110` = 1.1倍） |
| ⑤ | `id` の命名は `{mst_unit_id}_{attack_kind}_{unit_grade5桁}` の形式を厳守（自動生成システムが依存している可能性） |
| ⑥ | `action_frames` はアニメーションの総フレーム数に対応するため、アニメーションの実装と合わせる必要がある |
| ⑦ | VD敵キャラ（`e_xxx` / `c_xxx`）にも MstAttack レコードが必要（インゲームでの攻撃動作定義） |
