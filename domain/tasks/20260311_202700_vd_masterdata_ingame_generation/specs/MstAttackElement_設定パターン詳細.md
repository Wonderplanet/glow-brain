## 1. 概要

`MstAttackElement` は**攻撃アクション1回の中の個々の攻撃エレメント（ヒット判定・効果付与）**を定義するテーブル。
`MstAttack`（親テーブル）1件に対して `sort_order` 順に複数行が紐づき、マルチヒットや複合効果を表現する。

```
MstAttack（攻撃アクション）
  └─(1:N) MstAttackElement（攻撃エレメント）
               ├─ sort=1: メインダメージ（Foe に Damage）
               ├─ sort=2: サブダメージ（Foe に Damage / 追加ヒット）
               └─ sort=3: バフ付与（Self / Friend に Effect）
```

**MstAttack との役割分担**:
- `MstAttack` → 「いつ・どのくらいの時間で・キラー属性は何か」を定義（攻撃のメタ情報）
- `MstAttackElement` → 「どこに・何を・どんな効果で当てるか」を定義（攻撃の実体）

**C# 実装から確認した内容**（`AttackDataTranslator.cs`）:

```csharp
if (mstElement.AttackType == AttackType.None)
{
    // SubElement として処理
    var subElement = AttackElementDataTranslator.ToAttackSubElement(mstElement, hitEffectData);
    subElements.Add(subElement);
    continue;
}
// attack_type が None 以外 → AttackElement として処理
```

> **注意**: `attack_type=None` はCSV上では存在しない（実績 0件）が、C#コード上は `AttackType.None` を持つ SubElement という概念がある。SubElement は親の AttackElement に付随する効果として処理される。

## 12. IDの命名規則

### 基本パターン

```
{mst_attack_id}（親ID）＋ 追番サフィックス
```

#### 1エレメント（単純なケース）

```
mst_attack_id = chara_dan_00001_Normal_00000
→ id          = chara_dan_00001_Normal_00000  （= mst_attack_idと同値）
```

#### 複数エレメント（サフィックスあり）

```
mst_attack_id = chara_dan_00001_Special_00001
→ id（sort=1） = chara_dan_00001_Special_00001    （メインヒット）
→ id（sort=2） = chara_dan_00001_Special_00001a   （サブエフェクト、a〜zサフィックス）
```

#### さらに多段の場合

```
→ id（sort=3） = chara_dan_00001_Special_00001b
→ id（sort=4） = chara_dan_00001_Special_00001c
```

#### マルチヒット（通常攻撃の複数ヒット）

```
mst_attack_id = chara_dan_00002_Normal_00000
→ id（sort=1） = chara_dan_00002_Normal_00000      （1ヒット目）
→ id（sort=2） = chara_dan_00002_Normal_00000_01   （2ヒット目: `_01` サフィックス）
```

## 13. 注意事項まとめ

| 番号 | 注意事項 |
|------|---------|
| ① | `sort_order` の重複は許可されない（同一 `mst_attack_id` 内でユニークにする）。重複するとソート結果が不定になる |
| ② | `effect_type=None` の場合、`effective_count`・`effective_duration`・`effect_parameter` は `0` を設定する（null 代わり） |
| ③ | `damage_type=None` の場合（バフ付与専用エレメント）、`power_parameter` は `0` を設定する |
| ④ | `target=Self` の場合、`range_end_parameter=0`（範囲0 = 自分のみ）を設定するのが通例 |
| ⑤ | `max_target_count=-1` は無制限ターゲット。`Direct` + 全範囲攻撃の場合などに使用 |
| ⑥ | `effective_count=-1` と `effective_duration=-1` の組み合わせは「永続効果」を意味する（ステージ終了まで残る） |
| ⑦ | `hit_effect_id` は通常攻撃に設定（`dageki_1` 等）。必殺技のバフ付与エレメント（effect専用）では空文字にする |
| ⑧ | `PlaceItem` と `Deck` は特殊タイプのため、実装の挙動を確認せずに新規追加しないこと |
| ⑨ | `target_mst_series_ids` / `target_mst_character_ids` で特定キャラ/シリーズに絞り込む際は、存在するIDのみを指定する |
| ⑩ | `is_hit_stop=1` はヒットストップ演出を発生させる。通常攻撃には `1`、バフ付与エレメントには `0` を設定するのが標準 |
