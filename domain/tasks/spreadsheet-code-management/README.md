# spreadsheet-code-management

## 概要

インゲーム/イベント設計書スプレッドシートをコード管理できるようにし、Claude Codeが設計書を理解してマスタデータ生成式を追加したスプシテンプレートを作成・運用できる仕組みを構築する。

## 背景・経緯

インゲームやイベント設計のスプシファイルを、claude-codeを使って拡張して、設計書から直接マスタデータを作成できるスプレッドシートのテンプレを作成・運用できるようにしたい。

## 解決したい問題

スプシとして作成された設計書をCSVとしてダウンロードしてClaude Codeに投げて、マスタデータを一括生成させるようにチャレンジしているが、1文字も間違えず正確に大量のデータを作成することはAIには向いていない。

そのため、マスタデータを直接生成するのではなく、**設計書スプシにマスタデータを作成する式を追加する**アプローチに変更。こうすることで：
- 正確にマスタデータを作成できる
- プランナーがどの設計箇所からどのマスタデータが生成されたか確認できる
- 運用がしやすい

## 期待する成果

- **Claude Codeのコンテキストとしてスプレッドシートを活用できる**（XLSX→CSV/JSON変換による）
- マスタデータ入力の工数削減（今まではプランナーが手入力していた）
- 入力方法のブレ・ミスがなくなる（人によってやり方や正確さが変わる問題の解消）
- 設計書とマスタデータのトレーサビリティ確保

## ネクストアクション候補

1. インゲーム設計書スプレッドシートのテンプレート作成
2. 既存設計書スプレッドシートへのマスタデータ生成式の追加
3. コード管理仕組みの構築（XLSX→CSV/JSON変換パイプライン）

## 解決アイデア

- **openpyxlを使ってXLSX→CSV/JSON変換**: openpyxlでXLSXを読み込み、実データはCSV、式/書式情報はJSONに変換するPythonスクリプトを作成
- **コマンド一発で実行できるCLIツールとして整備**: 山手コマンドでXLSXをCSV/JSONに変換できるようにする
- **Claude Codeスキルとして組み込む**: XLSX読み込み→CSV/JSON化処理をClaude Codeスキルとして整備し、設計書分析に役立てる
