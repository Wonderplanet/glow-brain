# 【仕様確認】バトル準備ダイアログのスタミナ不足時の挙動について

【詳細】
スタミナブーストのバトル準備ダイアログで、
周回に必要なスタミナが不足した状態で周回回数を「2」以上に設定し、
スタミナ不足ダイアログからスタミナを回復させた後にバトル準備ダイアログへ戻ると、
スタミナブーストの周回回数が「1」に戻ってしまいます。

＜資料の記載＞
バトル準備ダイアログが開くタイミングでは、毎回1が選択された状態になります。

周回回数が「1」に戻ってしまう挙動は資料通りとなりますが、
ユーザーが周回回数を変更しないままバトルを開始してしまう懸念が発生する可能性がございます。
こちらに関しましては、現在の挙動で問題ありませんでしょうか。
ご確認のほど、よろしくお願いいたします。
